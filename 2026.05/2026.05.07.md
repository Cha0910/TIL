### [공원 산책](https://school.programmers.co.kr/learn/courses/30/lessons/172928)

```cpp
#include <string>
#include <vector>

using namespace std;
pair<int, int> check(const vector<string>& park, const string& route, pair<int, int> pos){
    int y = pos.first, x = pos.second;
    int dy[4] = {-1, 1, 0, 0};
    int dx[4] = {0, 0, -1, 1};
    int dist = route[2] - '0';
    int dir = 0;
    if(route[0] == 'N') dir = 0;
    else if(route[0] == 'S') dir = 1;
    else if(route[0] == 'W') dir = 2;
    else if(route[0] == 'E') dir = 3;
    
    for(int i=0; i<dist; i++){
        y += dy[dir];
        x += dx[dir];
        if(y<0 || y>=park.size() || x<0 || x>=park[0].size() || park[y][x] == 'X')
            return {-1, -1};
    }
    
    return {y, x};
}

vector<int> solution(vector<string> park, vector<string> routes) {
    vector<int> answer;
    pair<int, int> pos;
    for(int i=0; i<park.size(); i++)
        for(int j=0; j<park[i].size(); j++)
            if(park[i][j] == 'S') pos = {i, j};
    
    for(const string& route : routes){
        pair<int, int> next = check(park, route, pos);
        if(next.first != -1) pos = next;
    }
    return {pos.first, pos.second};
}
```

* **구현** 문제
* **이동 가능 여부**를 판별해 최종 위치를 찾으면 됨

### [[PCCP 기출문제] 2번 / 석유 시추](https://school.programmers.co.kr/learn/courses/30/lessons/250136)

```cpp
#include <string>
#include <vector>
#include <queue>
#include <set>

using namespace std;
void BFS(vector<vector<int>>& land, vector<int>& sum, int i, int j){
    if(land[i][j] == 0) return;
    queue<pair<int, int>> q;
    set<int> visit;
    int count = 0;
    
    q.push({i, j});
    count++;
    land[i][j] = 0;
    visit.insert(j);
    
    int dy[4] = {-1, 1, 0, 0};
    int dx[4] = {0, 0, -1, 1};
    
    while(!q.empty()){
        pair<int, int> pos = q.front();
        q.pop();
        int currY = pos.first;
        int currX = pos.second;
        
        for(int k=0; k<4; k++){
            int y = currY + dy[k];
            int x = currX + dx[k];
            
            if(y<0 || y>=land.size() || x<0 || x>=land[0].size() || land[y][x] == 0) continue;
            
            q.push({y, x});
            count++;
            land[y][x] = 0;
            visit.insert(x);
        }
    }
    
    for(const int x : visit) sum[x] += count;
}

int solution(vector<vector<int>> land) {
    int answer = 0;
    vector<int> sum(land[0].size(), 0);
    
    for(int i=0; i<land.size(); i++){
        for(int j=0; j<land[i].size(); j++){
            BFS(land, sum, i, j);
        }
    }
    
    for(int i=0; i<sum.size(); i++)
        answer = max(sum[i], answer);

    return answer;
}
```

* **BFS**를 활용한 문제
* 방문한 `land`는 `0`으로 바꿔 방문 표시를 함
* 최종적으로 필요한 데이터는 `x` 지점을 팠을 때 얻을 수 있는 석유의 양이므로 `x`가 중복되는 경우를 제거하기 위해 **set**을 사용

## 3. 프로세스와 스레드

### 프로세스
* 운영체제로부터 시스템 자원을 할당받아 실행되는 프로그램의 **인스턴스**
* 프로세스가 생성되면 아래의 **4가지**의 메모리 영역으로 **분리**해 할당
    * `Code`: 실제로 작성한 코드가 기계어로 번역되어 저장
    * `Data`: 프로그램의 전역 변수, static 변수, const 변수, 
    * `Stack`: 함수 호출과 관련된 데이터를 임시로 저장
    * `Heap`: 런타임 중 동적으로 할당되는 데이터를 저장

### PCB (Process Control Block)
* 프로세스의 **메타데이터**를 저장한 데이터 블록
* 프로세스가 생성될 때 같이 생성되며, 소멸할 때 같이 소멸함
* 운영체제는 **PCB**의 정보를 기반으로 **스케줄링**을 수행

### 프로세스 컨텍스트 스위칭
* 하나의 **CPU**가 실행 중인 **프로세스를 전환**하는 과정
* **CPU**는 한 번에 **하나의 프로세스만 실행**할 수 있으므로 여러 개의 프로세스를 번갈아 가면서 실행함
* **프로세스 컨텍스트 스위칭**은 **스케줄러** 혹은 **인터럽트**에 의해 발생
* **프로세스 컨텍스트 스위칭**은 **PCB**를 저장 및 로드하는 과정, **캐시 메모리 초기화**, **스케줄링 비용** 으로 인한 **오버헤드**가 발생

### 멀티 프로세스
* **하나의 프로그램**을 **여러 개의 프로세스**로 실행하는 것 
* 하나의 **부모 프로세스**가 여러개의 **자식 프로세스**를 생성하여 다중 프로세스 구조를 구성
* 각 프로세스는 `PID(Process ID)`를 갖고 있어 **자식과 부모 간의 통신**이 가능
* 각 프로세스가 **독립적**으로 실행되므로 프로그램의 **안정성**과 **확장성**이 향상됨
* 하지만 **프로세스 컨텍스트 스위칭**과 자원 공유를 위한 **IPC** 과정에서 **오버헤드**가 발생하며 **코드 복잡도** 또한 올라감

### IPC (Inter-Process Communication)
* 각 프로세스는 **별도의 메모리 주소**를 할당받기 때문에 기본적으로 **다른 프로세스에 접근이 불가능**

* **멀티 프로세스**는 **IPC**를 통해 다른 프로세스와 **공유 데이터**를 관리

* 독립적인 메모리 공간을 갖기에 높은 **오버헤드**와 **동기화** 문제가 발생

* 대표적인 **IPC**는 아래의 5가지 방법이 있음
    * `공유 메모리`: 여러 프로세스가 공유된 메모리 자원에 접근, 빠르지만 동기화 문제 발생
    * `소켓`: TCP, UDP를 통해 데이터를 주고받는 방식
    * `익명 파이프`: 부모 자식 프로세스 간 데이터를 주고 받는 단방향 파이프
    * `명명 파이프`: 파이프에 이름을 지어 사용, 연관이 없는 프로세스끼리도 사용 가능
    * `메시지 큐`: 데이터를 메시지 형태로 묶어 큐를 통해 전달

### 스레드
* 한 프로세스 내에서 실행되는 **실행 흐름**
* 스레드는 프로세스의 `Stack`만 할당받아 복사하며, 다른 3개의 영역은 다른 스레드들과 **공유하여 사용**

### TCB (Thread Control Block)
* 스레드의 **메타데이터**를 저장한 데이터 블록
* **PCB** 내부에 저장되어 있으며 **PCB**보다 **작은 크기**를 가짐
* **PCB**와 유사하게 동작

### 스레드 컨텍스트 스위칭
* **하나의 프로세스** 내에서 실행되는 **스레드를 전환**하는 것
* **TCB**는 **PCB**보다 작은 크기를 갖기에 저장 및 로드로 인한 오버헤드가 적음
* 공유된 메모리를 사용하기에 **캐시 메모리**가 초기화되지 않음
* 공유된 메모리에 접근할 때 **동기화** 문제가 발생할 수 있음

### 멀티 스레드
* **하나의 프로세스**에서 **여러 개의 스레드**가 실행되는 것
* 일반적으로 **멀티 프로세스**보다 **멀티 스레드**를 기본으로 함
* 스레드는 프로세스보다 가벼운 용량을 갖기에 **생성 및 소멸이 간단**함
* 스레드는 코드, 데이터, 힙 영역을 공유하기에 **오버헤드**가 적음
* 하지만 자원 공유로 인한 **안정성 하락**, 동기화로 인한 **성능 저하** 발생과 **복잡도 상승**이 존재
