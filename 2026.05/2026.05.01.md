### [삼총사](https://school.programmers.co.kr/learn/courses/30/lessons/131705)

```cpp
#include <string>
#include <vector>

using namespace std;

int combinationCount(const vector<int>& number, vector<int>& current, int start, int r){
    int result = 0;
    if(current.size() == r){
        int sum = 0;
        for(int i=0; i<current.size(); i++)
            sum += current[i];
        return sum == 0;
    }
    
    for(int i=start; i<number.size(); i++){
        current.push_back(number[i]);
        result += combinationCount(number, current, i+1, r);
        current.pop_back();
    }
    return result;
}

int solution(vector<int> number) {
    vector<int> temp;
    int answer = combinationCount(number, temp, 0, 3);
        
    return answer;
}
```

* **조합**을 구현하여 해결
* `number`의 길이가 최대 `13`밖에 안되므로 3중 반복문이 더 간단하긴 함

### [이진 변환 반복하기](https://school.programmers.co.kr/learn/courses/30/lessons/70129)

```cpp
#include <string>
#include <vector>

using namespace std;

string convert(string& str, int& sum){
    string result;
    int count = 0;

    // 1 개수 세기
    for(int i=0; i<str.length(); i++) {
        if(str[i] == '1') count++;
        else sum++; // 제거된 0 개수 증가
    }

    // 10진수로 변환
    while(count > 0){
        result += (count % 2) + '0';
        count /= 2;
    }
    return result;
}
vector<int> solution(string s) {
    int sum = 0;
    int count = 0;
    while(s != "1"){
        s = convert(s, sum);
        count++;
    }
    return {count, sum};
}
```

* **구현**, **문자열** 문제
* 10진수로 변환 시, 정렬 순서는 의미 없으므로 `reverse()`를 따로 안 해줘도 됨

### [봉인된 주문](https://school.programmers.co.kr/learn/courses/30/lessons/389481)

```cpp
#include <string>
#include <vector>
#include <algorithm>
#include <cmath>
using namespace std;

string solution(long long n, vector<string> bans) {
    string answer = "";

    // 밴 문자열을 정렬
    sort(bans.begin(), bans.end(), [](const string& A, const string& B){
        if(A.length() != B.length()) return A.length() < B.length();
        return A < B;
    });
    
    // 문자열을 index로 변환하여 비교
    for(const string& spell : bans){
        long long pos = 0;
        long long index = 0;
        for(int i=spell.length()-1; i>=0; i--){
            pos += (spell[i] - 'a' + 1) * pow(26, index);
            index++;
        }
        if(pos <= n) n++;   // 밴 된 문자열이 구할 문자열 보다 앞에 있던 경우, 커서를 뒤로 한칸 옮김
    }
    
    // 최종 index를 문자열로 변환
    while(n>0){
        // 1부터 시작하는 26진법 이므로 1을 빼줘야 함
        answer += (n-1) % 26 + 'a';
        n = (n-1) / 26; 
    }
    
    reverse(answer.begin(), answer.end());
    return answer;
}
```

* **파싱** 문제
* 문제의 조건대로 정렬된 알파벳 문자열은 사실상 **26진법**의 수라고 볼 수 있음
* 하지만 다른 진법과 달리, `1~26`의 값을 가지므로 일반적인 진법 변환의 방법은 적용되지 않음
* `n`에서 `1`을 빼주어 나머지가 `0일 경우 a`, `25일 경우 z`가 되게끔 **보정**해줘야 함

## 3. 템플릿 메소드 패턴 (Template Method Pattern)

* 여러 클래스에서 공통으로 사용하는 메서드를 템플릿화 하여 **상위 클래스에서 정의** 한 후, **하위 클래스마다 세부 동작을 구현**하는 디자인 패턴

* **Abstract Class(추상 클래스):** 템플릿 메서드를 정의한 클래스, 공통적인 로직은 구현하며 하위 클래스에서 구현할 로직을 선언만 함

* **Concrete Class(구현 클래스):** 추상 클래스를 상속받아, 추상 메서드를 구체적으로 구현한 클래스


### 주요 메서드
* **Template Method(템플릿 메서드):** 상위 클래스에서 정의한 메서드, **클래스의 구조를 정의**하며 보통 오버라이딩을 막아 **무결성**을 보장함

* **Abstract Method(추상 메서드):** 자식 클래스에서 **반드시 구현**해야 하는 메서드

* **Hook Method(훅 메서드):** **오버라이딩**을 통해 부모의 템플릿 메서드의 **영향이나 순서를 제어**하고 싶을 때 사용되는 메서드

### 장점
* 상위 추상 클래스로 로직을 공통화하여 **코드의 중복**을 줄일 수 있음
* 저수준 구성 요소에서 고수준 구성 요소를 호출하는 것을 막고, 반대로 고수준 구성 요소가 저수준 구성 요소를 호출하는 것은 허용하는 **할리우드 원칙**을 만족함

### 단점
* 상위 클래스에서 **너무 많은 템플릿 메서드**를 선언 시, 하위 클래스의 **유연성이 제한** 됨
* 반대로 **너무 많은 추상 메서드**가 구현되면, 클래스의 생성 및 관리가 어려워짐
* 부모 클래스에서 정의한 로직을 **오버라이딩** 하기에, **리스코프 치환 원칙(LSP)** 을 위반할 수 있음

## 4. 하위 클래스 샌드박스 패턴 (Subclass Sandbox Pattern)

* **상위 클래스**에서 다양한 **기능**들을 제공하는 메서드를 `protected`로 제공

* 하위 클래스는 **샌드박스 메서드**인 `activate()`를 오버라이딩 하여 `protected`로 제공된 기능 메서드를 호출하여 구현

### 장점
* 공통적인 기능을 상위 클래스에 몰아넣어 **코드의 중복**을 줄임
* 상위 클래스에서 구현된 메서드만 사용하기에, **높은 안정성**을 가짐
* 하위 클래스는 상위 클래스하고만 **결합도**를 갖게 됨

### 단점
* **상위 클래스**에 기능이 많아질수록 너무 커져 다른 외부 시스템과 **높은 결합도**를 갖게 됨
