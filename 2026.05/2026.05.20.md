## 2. 프로그래머스 문제 풀이

### [숫자 짝꿍](https://school.programmers.co.kr/learn/courses/30/lessons/131128)

```cpp
#include <string>
#include <vector>

using namespace std;

string solution(string X, string Y) {
    string answer = "";
    vector<int> countX(10, 0), countY(10, 0);
    for(int i=0; i<X.length(); i++) countX[X[i] - '0']++;
    for(int i=0; i<Y.length(); i++) countY[Y[i] - '0']++;
    for(int i=9; i>=0; i--){
        int count = min(countX[i], countY[i]);
        answer += string(count, '0' + i);
    }
    
    if(answer.length() == 0) return "-1";
    if(answer[0] == '0') return "0";
    
    return answer;
}
```

* **그리디** 유형의 문제
* **높은 수를 앞**에 두는 것이 가장 높은 해를 가짐
* 각 수를 카운팅 해 동일한 수가 몇 개인지 확인
* 가장 앞의 수가 `0`인 경우 0이 연속해서 있을 수 있으므로, `0`으로 바꿔줌

## 3. 32비트 OS와 64비트 OS

### CPU 아키텍처

* **CPU 아키텍처**에서 32비트와 64비트는 CPU의 **레지스터가 한번에 처리할 수 있는 비트 수**를 의미

* **32비트**는 한번에 32비트 단위로 데이터를 처리하며, **64비트**는 한번에 64비트 단위로 데이터를 처리

### 32비트 CPU와 64비트 CPU의 차이

* 두 CPU의 가장 큰 차이는 **표현할 수 있는 메모리 범위**

* **32비트**는 `0 ~ 2^32-1`의 범위를 표현할 수 있으며 (4GB), **메모리 또한 최대 4GB**만 가질 수 있음

* **64비트**는 최대 `0 ~ 2^64-1`의 범위를 표현할 수 있지만 (16EB, 약 100만 TB), **하드웨어적인 한계**로 **테라바이트 단위의 메모리**를 가질 수 있음

### 32비트 OS

* 최대 `4GB`의 메모리를 사용할 수 있음 

* 각 프로세스는 최대 `2GB`의 **가상 메모리**를 사용할 수 있음 (**유저 모드**, **커널 모드**가 각각 2GB 사용)

* **PAE**를 통해 메모리를 늘릴 수는 있지만, **오버헤드**가 존재

* **64비트** 소프트웨어는 실행할 수 없음

### 64비트 OS

* 이론적으로 최대 `16EB`의 메모리를 사용할 수 있으며, **하드웨어적 한계**로 제한 (윈도우 11의 경우 최대 6TB)

* 각 프로세스는 최대 `128TB`의 **가상 메모리**를 사용할 수 있음 (현대의 CPU는 메모리 표현에 48 비트를 사용함)

* 32비트, 64비트 소프트웨어 둘 다 실행 가능

* 윈도우의 경우, 32비트 소프트웨어는 **WOW64**라는 에뮬레이터를 통해 실행
