### [구명보트](https://school.programmers.co.kr/learn/courses/30/lessons/42885)

```cpp
#include <string>
#include <vector>
#include <deque>
#include <algorithm>
using namespace std;

int solution(vector<int> people, int limit) {
    int answer = 0;
    sort(people.begin(), people.end());
    deque<int> dq(people.begin(), people.end());
    while(!dq.empty()){
        if(dq.size() > 1 && dq.front() + dq.back() <= limit){
            dq.pop_front();
            dq.pop_back();
        }
        else dq.pop_back();
        answer++;
    }
    return answer;
}
```

* **그리디** 유형의 문제
* 구명보트에는 **최대 2명만 탑승 가능**하므로 현재 가장 무거운 사람과 가장 가벼운 사람을 같이 태우는 게 가장 최선
* **덱**이나 **투 포인터**를 사용하면 풀이 가능

### [택배 배달과 수거하기](https://school.programmers.co.kr/learn/courses/30/lessons/150369)

```cpp
#include <string>
#include <vector>
#include <queue>

using namespace std;

long long solution(int cap, int n, vector<int> deliveries, vector<int> pickups) {
    long long answer = 0;
    priority_queue<int> deliveries_pq, pickups_pq;
    for(int i=0; i<n; i++){
        for(int j=0; j<deliveries[i]; j++) deliveries_pq.push(i+1);
        for(int j=0; j<pickups[i]; j++) pickups_pq.push(i+1);
    }
    
    while(!deliveries_pq.empty() || !pickups_pq.empty()){
        int dist = 0;
        if(!deliveries_pq.empty()) dist = deliveries_pq.top();
        if(!pickups_pq.empty()) dist = max(dist, pickups_pq.top());

        for(int i=0; i<cap; i++){
            if(!deliveries_pq.empty()) deliveries_pq.pop();
            if(!pickups_pq.empty()) pickups_pq.pop();
        }
        answer += dist;
    }
    return answer * 2;
}
```

* **그리디** 유형의 문제
* 배달과 수거는 결국 **독립적**임
* **우선순위 큐**에 각 집의 배달, 수거해야 하는 상자 저장
* 가장 먼 곳부터 상자를 꺼내면 최소 거리를 구할 수 있음

### [n + 1 카드게임](https://school.programmers.co.kr/learn/courses/30/lessons/258707)

```cpp
#include <string>
#include <vector>
#include <unordered_set>

using namespace std;
bool check(unordered_set<int>& hands, unordered_set<int>& defaults, int n, int& coin){
    for(const int value : defaults){
        if(defaults.count(n-value)){
            defaults.erase(value);
            defaults.erase(n-value);
            return true;
        }
    }
    
    for(const int value : defaults){
        if(hands.count(n-value)){
            defaults.erase(value);
            hands.erase(n-value);
            coin--;
            if(coin < 0) return false;
            return true;
        }
    }
    
    for(const int value : hands){
        if(hands.count(n-value)){
            hands.erase(value);
            hands.erase(n-value);
            coin-=2;
            if(coin < 0) return false;
            return true;
        }
    }
    return false;
}

int solution(int coin, vector<int> cards) {
    int answer = 0;
    int cur = cards.size()/3;
    int n = cards.size();
    unordered_set<int> hands, defaults;
    
    for(int i=0; i<cur; i++) defaults.insert(cards[i]);
    
    while(cur <= n){
        answer++;
        for(int i=cur; i<cur+2; i++) hands.insert(cards[i]);
        cur+=2;
        if(!check(hands, defaults, n+1, coin)) break;
    }
        
    return answer;
}
```

* **그리디**, **시뮬레이션** 유형의 문제
* 코인을 사용하지 않아도 되는 처음 카드는 `defaults`에 저장
* 모든 뽑은 카드는 `hands`에 저장한 후 **필요한 경우에 코인을 사용**
* 각 라운드를 넘기기 위해서는 **코인을 최소한으로 사용하는 것을 우선시** 해야함

### [후보키](https://school.programmers.co.kr/learn/courses/30/lessons/42890)


```cpp
#include <string>
#include <vector>
#include <unordered_map>
#include <bitset>
#include <algorithm>
using namespace std;

bool isUniqueness(const vector<vector<string>>& relation, const vector<int>& combination){
    unordered_map<string, int> map;
    for(int i=0; i<relation.size(); i++){
        string str = "";
        for(int j=0; j<combination.size(); j++)
            str += relation[i][combination[j]];
        map[str]++;
        if(map[str] > 1) return false;
    }
    return true;
}

bool isMinimality(vector<bitset<8>>& candidates, bitset<8>& uniqueness){
    for(int i=0; i<candidates.size(); i++){
        if((candidates[i] & uniqueness) == candidates[i]) return false;
    }
    return true;
}

void getUniqueness(int n, int start, vector<int>& com, vector<string>& uniqueness,
          const vector<vector<string>>& relation, int& answer){
    if(isUniqueness(relation, com)){
        string bit = "00000000";
        for(int i=0; i<com.size(); i++) bit[7-com[i]] = '1';
        uniqueness.push_back(bit);
        return;
    }
    for(int i=start; i<n; i++){
        com.push_back(i);
        getUniqueness(n, i+1, com, uniqueness, relation, answer);
        com.pop_back();
    }
}

int solution(vector<vector<string>> relation) {
    int answer = 0;
    vector<int> com;
    vector<string> uniqueness;
    vector<bitset<8>> candidates;
    com.reserve(relation[0].size());
    
    getUniqueness(relation[0].size(), 0, com, uniqueness, relation, answer);
    sort(uniqueness.begin(), uniqueness.end());
    
    for(int i=0; i<uniqueness.size(); i++){
        bitset<8> key(uniqueness[i]);
        if(isMinimality(candidates, key)){
            candidates.push_back(key);
        }
    }
    
    return candidates.size();
}
```

* **백트래킹**, **비트마스킹**을 활용하는 문제
* 조합을 찾는 도중 이미 **유일성이 증명**되면 해당 조합에서 확장하면 **최소성을 만족하지 못하므로 가지치기**
* 유일성을 만족하는 조합을 **비트마스킹을 통해 최소성을 검증**

## 3. 스트링 처리

### FName

* **문자열 풀링**을 통해 문자열 사용에 있어서 **초경량 시스템**을 제공

* **FName**을 선언하면 전역 데이터 테이블에 **Index**로 변환하여 **한 번만** 저장

* **FName** 변수는 실제 문자열을 갖고 있는 것이 아닌, **변환한 Index 값을 들고 있음**

* **비교 연산**은 `uint32` 값 비교를 통해 연산하므로 매우 빠름

* **FName**은 대소문자를 구분하지 않음

* **FName**은 내부적으로 3개의 멤버 변수를 가짐
    * **`FNameEntryId	ComparisonIndex`**: 전역 이름이 저장된 **Names** 배열에서의 Index를 저장 (소문자로 전환)
    * **`uint32			Number`**: 식별자를 구분하기 위한 번호 저장 (실제 번호보다 1 높게 저장함)
    * **`FNameEntryId	DisplayIndex`**: **원본 문자열**이 저장된 **Names** 배열에서의 Index를 저장 (대소문자 유지)



### FText

* **텍스트 현지화**를 위한 주요 컴포넌트

* 현지화된 **텍스트 리터럴 생성**, **텍스트 포매팅** 등의 기능을 지원

* 현지화하지 않거나, **컬쳐 무관으로 텍스트 생성** 또한 가능

* 내부적으로 **FText** 는 `TRefCountPtr<ITextData> TextData`, `uint32 Flags`로 구현되어 있어 **매우 낮은 비용으로 복사 가능**

* **TextData**에는 실제 텍스트와 관련된 데이터가 저장되어 **수정이 불가능**하며, **Flags**로 FText의 상태를 구분

* 각 언어별 대사를 **스트링 테이블**에 저장하여, 사용자가 설정한 언어에 맞게 대사를 출력


### FString

* FName 이나 FText 와는 달리 **조작이 가능한 유일한 가변 스트링 클래스**

* 기본적으로 **TCHAR 의 TArray**로 구성되어 있으며, 매크로를 통해 **인코딩을 변환**할 수 있음

* 대소문자 변환, 부분 문자열 발췌, 역순, 검색, 변경 등 **다양한 기능을 지원**

* 다양한 기능을 지원하는 만큼 **가장 무거운 복사 비용**을 가짐
