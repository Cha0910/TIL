### [[PCCP 기출문제] 3번 / 충돌위험 찾기](https://school.programmers.co.kr/learn/courses/30/lessons/340211)

```cpp
#include <string>
#include <vector>
#include <map>
using namespace std;

int solution(vector<vector<int>> points, vector<vector<int>> routes) {
    int answer = 0;
    int max = 0;
    vector<map<pair<int,int>, int>> maps(1);
    
    for(int i=0; i<routes.size(); i++){
        int start = routes[i][0]-1;
        int turn = 0;
        pair<int, int> pos = {points[start][0], points[start][1]};
        maps[turn][pos]++;
        if(maps[turn][pos] == 2) answer++;
        
        for(int j=1; j<routes[i].size(); j++){
            int dest = routes[i][j]-1;
            pair<int, int> target = {points[dest][0], points[dest][1]};
            
            while(pos != target){
                turn++;
                if(maps.size() <= turn) maps.push_back(map<pair<int,int>, int>());
                
                if(pos.first != target.first) 
                    pos.first > target.first ? pos.first-- : pos.first++;  
                else if(pos.second != target.second)
                    pos.second > target.second ? pos.second-- : pos.second++;
                
                maps[turn][pos]++;
                if(maps[turn][pos] == 2) answer++;
            }
            start = dest;
        }
    }
    return answer;
}
```

* **시뮬레이션** 문제
* 초마다 로봇의 좌표의 **count**를 `1`씩 증가
* 좌표의 **count**가 `2`라는 것은 해당 시간에, 해당 좌표에서 충돌이 발생했다는 뜻
* **여러 로봇의 충돌은 1회의 충돌로 처리**하므로, 2인 경우에만 충돌 카운트를 올림

### [금과 은 운반하기](https://school.programmers.co.kr/learn/courses/30/lessons/86053)

```cpp
#include <string>
#include <vector>

using namespace std;
bool check(int a, int b, vector<int>& g, vector<int>& s, vector<int>& w, vector<int>& t, long long n){
    long long goldSum = 0, silverSum = 0, sum = 0;
    for(int i=0; i<g.size(); i++){
        if(n < t[i]) continue;
        long long count = ((n - t[i]) / (t[i]*2)) + 1;
        goldSum += min((long long)g[i], w[i]*count);
        silverSum += min((long long)s[i], w[i]*count);
        sum += min((long long)g[i] + s[i], w[i]*count);
    }
    if(goldSum < a || silverSum < b || sum < (long long)a + b) return false;
    else return true;
}

long long solution(int a, int b, vector<int> g, vector<int> s, vector<int> w, vector<int> t) {
    long long answer = -1;
    long long left = 1, right = 1000000000000000;
    
    while(left <= right){
        long long mid = (right - left) / 2 + left;
        if(check(a, b, g, s, w, t, mid)){
            answer = mid;
            right = mid - 1;
        }
        else{
            left = mid + 1;
        }
    }
    return answer;
}
```

* **파라메트릭 서치** 문제
* `n` 시간에 옮길 수 있는 **금의 양**, **은의 양**, **금과 은의 양의 합**이 모두 `a, b, a+b` 보다 크면 조건을 만족

## 3. Delegate (델리게이트)

### 특징

* C++ 오브젝트 상의 멤버 함수를 안전하게 호출하는 언리얼 엔진의 **디자인 패턴**

* **발행-구독 패턴**, **옵저버 패턴**과 유사함

* 임의 오브젝트의 멤버 함수에 **동적으로 바인딩**해 그 오브젝트에서 바인딩한 함수를 호출

* 델리게이트는 내부에 **함수 포인터**를 저장하며, 이벤트 발생 시 저장된 함수를 실행

* 언리얼에서는 **C++의 함수 포인터를 대체**하는 **델리게이트로 안전하게 바인딩**을 지원

* 클래스 간의 의존성을 줄여 **느슨한 결합**을 구현할 수 있음

### 델리게이트 선언

* 헤더에서 **매크로**를 통해 선언할 수 있으며, 아래의 **함수 시그니처**에 대해 지원
    * 값을 반환하는 함수
    * 최대 8개의 파라미터
    * const로 선언된 함수
    * 최대 4개의 페이로드

* 선언 예시
    |함수 시그너처|선언 매크로|
    |---|---|
    |`void Function()`|DECLARE_DELEGATE( DelegateName )|
    |`void Function( <Param1>, <Param2> )`|	DECLARE_MULTICAST_DELEGATE_TwoParams( DelegateName, Param1Type, Param2Type )|
    |`<RetVal> Function( <Param1> )`|DECLARE_DELEGATE_RetVal_OneParam( RetValType, DelegateName, Param1Type )|

### 싱글 캐스트

* 하나의 델리게이트에 하나의 함수만 바인딩하는 **1:1 관계**의 델리게이트

* **반환 값을 갖는 함수**를 바인딩할 수 있음

* 바인딩 된 함수가 없는 경우, 호출 시 **크래시가 발생**할 수 있음

* **바인딩**
    |함수|설명|
    |---|---|
    |Bind()	|기존 델리게이트 오브젝트에 바인딩|
    |BindStatic()|raw C++ 포인터 **글로벌 함수** 델리게이트를 바인딩|
    |BindRaw()|raw C++ 포인터 델리게이트에 바인딩, 호출 시 안전하지 않을 수 있음|
    |BindSP()|SharedPtr 기반 멤버 함수 델리게이트에 바인딩, 오브젝트로의 약한 레퍼런스를 유지 `ExecuteIfBound()`로 호출|
    |BindUObject()	|UObject 기반 멤버 함수 델리게이트를 바인딩, 오브젝트로의 약한 레퍼런스를 유지 `ExecuteIfBound()`로 호출|
    |UnBind()|이 델리게이트 바인딩을 해제|

* **실행**
    |실행 함수|
    |---|
    |Execute()|
    |ExecuteIfBound()|
    |IsBound()| 

### 멀티 캐스트

* 하나의 델리게이트에 여러 개의 함수를 바인딩하는 **1:N 관계**의 델리게이트

* 내부적으로 **TArray**를 통해 함수의 데이터를 관리

* **반환 값을 갖는 함수**를 바인딩할 수 없음

* **바인딩**
    |함수|설명|
    |---|---|
    |Add()	|멀티 델리게이트 오브젝트에 함수를 추가|
    |AddStatic()|raw C++ 포인터 **글로벌 함수** 델리게이트를 추가|
    |Addaw()|raw C++ 포인터 델리게이트에 추가, 호출 시 안전하지 않을 수 있음|
    |AddSP()|SharedPtr 기반 멤버 함수 델리게이트에 추가, 오브젝트로의 약한 레퍼런스를 유지 |
    |AddUObject()	|UObject 기반 멤버 함수 델리게이트를 바인딩, 오브젝트로의 약한 레퍼런스를 유지|
    |Remove()|델리게이트의 실행 목록에서 함수를 삭제|
    |RemaveAll()|델리게이트의 실행 목록에서 모든 함수를 삭제|

* **실행**
    |실행 함수|
    |---|
    |Broadcast()|

## 4. Dynamic Delegate (다이내믹 델리게이트)

### 특징

* **직렬화**와 **리플렉션** 시스템을 지원하는 델리게이트

* 함수의 이름으로 **FName**을 기반으로 동작하며 내부적으로 오브젝트의 **TWeakObjectPtr**과 **함수의 이름**을 저장

* 기본적인 델리게이트보다 **상대적으로 성능이 느림**

* 리플렉션에 등록된 함수만 지원하므로 바인딩 할 함수에 **반드시 UFUNCTION 매크로가 등록되어야 함**

### 다이내믹 델리게이트 선언
|함수 시그너처|선언 매크로|
|---|---|
|`void Function()`|DECLARE_DYNAMIC_DELEGATE( DelegateName )|
|`void Function( <Param1>, <Param2> )`|	DECLARE_DYNAMIC_DELEGATE_TwoParams( DelegateName, Param1Type, Param2Type )|
|`<RetVal> Function( <Param1> )`|DECLARE_DYNAMIC_DELEGATE_RetVal_OneParam( RetValType, DelegateName, Param1Type )|  

### 다이내믹 델리게이트 바인딩 및 실행
* **바인딩** (모든 매크로는 함수 이름 문자열을 자동으로 생성)
    |헬퍼 매크로|설명|
    |---|---|
    |BindDynamic( UserObject, FuncName )|	다이내믹 델리게이트에 함수를 바인딩|
    |AddDynamic( UserObject, FuncName )|다이내믹 멀티-캐스트 델리게이트에 함수를 추가|
    |RemoveDynamic( UserObject, FuncName )|다이내믹 멀티-캐스트 델리게이트에서 함수를 제거|

* **실행**
    |실행 함수|
    |---|
    |Execute()|
    |ExecuteIfBound()|
    |IsBound()| 
    |Broadcast()|
