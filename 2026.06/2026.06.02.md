### [[3차] 방금그곡](https://school.programmers.co.kr/learn/courses/30/lessons/17683)

```cpp
#include <string>
#include <vector>
#include <sstream>
#include <algorithm>
using namespace std;

vector<string> split(const string& str, char c){
    vector<string> result;
    string temp;
    stringstream ss(str);
    while(getline(ss, temp, c)){
        result.push_back(temp);
    }
    return result;
}

int getTime(const string& start, const string& end){
    int s = stoi(start.substr(0, 2))*60 + stoi(start.substr(3, 2));
    int e = stoi(end.substr(0, 2))*60 + stoi(end.substr(3, 2));
    return e - s;
}

string convert(const string& music){
    string result = "";
    int cur = 0;
    while(cur < music.length()){
        if(cur < music.length()-1 && music[cur+1] == '#') {
            result += tolower(music[cur]);
            cur+=2;
        }
        else{
            result += music[cur];
            cur++;
        }
    }
    return result;
}

string solution(string m, vector<string> musicinfos) {
    string answer = "(None)";
    m = convert(m);
    int max = -1;
    
    for(const string& musicinfo : musicinfos){
        vector<string> data = split(musicinfo, ',');
        int time = getTime(data[0], data[1]);
        data[1] = to_string(time);
        data[3] = convert(data[3]);
        
        string temp = "";
        for(int i=0; i<time/data[3].length(); i++)
            temp += data[3];
        temp += data[3].substr(0, time%data[3].length());
        data[3] = temp;
        
        if(data[3].find(m) != string::npos && time > max) {
            answer = data[2];
            max = time;
        }
    }
    
    return answer;
}
```

* **문자열**, **구현** 문제
* **#음** 때문에 잘못 판별되거나, **문자열의 길이와 음의 수**가 같지 않음
* **#음** 을 미리 **소문자로 치환**하면 **문자열의 길이와 음의 수**가 같아져 간단해짐

### [혼자 놀기의 달인](https://school.programmers.co.kr/learn/courses/30/lessons/131130)

```cpp
#include <string>
#include <vector>
using namespace std;

int solution(vector<int> cards) {
    int answer = 0;
    pair<int ,int> max;
    vector<bool> visit(cards.size(), false);
    for(int i=0; i<cards.size(); i++){
        if(visit[i]) continue;
        visit[i] = true;
        int count = 1;
        int next = cards[i]-1;
        while(!visit[next]){
            count++;
            visit[next] = true;
            next = cards[next]-1;
        }
        if(count > max.first) {
            max.second = max.first;
            max.first = count;
        }
        else if(count > max.second) max.second = count;    
    }
    return max.first * max.second;
}
```

* **구현**, **그래프?** 문제 
* 상자가 연결된 형태는 그래프의 형태
* **DFS**처럼 상자에 방문 처리를 하며, **사이클이 발생**하면 하나의 그룹

### [가장 큰 정사각형 찾기](https://school.programmers.co.kr/learn/courses/30/lessons/12905)

```cpp
// 누적 합 + 이분 탐색
#include <iostream>
#include <vector>

using namespace std;

bool check(vector<vector<int>>& pref, int k){
     int s = k-1;
     for(int i=1; i<pref.size()-s; i++){
        for(int j=1; j<pref[0].size()-s; j++){
            int sum = pref[i+s][j+s] - pref[i-1][j+s] - pref[i+s][j-1] + pref[i-1][j-1];
            if(sum == k*k) return true;
        }
    }
    return false;
}
int solution(vector<vector<int>> board)
{
    int answer = 0;
    vector<vector<int>> pref(board.size()+1, vector<int>(board[0].size()+1, 0));
    for(int i=1; i<pref.size(); i++)
        for(int j=1; j<pref[0].size(); j++)
            pref[i][j] += board[i-1][j-1] + pref[i-1][j] + pref[i][j-1] - pref[i-1][j-1];

    int left = 1; int right = min(board.size(), board[0].size());
    while(left<=right){
        int mid = left + (right - left) / 2;
        if(check(pref, mid)){
            answer = mid * mid;
            left = mid + 1;
        }
        else right = mid - 1;
    }
    
    return answer;
}
```

```cpp
// DP
#include <iostream>
#include <vector>

using namespace std;

int solution(vector<vector<int>> board)
{
    int answer = 0;
    for(int i=0; i<board.size(); i++){
        for(int j=0; j<board[i].size(); j++){
            if(board[i][j] == 1){
                int min = 1001*1001;
                if(i>0) min = min < board[i-1][j] ? min : board[i-1][j];
                if(j>0) min = min < board[i][j-1] ? min : board[i][j-1];
                if(i>0 && j>0) min = min < board[i-1][j-1] ? min : board[i-1][j-1];
                if(min != 1001*1001) board[i][j] = min+1;
                if(board[i][j] > answer) answer = board[i][j];
            }
        }
    }
    
    return answer*answer;
}
```

* **누적 합**, **이분 탐색**을 활용해 풀이
* 문제의 범위가 `1 <= n, m <= 1,000`이므로 위 풀이의 $O(n^2logn)$ 으로 풀어도 충분히 통과
* 다른 사람들의 풀이를 보고 **DP**로 풀 수 있다는걸 알았음
* 아래의 코드는 **DP**를 활용해 푼 코드

## 3. UObject

### UObject

* 언리얼 엔진의 **오브젝트에 대한 최상위 베이스 클래스**

* **Transform** 데이터를 갖고 있지 않아 월드에 배치가 불가능

* **UObject** 타입을 상속받아 언리얼 엔진의 핵심 기능인 **가비지 컬렉션**과 **리플랙션** 등을 사용할 수 있음

### UCLASS 매크로

* **UObject**에 **UCLASS()** 매크로를 추가하면 런타임 중 해당 오브젝트가 어떤 변수, 함수를 멤버로 가지고 있는지 저장하는 **클래스 디폴트 오브젝트(Class Default Object, CDO)** 를 생성함

* **CDO**는 기본적으로 클래스 생성자에 의해 생성되고 이후에는 수정되지 않는 **디폴트 템플릿 오브젝트**

### 오브젝트 생성

* 엔진 실행 시 모든 **UObject**는 한 번만 **CDO**를 생성하여 메모리에 저장함

* 따라서 **UObject**의 생성자는 파라미터를 받을 수 없으며, 파라미터가 없는 **디폴트 생성자를 필수로 갖고 있어야 함**

* **CDO**는 실제 생성 여부와 관계없이 실행되기에 무거운 로직은 `BeginPlay()`에서 실행되어야 함

* **UObject**는 언리얼 엔진에서 가비지 컬렉션에서 자동으로 관리되기에, `new` 연산자를 사용해서는 안 됨

* UObject를 생성하는 방법은 `NewObject<class>`, `CreateDefaultSubobject<class>`의 두 가지 방법이 존재
    * `NewObject<class>`: 런타임 도중 **파라미터**를 사용해 유연하게 새로운 인스턴스를 생성

    * `CreateDefaultSubobject<class>`: **생성자**에서만 호출할 수 있으며 **CDO**에 오브젝트 정보를 저장, 주로 **컴포넌트**를 생성하는데 사용

### 오브젝트 파괴

* 오브젝트가 더 이상 참조되지 않으면, **가비지 컬렉터가 자동으로 오브젝트를 파괴**

* `TWeakObjectPtr`은 가비지 컬렉터에 **영향을 주지 않음**

* `Obj->MarkAsGarbage()` 함수는 더 이상 사용하지 않는 포인터에 **가비지 플래그**를 꽂으며, **nullptr로 바꾸지는 않음**

* `IsValid()` 함수는 nullptr과 가비지 플래그를 둘 다 확인하므로 `IsValid()`를 사용해야 함

* **액터**의 경우 `Destroy()` 함수로 레벨에서 제거되었어도, **참조가 남아있으면** 가비지 컬렉터의 대상이 되지 않음

## 4. UObject의 기능

### 자동 프로퍼티 초기화 

* **UObject**는 **생성자를 호출하기 전** 자신의 모든 멤버 변수를 **nullptr**, **0** 등으로 초기화 함

* 이후 **생성자**나 **BeginPlay()** 함수에서 값을 지정할 수 있음

### 프로퍼티 값 업데이트

* 언리얼 엔진은 레벨 파일의 액터 데이터를 **CDO와 차이가 존재하는 부분만 저장**

* **CDO** 값이 변경되어도 따로 액터의 데이터에 저장된 데이터가 없으므로, 변경된 **CDO**의 값을 그대로 가져옴

* 만약 **액터의 데이터를 수정**했다면, CDO 값이 변경되어도 **수정한 데이터가 남아있으므로** 수정한 데이터의 값을 가져옴

### 레퍼런스 자동 업데이트

* **AActor** 또는 **UActorComponent** 가 소멸, 삭제되면 리플랙션 시스템에 보이고 있는 모든 참조는 **자동으로 nullptr로 변경** 됨

* 리플랙션 시스템에 보이고 있는 모든 참조만 적용되기에 `UPROPERTY()`가 붙어야만 추적 가능

* `TWeakObjectPtr<class>`는 대상 액터가 삭제되면 **자동으로 nullptr로 변경**됨

### 직렬화 (Serialization)

* `UPROPERTY()`가 붙은 멤버 변수는 자동으로 **세이브/로드** 기능을 지원

* `UPROPERTY(Transient)` 처럼 지정자를 통해 **직렬화 대상에서 제외**할 수 있음

* 프로퍼티 값 업데이트와 동일하게 **CDO의 값과 동일한 경우 따로 저장하지 않음**

* 기존의 변수가 추가, 삭제되어도 엔진은 자동으로 CDO 값을 복사하거나 무시함

* 직렬화에 **커스텀**이 필요할 경우 `UObject::Serialize(FArchive& Ar)` 함수를 **오버라이딩** 할 수 있음

### 에디터 통합

* **UObject** 와 **UProperty** 는 에디터에 인식되며 자동으로 노출시킬 수 있음

* **블루프린트**를 통한 비주얼 스크립팅이 가능함

* **지정자**를 통해서 에디터가 C++의 데이터에 접근할 수 있는 정도를 조절할 수 있음

### 런타임 유형 정보 및 형변환

* **UObject**는 자신이 어떤 클래스인지 저장되어 있는 **UClass*의 주소를 갖고 있음**

* 해당 주소를 통해 **포인터 비교만으로 RTTI (Run Time Type Information)** 가 가능해짐

### 가비지 컬렉션

* 가비지 컬렉션은 **Mark and Sweep** 알고리즘으로 메모리를 관리

* 엔진은 **레퍼런스 그래프**로 오브젝트의 참조 여부를 판단하며, 그래프의 루트는 **루트 세트**로 지정된 오브젝트가 존재

* 가비지 컬렉터가 제거하지 않게 하기 위해서는 `UPROPERTY()`를 붙이거나, `UPROPERTY()`가 붙은 컨테이너에 저장해야 함

* 종종 **액터**, **컴포넌트**는 레벨과 같이 **루트 세트로 다시 링크되는 오브젝트가 참조**하기에 제거되지 않음

* `Destroy()`, `DestroyComponent()`를 통해 명시적으로 제거하며, 컴포넌트는 보통 액터가 삭제되면 같이 삭제됨

### 네트워크 리플리케이션

* 언리얼 엔진은 서버가 클라이언트에게 **복사본**을 주어 동기화하는 **리플리케이션 (Replication)** 을 통해 네트워크를 지원

* `UPROPERTY(Replicated)`와 같이 **Replicated** 지정자로 변수를 복제할 수 있음

* `UPROPERTY(ReplicatedUsing = OnRep_)`과 같이 **콜백 함수**를 바인딩할 수 있음

* `UFUNCTION(Server, Reliable)`, `UFUNCTION(Client, Unreliable)` 와 같이 **RPC (Remote Procdure Call)** 또한 호출 가능
