### [점프와 순간 이동](https://school.programmers.co.kr/learn/courses/30/lessons/12980)

```cpp
#include <iostream>
using namespace std;

int solution(int n)
{
    int ans = 0;
    
    while(n>0){
        ans += n % 2;
        n /= 2;
    }

    return ans;
}
```

* **수학** 문제
* **홀수**인 경우에만 1 이동해 주면 됨

### [짝지어 제거하기](https://school.programmers.co.kr/learn/courses/30/lessons/12973)

```cpp
#include <string>

using namespace std;

int solution(string s)
{
    char* start = s.data();
    char* cur = start - 1;
    
    for (char& c : s){
        if (cur >= start && *cur == c) --cur;
        else *++cur = c;
    }
    return cur < start;
}
```

* **스택**을 활용하는 문제
* 코드가 직관적이지는 않은데, 문제의 효율성이 굉장히 타이트하게 잡혀있어 최적화한 것
* 아래에 문제 풀이 과정과 시행착오를 정리해 보려고 함

#### 1. 스택 활용

```cpp
#include <string>
#include <stack>

using namespace std;

int solution(string s)
{
    if(s.length() % 2 == 1) return 0;
    
    stack<char> st;
    for(int i=0; i<s.length(); ++i){
        if(!st.empty() && st.top() == s[i]) st.pop();
        else st.push(s[i]);
    }
    
    return st.empty();
}
```

* 문제는 **괄호 유효성 검사** 문제와 사실상 동일함
* 문자열을 검사를 완료했을 때, **스택이 비어 있는지 여부**로 수행 가능 여부를 판단
* 하지만 이 코드는 효율성 검사에서 시간초과가 발생함
* 위 코드는 이미 `O(n)`의 시간복잡도를 가지며, 문자열의 모든 문자를 검사해야 하기에 이보다 **낮은 시간 복잡도로 풀 수는 없음**

#### 2. 조기 종료

```cpp
#include <string>
#include <stack>
#include <vector>

using namespace std;

int solution(string s)
{
    if(s.length() % 2 == 1) return 0;

    vector<int> count(26, 0);
    for(int i=0; i<s.length(); i++) ++count[s[i] - 'a'];
    for(int i=0; i<count.size(); i++) if(count[i] % 2 != 0) return 0;

    stack<char> st;
    for(int i=0; i<s.length(); ++i){
        if(!st.empty() && st.top() == s[i]) st.pop();
        else st.push(s[i]);
    }
    
    return st.empty();
}
```

* 문자열의 유효성을 검사하기 전, **알파벳 개수를 확인해 조기에 종료**
* 기존의 시간초과가 발생하던 테스트 케이스 중 일부는 통과되었지만, 다른 코드는 **오히려 실행 시간이 늘어남**
* 만약 알파벳 개수가 모두 짝수라면, 조기 종료가 무의미해지고 오히려 **문자열을 두 번 순회하기에 더 비효율적**

#### 3. 캐시 히트율

```cpp
#include <string>

using namespace std;

int solution(string s)
{
    if (s.length() % 2 == 1) return 0;
    
    string st = "";
    st.reserve(s.length()); 
    
    for (int i=0; i<s.size(); i++) {
        if (!st.empty() && st.back() == s[i]) st.pop_back();
        else st.push_back(s[i]);  
    }
    
    return st.empty();
}
```

* `std::stack`은 내부적으로 `std::deque`로 구현되어 있기에, 캐시 히트율이 낮은편임
* 캐시 히트율을 높이기 위해 메모리에 연속된 위치를 갖는 `string`을 사용해 보았지만 결과는 동일하거나 오히려 약간 느려짐

#### 4. 스택 제거

```cpp
#include <string>

using namespace std;

int solution(string s)
{
    if(s.length() % 2 == 1) return 0;
    
    int cur = -1;
    for(int i=0; i<s.length(); ++i){
        if(cur != -1 && s[cur] == s[i]) --cur;
        else s[++cur] = s[i];
    }
    
    return cur < 0;
}
```

* 스택을 따로 만들지 않고, 기존에 주어진 문자열 `s` 를 스택으로 활용
* `cur`가 스택의 top을 가리키고 있으며, 이미 방문한 s[i]는 다시 방문하지 않으므로 해당 부분을 스택으로 활용함
* 이는 유의미하게 실행 시간이 감소했지만, 여전히 시간초과가 발생

#### 5. 향상된 반복문

```cpp
#include <string>

using namespace std;

int solution(string s)
{
    int cur = -1;
    for(const char& c : s){
        if(cur != -1 && s[cur] == c) --cur;
        else s[++cur] = c;
    }
    
    return cur < 0;
}
```

* 반복문을 **향상된 반복문**으로 변경하니, 실행 시간이 약간 감소함
* 해당 결과를 보고 나니 문자열에 **인덱스로 접근**할 때, **더하기 연산**이 발생하고 이를 줄일 수 있지 않을까? 생각하게 됨
* 인덱스 접근 자체를 없애는 방법을 고민

## 3. LeetCode 문제 풀이

### [Maximum Average Subarray I](https://leetcode.com/problems/maximum-average-subarray-i/description/)

```cpp
class Solution {
public:
    double findMaxAverage(vector<int>& nums, int k) {
        
        double sum = 0;
        for(int i=0; i<right; i++) sum += nums[i];
        double max = sum;
        
        int left = 0, right = k; 
        for(int i=right; i<nums.size(); i++){
            sum += nums[i];
            sum -= nums[left++];
            max = sum > max ? sum : max;
        }
        return max / k;
    }
};
```

* **슬라이딩 윈도우** 유형의 대표적인 문제
* 윈도우를 오른쪽으로 **한 칸씩 밀면서** 최댓값을 저장

### [Longest Substring Without Repeating Characters](https://leetcode.com/problems/longest-substring-without-repeating-characters/description/)

```cpp
#include <unordered_map>

class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        unordered_map<char, int> index;
        int left = 0, max = 0;

        for(int i=0; i<s.length(); i++){
            if(index.count(s[i]) && index[s[i]] >= left) left = index[s[i]] + 1;
            index[s[i]] = i; 
            max = max > i - left + 1 ? max : i - left + 1;
        }

        return max;
    }
};
```

* **슬라이딩 윈도우** 유형의 문제
* `index[s[i]]`는 현재 윈도우에서 `s[i]`가 몇 번째 인덱스에 있는지를 저장함
* 만약 s[i]의 위치가 현재 윈도우 내부일 경우, `left`를 `index[s[i]] + 1`로 옮겨 중복을 지움

## 4. 좀비 프로세스와 고아 프로세스

### 프로세스 생성과 종료

* **UNIX/Linux** 환경에서 프로세스는 **부모-자식 관계**를 가짐

* 커널이 실행되고 모든 프로세스의 루트가 되는 **init 프로세스**가 생성됨

* **부모 프로세스**는 `fork()` 시스템콜을 통해 자신의 복제본인 **자식 프로세스**를 생성한 뒤, `exec()` 시스템콜을 호출해 새로운 프로세스로 대체

* 이후 부모 프로세스는 `wait()` 시스템콜을 호출해 자식 프로세스의 `exit()` 시스템콜을 대기

* 자식 프로세스가 `exit()` 시스템콜을 호출하면 **자식 프로세스의 대부분의 리소스를 반환하고 종료**

* 이후 부모 프로세스는 자식 프로세스의 **종료 신호**를 받아 **남은 기록을 지우고 완전히 종료**시킴

### 좀비 프로세스

* 자식 프로세스가 부모 프로세스보다 먼저 종료되었을 때 발생

* 자식 프로세스는 모든 로직을 완료하고 `exit()` 시스템콜을 호출했지만, 부모가 `wait()` **시스템콜을 호출하지 않았을 때 발생**

* 이로 인해 자식 프로세스는 **모든 자원을 정상적으로 반환하지 못하고**, 일부 자원을 가진 채로 **계속 살아있음**

* 좀비 프로세스를 해결하는 방법은 3가지가 존재
    * 부모가 늦게라도 `wait()` 시스템콜을 호출해 자식을 수거
    * 부모 프로세스를 종료, 이는 다른 자식 프로세스까지 모두 종료시켜 **사이드 이펙트**가 발생함
    * 좀비가 된 자식 프로세스를 강제로 종료

### 고아 프로세스

* 부모 프로세스가 자식 프로세스보다 먼저 종료되었을 때 발생

* 이 경우 **init 프로세스**가 **고아 프로세스의 부모**가 되어 **wait()** 시스템콜을 호출함
