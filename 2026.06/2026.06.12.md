## 2. 프로그래머스 문제 풀이

### [홀짝트리](https://school.programmers.co.kr/learn/courses/30/lessons/388354)

```cpp
#include <string>
#include <vector>
#include <unordered_map>
#include <queue>

using namespace std;

int BFS(unordered_map<int, vector<int>>& graph, unordered_map<int, int>& visit, 
       unordered_map<int, int>& node_data, int start){
    if(visit[start]) return -1;
    vector<int> count(2, 0);
    queue<int> q;
    q.push(start);
    visit[start] = true;
    count[node_data[start]]++;
    
    while(!q.empty()){
        int curr = q.front();
        q.pop();
        
        for(const int next : graph[curr]){
            if(visit[next]) continue;
            q.push(next);
            visit[next] = true;
            count[node_data[next]]++;
        }
    }
    
    if(count[0] == 1 && count[1] == 1) return 2;
    else if(count[0] == 1) return 0;
    else if(count[1] == 1) return 1;
    else return -1;
}

vector<int> solution(vector<int> nodes, vector<vector<int>> edges) {
    vector<int> answer(2, 0);
    unordered_map<int, vector<int>> graph;
    unordered_map<int, int> node_data; // 0: 홀짝, 1:역홀짝
    unordered_map<int, int> visit;
    for(const vector<int>& edge : edges){
        graph[edge[0]].push_back(edge[1]);
        graph[edge[1]].push_back(edge[0]);
    }
    
    for(const int node : nodes){
        int count = graph[node].size();
        visit[node] = false;
        if(node%2 == count%2) node_data[node] = 0;
        else node_data[node] = 1;
    }
    
    for(const int node : nodes){
        int result = BFS(graph, visit, node_data, node);
        if(result == 0 || result == 1) answer[result]++;
        else if(result == 2){
            answer[0]++;
            answer[1]++;
        }
    }
    return answer;
}
```

* **그래프**, **연결요소**, **BFS** 유형의 문제
* 어떤 노드에 연결된 간선이 `n`개라고 하면, 해당 **노드가 자식으로 결정될 경우 그 노드의 자식 수는 `n-1`개가 됨**
* 즉, 하나의 연결요소에서 **단 하나의 노드만 홀짝 노드인 경우 해당 트리는 홀짝 트리**이고 **반대의 경우 역홀짝 트리**임
* 노드가 2개고 2개의 노드가 각각 홀짝 노드, 역홀짝 노드 일경우 **홀짝 트리, 역홀짝 트리가 둘 다 가능한 경우**

## 4. 언리얼 엔진의 스마트 포인터

### 언리얼 엔진의 스마트 포인터

* 언리얼 엔진은 스마트 포인터를 **C++11**의 스마트 포인터를 커스텀하여 구현

* C++의 스마트 포인터와 유사하게 **Shared Pointers**, **Weak Pointers**, **Unique Pointers**를 지원하며 추가로 **Shared References**까지 총 **네 가지의 스마트 포인터**를 지원

* **UObject**는 GC가 자체적으로 관리하기에 **스마트 포인터를 사용할 수 없음**

* 기본적으로 언리얼 엔진의 스마트 포인터는 **스레드 세이프하지 않으며** 스마트 포인터 생성 시 **스레드 세이프 버전**을 사용할 수 있음

### TSharedPtr

* C++의 `std::shared_ptr`에 대응하는 스마트 포인터

* **컨트롤 블럭**의 참조 횟수를 기반으로 동작하며, 참조 횟수가 0이 될 경우 자동으로 해제됨

* `std::shared_ptr`과 동일하게 순환 참조 문제가 발생할 수 있음

* 예시 코드
    ```cpp
	TSharedPtr<FMyObjectType> MyObject = MakeShared<FMyObjectType>();

    {
        TSharedPtr<FMyObjectType> OhterObject = MyObejct; // 참조 횟수 2

    } // 스코프가 종료되며 참조 횟수 1
    
    ```

### TSharedRef

* `TSharedPtr`과 동일하게 동작하지만, **null을 허용하지 않는** Shared Pointer

* `TSharedRef`은 `TSharedPtr`로 변환될 수 있음

* **null**을 허용하는 오브젝트가 필요하지 않는 한 `TSharedRef`를 **권장**

* 예시 코드
    ```cpp
    //TSharedRef<FMyObjectType> MyObject = nullptr; // TSharedRef는 nullptr을 가질 수 없음 ,컴파일 에러
    TSharedRef<FMyObjectType> MyObject = MakeShared<FMyObjectType>(); // 반드시 초기화

    {
        // TSharedRef -> TSharedPtr 는 단순 대입 연산자로 가능
        TSharedPtr<FMyObjectType> OtherObject = MyObject; // 참조 횟수 2

    } // 스코프가 종료되며 참조 횟수 1

    // MyObject는 nullptr이 아님이 보장되므로, 유효성 검사하지 않고 사용해도 됨
    MyObject->DoSomething();
    ```

### TWeakPtr

* C++의 `std::weak_ptr`에 대응하는 스마트 포인터

* 오브젝트에 대해 약한 참조하기에 **참조 횟수를 증가시키지 않음**

* `TSharedPtr`에서 발생하는 순환 참조 문제를 해결하는 데 사용할 수 있음

* `Pin()`함수로 `TWeakPtr`을 `TSharedPtr`로 변환할 수 있으며, 이는 `std::weak_ptr`의 `std::lock()`와 동일하게 동작

* 예시 코드
    ```cpp
    // TSharedPtr 생성, 참조 횟수 1
    TSharedPtr<FMyObjectType> MyObject = MakeShared<FMyObjectType>();

    // TWeakPtr 생성, 참조 횟수는 증가되지 않고 1로 유지
    TWeakPtr<FMyObjectType> WeakObject = MyObject;

    {
        // WeakData->DoSomething(); // 컴파일 에러, TWeakPtr은 직접 접근할 수 없음
        
        // Pin()을 통해 임시 TSharedPtr로 변환해야 함 (std::lock()와 동일)
        // TSharedPtr로 변환되며 참조 횟수 2로 증가
        if (TSharedPtr<FMyObjectType> LockObject = WeakObject.Pin())
        {
            LockObject->DoSomething();

        } // 스코프 종료로 LockObject 파괴, 참조 횟수 1
    } 

    MyObject = nullptr;

    // 원본이 nullptr인 경우 변환하지 않음
    if (TSharedPtr<FMyObjectType> LockObject = WeakObject.Pin())
    {
        LockObject->DoSomething();
    } 
    ```

### TUniquePtr

* C++의 `std::unique_ptr`에 대응하는 스마트 포인터

* 오브젝트의 유일한 **소유권**을 가지고 있어 **오브젝트가 파괴될 때 같이 해제** 됨

* 유일한 소유권을 갖고 있기에 **복사 시 컴파일 에러**가 발생

* `MoveTemp()`로 소유권을 이전할 수 있으며, 이는 `std::unique_ptr`의 `std::move()`와 동일하게 동작

* 예시 코드
    ```cpp
    TUniquePtr<FMyObjectType> MyObject = MakeUnique<FMyObjectType>();
    TUniquePtr<FMyObjectType> OtherObject;

    // OtherObject = MyObject; // 컴파일 에러
        
    OtherObject = MoveTemp(MyObject); // 소유권 이전
    ```
