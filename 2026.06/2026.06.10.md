## 2. 프로그래머스 문제 풀이

### [리코쳇 로봇](https://school.programmers.co.kr/learn/courses/30/lessons/169199)

```cpp
#include <string>
#include <vector>
#include <queue>
using namespace std;
pair<int, int> getNext(const vector<string>& board, int y, int x, int dir){
    int dy[4] = {-1, 1, 0, 0};
    int dx[4] = {0, 0, -1, 1};
    
    while(true){
        int ny = y + dy[dir];
        int nx = x + dx[dir];
        if(ny<0 || ny>=board.size() || nx<0 || nx>=board[0].size() || board[ny][nx]=='D')
            return {y, x};
        y = ny;
        x = nx;
    }
    return {-1, -1};
}

int BFS(const vector<string>& board, vector<vector<int>>& visit, int y, int x){  
    queue<pair<int,int>> q;
    q.push({y, x});
    visit[y][x] = 0;
    
    while(!q.empty()){
        pair<int,int> pos = q.front();
        q.pop();
        int cy = pos.first, cx = pos.second;
        if(board[cy][cx] == 'G') return visit[cy][cx];
        
        for(int i=0; i<4; i++){
            pair<int,int> next = getNext(board, cy, cx, i);
            int ny = next.first, nx = next.second;
            if(ny == -1 || nx == -1 || visit[ny][nx] != -1) continue;
            q.push({ny, nx});
            visit[ny][nx] = visit[cy][cx] + 1;
        }
    }
    
    return -1;
}

int solution(vector<string> board) {
    vector<vector<int>> visit(board.size(), vector<int>(board[0].length(), -1));
    for(int i=0; i<board.size(); i++)
        for(int j=0; j<board[i].size(); j++)
            if(board[i][j] == 'R')
                return BFS(board, visit, i, j);
}
```

* **BFS** 문제
* **이동 시 벽을 만날 때까지 이동**한다는 것만 제외하면 기본적인 BFS와 동일함

## 3. Replication

### Replication이란

* 언리얼 엔진에서 멀티 플레이 게임에서 **게임 상태를 동기화**할 때 사용하는 시스템

* 데이터 복제를 통해 **서버와 클라이언트가 동일한 게임 상태를 유지**할 수 있도록 도움

* 언리얼 엔진에서 사용하는 리플리케이션은 크게 **프로퍼티 리플리케이션,** **컴포넌트 리플리케이션**, **RPC**의 세 가지 종류로 나눌 수 있음

### NetMode

* 게임 프로세스가 **네트워크상에서 어떤 역할**을 하는지 의미

|네트워크 모드| 설명|
|---|---|
|**Standalone** |게임이 **원격 클라이언트 접속을 허용하지 않는 서버**로 실행, 싱글 혹은 로컬 멀티 플레이만 허용하는 게임에서 사용|
|**Client**|게임이 네트워크 멀티플레이 세션에서 **서버에 연결된 클라이언트**로 실행, 어떠한 서버 로직도 실행하지 않음|
|**Listen Server**|게임이 네트워크 멀티플레이어 세션을 **호스팅하는 서버**로 실행, 서버에 로컬 플레이어가 공존|
|**Dedicated Server**|게임이 네트워크 멀티플레이어 세션을 **호스팅하는 서버**로 실행, 서버에 로컬 플레이어가 존재하지 않음|

### NetRole

* **Replication**은 Listen Server와 Dedicated Server에서 사용되며 **중요한 로직은 서버에서 실행되어야 함**

* **NetMode**는 월드가 소유하고 있기에, **각 액터의 권한을 관리**하기 위해 **역할**이 필요

* **액터가 어떤 권한을 갖는지 구분**하기 위해 `Role`, `Remote Role`의 두 가지 프로퍼티를 소유하고 있음

* `Role`은 현재 프로세스에서의 역할을 의미하며, `Remote Role`은 원격 프로세스에서의 역할을 의미

|네트워크 역할|설명|
|---|---|
|**None** |액터가 네트워크 게임에서 역할이 없으며 **리플리케이트하지 않음**|
|**Authority**|액터가 **권한을 갖고 있으며** 다른 컴퓨터에 있는 액터의 원격 프록시로 **정보를 리플리케이트 함**|
|**Simulated Proxy**|Authority를 갖고 있는 액터에 의해 **완벽히 제어되는 프록시**|
|**Autonomous Proxy**|액터가 **일부 로직을 직접 수행**하며, Authority를 갖고 있는 액터에게 **보정을 수신**|

### 프로퍼티 리플리케이션

* 액터의 생성자에서 `bReplicates`를 true로 설정하면 리플리케이션을 지원

* `UPROPERTY` 매크로에서 **Replicated** 또는 **ReplicateUsing** 지정자를 사용하거나 블루프린트의 **Replicated**를 활성화해 **변수 및 오브젝트**를 리플리케이션 할 수 있음

* **프로퍼티 리플리케이션**을 활성화하면 서버에서 값이 변경 시 클라이언트에게 자동으로 리플리케이션 함

* 클라이언트에서 **리플리케이션된 값이 갱신**되었을 때, **RepNotify**를 통해 **클라이언트가 실행할 함수를 지정**할 수 있음

```cpp
class ENGINE_API AActor : public UObject
{
	UPROPERTY(replicated)
	AActor* Owner;
};
```

```cpp
AActor::AActor(const class FPostConstructInitializeProperties & PCIP ) : Super( PCIP )
{
	bReplicates = true;
}

void AActor::GetLifetimeReplicatedProps(TArray< FLifetimeProperty > & OutLifetimeProps ) const
{
	DOREPLIFETIME(AActor, Owner);
}
```

### 컴포넌트 리플리케이션

* 액터 뿐만 아니라 **액터가 소유한 컴포넌트로 리플리케이션**을 지원

* 컴포넌트의 생성자에서 `SetIsReplicated(true)`를 통해 리플리케이션을 활성화
          
### 함수 리플리케이션 (RPC)

* 네트워크 세션에 접속된 **특정 컴퓨터에서 함수를 호출**하여 실행

* `UFUNCTION` 매크로에 `Server`, `Client`, `NetMulticast` 지정자를 사용하여 RPC를 선언할 수 있음

    |RPC 타입|설명|
    |---|---|
    |**Server**|게임을 호스팅하는 서버에서만 호출|
    |**Client**|함수가 속한 액터를 소유한 클라이언트에서만 호출|
    |**NetMulticast**|서버에 접속된 모든 클라이언트와 서버 자체에서 호출|

* 액터는 **오너십(Ownership)** 을 가지며, 클라이언트가 **Server RPC**를 호출할 경우 **해당 액터의 Owner가 아닌 클라이언트면 무시**

* RPC는 매크로에 `Reliable` 또는 `Unreliable` 지정자로 **신뢰성**을 설정할 수 있으며, 기본적으로 `Unreliable`로 설정
    * `Reliable`: 원격 프로세스에서 **순서와 실행을 보장**
    * `Unreliable`: 원격 프로세스에서 **전달을 보장하지 않아 일부 유실이 발생할 수 있음**

* `WithValidation` 지정자는 RPC 함수를 `_Implementation()`, `_Validate()` 두 개로 나누어 **유효성 검사**
    * `_Implementation()`: 함수의 실제 구현부
    * `_Validate()`: 함수 요청이 정상적인지 서버에서 검사하는 함수

```cpp
UFUNCTION(Server, Reliable)
void ServerRPC();

UFUNCTION(Client, Unreliable)
void ClientRPC();

UFUNCTION(NetMulticast)
void NetMulticastRPC();
```
