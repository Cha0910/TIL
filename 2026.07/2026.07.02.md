### [[1차] 캐시](https://school.programmers.co.kr/learn/courses/30/lessons/17680)

```cpp
#include <string>
#include <vector>
#include <algorithm>
#include <list>

using namespace std;

void toLower(string& str){
    for(char& c : str)
        c = tolower(c);
}

int solution(int cacheSize, vector<string> cities) {
    int answer = 0;
    if(cacheSize == 0) return cities.size() * 5;
    list<string> cache;
    
    for(string& city : cities){
        toLower(city);
        
        if(find(cache.begin(), cache.end(), city) == cache.end()){
            if(cache.size() >= cacheSize) cache.pop_front(); 
            cache.push_back(city);
            answer += 5;
        }
        
        else{
            cache.remove(city);
            cache.push_back(city);
            answer++;
        }
    }
    
    return answer;
}
```

* **시뮬레이션** 문제
* **LRU 알고리즘**을 구현하는 문제
* 캐시 히트에 따라서 제거 우선순위가 변경되므로 `list`를 사용

### [[1차] 뉴스 클러스터링](https://school.programmers.co.kr/learn/courses/30/lessons/17677)

```cpp
#include <string>
#include <unordered_map>
#include <unordered_set>

using namespace std;

int solution(string str1, string str2) {
    int answer = 0;
    unordered_map<string, int> subset1, subset2;
    int unionSize = 0, productSize = 0;
    unordered_set<string> subwords;
    
    for(int i=1; i<str1.length(); i++){
        char c1 = tolower(str1[i-1]), c2 = tolower(str1[i]);
        if(!isalpha(c1) || !isalpha(c2)) continue;
        string sub = "";
        sub += c1; sub += c2;
        subset1[sub]++;
        subwords.insert(sub);
    }
    
    for(int i=1; i<str2.length(); i++){
        char c1 = tolower(str2[i-1]), c2 = tolower(str2[i]);
        if(!isalpha(c1) || !isalpha(c2)) continue;
        string sub = "";
        sub += c1; sub += c2;
        subset2[sub]++;
        subwords.insert(sub);
    }
    
    for(const string& subword : subwords){
        int subset1Num = subset1[subword], subset2Num = subset2[subword];
        unionSize += max(subset1Num, subset2Num);
        productSize += min(subset1Num, subset2Num);
    }
    
    answer = unionSize != 0 ? (productSize * 65536) / unionSize : 65536;
    return answer;
}
```

* **구현** 문제
* 문자열을 **파싱**해 각 문자열의 집합을 구함
* 집합의 원소의 개수를 **해시맵**에 저장해 교집합, 합집합의 크기를 계산

### 1094번: 막대기

#### 문제 설명
지민이는 길이가 64cm인 막대를 가지고 있다. 어느 날, 그는 길이가 Xcm인 막대가 가지고 싶어졌다. 지민이는 원래 가지고 있던 막대를 더 작은 막대로 자른다음에, 풀로 붙여서 길이가 Xcm인 막대를 만들려고 한다.
막대를 자르는 가장 쉬운 방법은 절반으로 자르는 것이다. 지민이는 아래와 같은 과정을 거쳐서 막대를 자르려고 한다.
지민이가 가지고 있는 막대의 길이를 모두 더한다. 처음에는 64cm 막대 하나만 가지고 있다. 이때, 합이 X보다 크다면, 아래와 같은 과정을 반복한다.
가지고 있는 막대 중 길이가 가장 짧은 것을 절반으로 자른다.
만약, 위에서 자른 막대의 절반 중 하나를 버리고 남아있는 막대의 길이의 합이 X보다 크거나 같다면, 위에서 자른 막대의 절반 중 하나를 버린다.
이제, 남아있는 모든 막대를 풀로 붙여서 Xcm를 만든다.
X가 주어졌을 때, 위의 과정을 거친다면, 몇 개의 막대를 풀로 붙여서 Xcm를 만들 수 있는지 구하는 프로그램을 작성하시오.

### 1987번: 알파벳

#### 문제 설명
세로 R칸, 가로 C칸으로 된 표 모양의 보드가 있다. 보드의 각 칸에는 대문자 알파벳이 하나씩 적혀 있고, 좌측 상단 칸 (1행 1열) 에는 말이 놓여 있다.
말은 상하좌우로 인접한 네 칸 중의 한 칸으로 이동할 수 있는데, 새로 이동한 칸에 적혀 있는 알파벳은 지금까지 지나온 모든 칸에 적혀 있는 알파벳과는 달라야 한다. 즉, 같은 알파벳이 적힌 칸을 두 번 지날 수 없다.
좌측 상단에서 시작해서, 말이 최대한 몇 칸을 지날 수 있는지를 구하는 프로그램을 작성하시오. 말이 지나는 칸은 좌측 상단의 칸도 포함된다.

#### 입력
첫째 줄에 R과 C가 빈칸을 사이에 두고 주어진다. (1 ≤ R,C ≤ 20) 둘째 줄부터 R개의 줄에 걸쳐서 보드에 적혀 있는 C개의 대문자 알파벳들이 빈칸 없이 주어진다.

#### 출력
첫째 줄에 말이 지날 수 있는 최대의 칸 수를 출력한다.

#### 예제 입력 1
```
2 4
CAAB
ADCB
```
#### 예제 출력 1
```
3
```

#### 예제 입력 2
```
3 6
HFDFFB
AJHGDH
DGAGEH
```
#### 예제 출력 2
```

```

#### 예제 입력 3

#### 예제 출력 3

#### 정답 코드

```cpp
#include <iostream>
#include <bitset>
#include <vector>

using namespace std;

int dy[4] = {-1, 1, 0, 0};
int dx[4] = {0, 0, -1, 1};

void DFS(const vector<string>& board, bitset<26>& state, int y, int x, int& answer){
    answer = answer > state.count() ? answer : state.count();

    for(int i=0; i<4; i++){
        int ny = y + dy[i], nx = x + dx[i];

        if(ny<0 || ny>=board.size() || nx<0 || nx>=board[0].size()) continue;
        int index = board[ny][nx] - 'A';
        if(state[index] == 1) continue;

        state[index] = 1;
        DFS(board, state, ny, nx, answer);
        state[index] = 0;
    }
}

int main() {
    int R, C, answer = 1;
    cin >> R >> C;
    vector<string> board(R);
    bitset<26> state;
    
    for(int i=0; i<R; i++)
        cin >> board[i];
    state[board[0][0] - 'A'] = 1;
    DFS(board, state, 0, 0, answer);
    cout << answer;
    return 0;
} 
```

* **DFS**, **비트마스킹** 문제
* 방문 여부를 `state`로 검증

## 4. 동기화 방법

### 동기화

* 여러 프로세스, 스레드가 **공유 자원에 동시에 접근**하여 실행 순서에 따라서 결과가 달라지는 상황을 **레이스 컨디션**이라 하며, 이를 해결하기 위해 **동기화**를 함

* 대표적인 동기화 방법은 **크리티컬 섹션**, **세마포어**, **뮤텍스**가 있음

### 크리티컬 섹션 (Critical Section)

* 개념적으로는 두 개 이상의 프로세스, 스레드가 **동시에 접근해서는 안 되는 코드영역**을 의미

* 윈도우 API에는 `CRITICAL_SECTION`이라는 객체로 동기화를 지원

* `CRITICAL_SECTION`은 **유저 모드**에서 동작해 세마포어, 뮤텍스보다 **빠른 속도**를 가짐

* **뮤텍스**와 유사하게 `EnterCriticalSection`으로 lock을 획득하고, `LeaveCriticalSection`으로 lock을 해제하는 구조

* 유저 모드에서 동작하므로 다른 프로세스와는 동기화할 수 없고 하나의 **프로세스 내부의 스레드끼리만 동기화** 가능

### 뮤텍스 (Mutex)

* **상호 배제(Mutual Exclusion)** 의 약자

* **커널 모드**에서 관리하기에 프로세스와 스레드 둘 다 사용 가능

* **소유권(Ownership)** 개념이 존재해 **lock**을 획득한 객체가 소유권을 가지며 **해당 객체만이 lock을 해제**할 수 있음

* 단 **한 개**의 프로세스, 스레드만이 소유권을 가질 수 있음

### 세마포어 (Semaphore)

* **커널 모드**에서 관리하기에 프로세스와 스레드 둘 다 사용 가능

* 크리티컬 섹션이나 뮤텍스와 달리, 동시에 **원하는 개수의 프로세스, 스레드가 접근**하도록 설정할 수 있음

* 보통 **큐**, **버퍼**와 같이 여러 개의 공간을 갖는 객체에서 사용함

* **카운터**를 기반으로 동작하며, 소유권 개념이 없어 **모든 스레드가 카운터 증가 및 감소**시킬 수 있음
