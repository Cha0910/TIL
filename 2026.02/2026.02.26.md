## 2. 블루프린트 캐릭터 및 레벨 구현

### 블루프린트 캐릭터 생성

* 블루프린트를 통한 커스텀 캐릭터 생성 및 Mesh 적용, 카메라 추가 방법 학습
* Enhanced Input을 활용하여 커스텀 캐릭터의 이동, 점프, 카메라 조작 구현 및 매핑

### 플랫포머 레벨 구현

* 일정 시간마다 리스폰하는 구체를 특정 방향으로 이동시켜, 플레이어와 충돌 시 캐릭터를 강력하게 밀어내는 장애물 구현함
* 단순한 Mesh 추가와 `Simulate Physics`를 `true`로 설정하면 플레이어와 충돌할 경우 캐릭터와 충돌하면 캐릭터를 밀지 않고 구체가 멈춤
* 따라서 구체에 틱마다 `Add Force`로 힘을 주어 특정 방향으로 이동시키고 `On Component Hit`와 `Launch Charater`로 플레이어를 밀음

### 장애물 충돌 트러블슈팅

* **현상:** 캐릭터를 조작하여 구체에 부딪히면 Hit 이벤트가 발생하지만, 구체가 캐릭터를 향해 다가와 충돌할 때는 Hit 이벤트가 발생하지 않고 구체가 멈춰버림
* **원인:** Hit 이벤트가 발생하려면 충돌하는 두 액터 중 하나는 `Simulation Generates Hit Events` 옵션이 `true`로 설정되어야 하지만 둘 다 `false`로 설정되어 있었음
* **해결:** 구체의 `Simulation Generates Hit Events` 옵션을 `true`로 설정하여 해결

### 캐릭터를 이동하면 충돌이 발생한 이유
* 캐릭터가 이동할 때 사용하는 `Character Movement Component`는 이동하기 전 경로에 충돌 여부를 검사하는 `Sweep` 방식을 통해 충돌을 판정함
* 구체가 이동하는 것은 단순 시뮬레이션이므로 `Block` 처리는 되지만 이벤트 발생은 하지 않음
* 즉 캐릭터 이동 중 발생한 충돌은 `Simulation Generates Hit Events` 옵션과 관계없기에 제한적으로 충돌 이벤트가 발생한 것

## 3. 장착 아이템 구조 설계

### 기존의 아이템 구조 수정
* 기존 `ItemDefinition`의 `ReferenceItem`은 하위 클래스의 프로퍼티를 포함하지 못하므로 `CreateItemCopy()`를 `가상 함수`로 선언해 다형성을 챙김
  ```cpp
  // 기존의 ReferenceItem 방식
  ReferenceItem = NewObject<UItemDefinition>(this, UItemDefinition::StaticClass());
  
  ReferenceItem->ID = ItemDataRow->ID;
  ReferenceItem->ItemType = ItemDataRow->ItemType;
  ReferenceItem->ItemText = ItemDataRow->ItemText;
  ReferenceItem->WorldMesh = ItemDataRow->ItemBase->WorldMesh;

  // 가상 함수로 교체 ItemDefinition.h 에서 virtual로 선언함
  UItemDefinition* UItemDefinition::CreateItemCopy() const 
  {
    UItemDefinition* ItemCopy = NewObject<UItemDefinition>(StaticClass());

    ItemCopy->ID = this->ID;
    ItemCopy->ItemType = this->ItemType;
    ItemCopy->ItemText = this->ItemText;
    ItemCopy->WorldMesh = this->WorldMesh;
      
    return ItemCopy;
  }
  ```

### 장착 아이템 구조 설계

* 캐릭터에게 장착하는 아이템을 정의하고 액터를 생성하였으며 아래의 프로퍼티를 필요로 함
* 장착 아이템은 1인칭, 3인칭을 구분하기 위한 각각의 `Mesh`와 `애니메이션`
* 아이템을 소유하고 장착하는 `오너 캐릭터`
* 아이템 사용을 위한 `MappingContext`

## 4. 다형성과 가상 함수
장착 아이템 구조 설계에 가상 함수를 사용하였는데, 다형성과 가상 함수는 객체 지향에서 매우 중요한 개념이기에 복습하는 기회를 가짐

* **다형성:** 같은 이름의 메서드나 함수가 다른 동작을 하게끔 구현하는 것, 이를 통해 코드의 재사용성, 유지 보수성, 유연성을 향상할 수 있으며 주로 `상속`, `오버로딩`, `오버라이딩`을 통해 구현
* **가상 함수:** 객체 지향 언어에서 부모 클래스가 자식 클래스에서 `오버라이딩`할 것으로 정의한 함수, `가상 함수 테이블`을 통해 런타임 중 실제 객체의 함수를 호출함

### 가상 함수 예시
```cpp
#include <iostream>

class Parent {
public:
    virtual void Print() // virtual로 선언
    { 
        std::cout << "부모"; 
    }
};

class Child : public Parent {
public:
    void Print() override 
    { 
        std::cout << "자식"; 
    }
};

int main() {
    Parent* p = new Child();  // 부모 포인터로 자식 객체를 가리킴(업캐스팅)
    p->Print();               // Output: 자식
                              // print()가 virtual로 선언 되었으므로 자식의 print()를 실행
                              
    delete p;
    return 0;
}
```
