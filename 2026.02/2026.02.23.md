### 2. 에디터 주요 인터페이스 핵심 정리

공식 문서를 읽으며 각 모드의 역할, 주요 기능들을 `UnrealLearningKit` 에셋 환경에서 실습

#### 메인 툴바 (Main Toolbar): 작업 모드

| 모드 이름 | 주요 기능 및 역할 |
| --- | --- |
| **선택** | 액터 선택, 이동, 회전, 스케일 조절 |
| **랜드스케이프** | 거대한 지형을 생성 및 조각 |
| **폴리지** | 나무, 풀, 돌 같은 에셋을 넓은 영역에 배치 |
| **메시 페인트** | 액터의 버텍스 컬러나 텍스처를 수정 |
| **모델링** | 엔진 내에서 직접 메쉬를 생성하거나 변형 |
| **프랙처** | 파괴 가능한 오브젝트 제작 |
| **브러시** | 레벨 디자인 초기에 사용하는 레벨 생성 툴 |
| **애니메이션** | 엔진 내에서 애니메이션 작업 |

####  주요 패널 구성

| 구성 요소 | 설명 |
| --- | --- |
| **메뉴바** | 파일 저장, 편집, 창 설정 등 일반적인 애플리케이션 액션 수행 |
| **뷰포트 툴바** | 트랜스폼, 스냅, 카메라 등을 설정 하는 툴바  |
| **레벨 뷰포트** | 현재 로드된 레벨을 시각적으로 확인하고 작업하는 창 |
| **아웃라이너** | 현재 레벨에 배치된 모든 액터를 계층 구조로 보여줌 |
| **디테일 패널** | 선택한 액터의 다양한 프로퍼티 표시 및 수정 |
| **콘텐츠 드로어** | 모든 에셋을 관리하는 브라우저 창 |
| **하단 툴바** | 출력 로그, 명령줄 입력 등 개발 중 디버깅 및 파이프라인 확인 |

### 3. 배운점 및 느낀점

**1. Unity 엔진과의 유사성과 차이점** 

이전에 유니티 엔진을 학습한 경험이 있었기에 두 엔진 간 유사한 기능들이나 대응이 되는 것들이 여럿 있었다. 언리얼 엔진의 `GameObject`, `Outliner`, `Details`는 각각 유니티 엔진의 `Actor`, `Hierarchy`, `Inspector` 와 대응한다고 생각되었고 이외에도 다양한 기능들이 유사하다고 느껴진다.

언리얼의 `Mode` 전환 기능이 가장 크게 차이가 느껴졌다. 언리얼에서 모드를 변경하면 에디터가 그 모드의 전용으로 전환된다는 느낌을 받았다. 유니티는 `Terrain` 같은 기능도 Component이기에 이러한 차이가 크게 느껴진 듯하다.

**2. C언어 복습**

실시간 강의를 통하여 학부 1학년 때 배운 내용을 복습할 수 있었다. 대부분 간단한 내용이었지만 수업을 따라가고 문서를 읽어보며. C언어의 기초와 구조를 다시 생각해보는 기회가 되었다.
