### 2. 강의를 통한 언리얼 에디터의 기초 학습 및 용어 정리

어제 자습으로 학습한 뷰포트, 아웃라이너, 디테일 등에 대하여 `UnrealLearningKit` 에셋 환경에서 추가로 학습 

### 에디터 인터페이스 심화 학습

* **뷰포트(Viewport):** 현재 로드된 레벨을 시각적으로 확인하고 작업하는 창
* **아웃라이너(Outliner):** 현재 레벨에 배치된 모든 액터를 계층 구조로 보여줌
* **디테일(Details):** 선택한 액터의 다양한 프로퍼티 표시 및 수정

### 공간 및 오브젝트 체계

| 용어 | 핵심 정의 | 
| --- | --- | 
| **월드 (World)** | 게임 전체를 담는 최상위 컨테이너 (레벨, 물리 환경 포함) |
| **레벨 (Level)** | 월드를 구성하는 단위 (유니티의 `'Scene'`과 유사) |
| **액터 (Actor)** | 월드에 배치 가능한 모든 오브젝트의 기본 단위 |
| **폰 (Pawn)** | 플레이어나 AI가 **빙의(Possess)** 하여 조종할 수 있는 액터 |

### 빙의 (Possess)

* **정의:** 컨트롤러(Controller)가 특정 폰(Pawn)을 점유하여 주도권을 갖는 행위
* **규칙:** 기본적으로 **하나의 컨트롤러는 하나의 폰**만 조종할 수 있음

### 액터(Actor)와 폰(Pawn) 비교

* **액터:** 월드에 배치 되는 모든 객체의 부모 클래스, 트랜스폼(위치/회전/크기) 정보를 가지지만, 스스로 움직이거나 입력을 처리하는 기능이 없음
* **폰:** 액터를 상속받아 **빙의** 기능이 추가된 클래스로, 플레이어의 의지나 AI 가 조종 가능

### 3. 1인칭 캐릭터 구현 및 트러블 슈팅

공식 문서의 [1인칭 어드벤처 게임 코딩하기](https://dev.epicgames.com/documentation/ko-kr/unreal-engine/code-a-firstperson-adventure-game-in-unreal-engine)를 참고하여 언리얼 엔진에서 C++ 클래스를 생성하는 방법 학습 및 실습

### C++ 클래스 생성 및 블루프린트 확장

* `AGameModeBase`를 상속받은 `AdventureGameMode` C++ 클래스 생성 및 `BP_AdventureGameMode` 블루프린트 생성
* `ACharacter`를 상속받은 `AdventureCharacter` C++ 클래스 생성 및 `BP_AdventureCharacter` 블루프린트 생성

### 입력과 캐릭터 무브먼트 링크

* 기존의 `BP_FirstPersonCharacter`의 점프 입력, 무브먼트 입력 로직 이해
* `Move()`, `Jump()`, `Look()` 함수 구현 및 `BP_AdventureGameMode`에 바인딩
* 입력 매핑 컨텍스트를 캐릭터에 바인딩

### Mesh(메시), 카메라 컴포넌트 생성 및 애니메이션 할당

* 메시, 카메라 컴포넌트 생성 및 `AdventureCharacter`에 `Attach(어태치)`
* 언리얼 에디터에서 메시 할당
* 캐릭터에 애니메이션 추가

### C++ 입력 바인딩 후 에셋 할당 프로퍼티 미표시

* **현상:** `AdventureCharacter`에서 `UPROPERTY()`를 사용하여 코드를 빌드했음에도, 블루프린트 에디터 디테일 패널에서 에셋을 할당할 수 있는 프로퍼티가 나타나지 않음
* **원인:** `BP_AdventureGameMode`의 `Default Pawn Class`가 언리얼의 기본 설정인 `DefaultPawn`으로 설정되어 있어 `AdventureCharacter` 스폰되지 않음
* **해결:** `BP_AdventureGameMode`의 `Default Pawn Class`를 `BP_AdventureCharacter`로 수정

### 1인칭 애니메이션 적용 시 그림자 미동작

* **현상:** 화면상에서는 캐릭터의 애니메이션이 정상 출력되지만, 그림자는 움직이지 않고 `T-Pose`로 고정됨
* **원인:** 1인칭용 메시와 3인칭용 메시가 구분되어 구현했으며, 애니메이션은 1인칭용 메시에만 적용되고 있었음
* **해결:** `AAdventureCharacter::BeginPlay()` 내부에 아래 코드를 추가하여 3인칭 메시에도 동일한 애니메이션이 적용되게 수정

```cpp
// 3인칭 메시(GetMesh)에 애니메이션 인스턴스 클래스를 설정하여 그림자 애니메이션 동기화
if (FirstPersonDefaultAnim)
{
    GetMesh()->SetAnimInstanceClass(FirstPersonDefaultAnim->GeneratedClass);
}

```

### 4. C++과 블루프린트 비교

`Alex Forsythe`의 [Blueprints vs. C++: How They Fit Together and Why You Should Use Both](https://www.youtube.com/watch?v=VMZftEVDuCE) 영상을 참고하여 C++과 블루프린트의 장단점 및 설계 방식을 학습함

| 구분 | C++ (Programming) | 블루프린트 (Scripting) |
| --- | --- | --- |
| **성능** | **높음** (컴파일 시 기계어로 변환) | **상대적으로 낮음** (가상 머신 기반 바이트코드 실행) |
| **사용 편의성** | 컴파일 시간이 필요해 반복 작업 불리 | 실시간 수정 및 테스트가 가능해 반복 작업에 유리 |
| **비교 및 병합** | 텍스트 기반으로 코드 리뷰 및 병합에 유리 | 바이너리 파일로 에디터를 필요로 해 병합은 제한적|  
| **주 사용처** | 반복 루프, 물리 계산, 기본 데이터 구조 | 에셋 및 시각효과 |
