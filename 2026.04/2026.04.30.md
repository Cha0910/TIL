### [피보나치 수](https://school.programmers.co.kr/learn/courses/30/lessons/12945)

```cpp
#include <string>
#include <vector>

using namespace std;

int solution(int n) {
    int answer = 0;
    vector<long long> dp(n+1, 0);
    dp[1] = 1;
    for(int i=2; i<dp.size(); i++)
        dp[i] = (dp[i-2] + dp[i-1]) % 1234567;
    
    return dp[n];
}
```

* **DP** 문제
* 가장 대표적인 DP의 기초 문제인 피보나치 수열

### [서버 증설 횟수](https://school.programmers.co.kr/learn/courses/30/lessons/389479)

```cpp
#include <string>
#include <vector>
#include <queue>

using namespace std;

int solution(vector<int> players, int m, int k) {
    int answer = 0;
    queue<int> q;
    
    for(int i=0; i<players.size(); i++){
        while(!q.empty() && q.front() == i) q.pop();
        
        int server = players[i] / m;
        for(int j=q.size(); j<server; j++) {
            q.push(i+k);
            answer++;
        }
    }
    return answer;
}
```

* **queue**를 활용하여 해결
* `players`의 크기가 `24`로 고정되어 있어서 각 시간을 모두 확인하면 무난히 풀림

### [완전범죄](https://school.programmers.co.kr/learn/courses/30/lessons/389480)

```cpp
#include <string>
#include <vector>
#include <set>
using namespace std;


int solution(vector<vector<int>> info, int n, int m) {
    int answer = 121;
    vector<set<pair<int, int>>> index(info.size());
    if(info[0][0] < n) index[0].insert({info[0][0], 0});
    if(info[0][1] < m) index[0].insert({0, info[0][1]});
    
    for(int i=1; i<info.size(); i++){
        for(const pair<int, int>& data : index[i-1]){
            int a = data.first, b = data.second;
            if(a + info[i][0] < n) index[i].insert({a + info[i][0], b});
            if(b + info[i][1] < m) index[i].insert({a, b + info[i][1]});
        }
        if(index[i].size() == 0) return -1;
    }
    
    pair<int, int> result = *index[info.size()-1].begin();
    answer = result.first;

    return answer;
}
```

* **DP**, **완전 탐색**을 활용한 문제
* `info`의 크기가 최대 `40` 이므로 단순히 완전 탐색을 하면 시간 초과
* 이전의 결과를 저장해두고 이를 기반으로 계산해야 함

## 3. std::unordered_map의 성능 하락

* **std::unordered_map** 은 내부적으로 **해시 테이블**을 기반으로 구현되어 있어 삽입, 삭제, 접근에 $O(1)$의 시간 복잡도를 가짐
* 하지만 다양한 요인으로 인하여 성능에 하락이 발생할 수 있음

### 해시 충돌
* 서로 다른 키의 해시 함수 결과가 같을 경우 **해시 충돌**이 발생함
* 이때 **체이닝**이나, **개방 주소법**을 통해 다른 위치에 데이터를 저장해야 함
* 충돌로 인한 **저장 과정에서도 오버헤드**가 발생하며, 이후 **데이터에 접근할 때도 오버헤드**가 발생함

### 재해싱
* **std::unordered_map** 또한 **std::vector**와 유사하게 **capacity**를 미리 할당해 둠
* **capacity** 이상으로 요소가 추가되면 **재할당**과 **복사** 과정을 가짐
* **std::unordered_map**의 크기를 예상할 수 있다면, **생성자**나 **reverse()** 를 통해 미리 공간을 할당해 줄 수 있음

### 캐시 미스
* **std::unordered_map**은 각 요소를 저장할 때 별도의 노드를 저장하며 **std::list**와 유사하게 요소들이 메모리에 흩어져서 저장됨
* 이는 **캐시 히트율**이 낮아져 성능 하락에 영향을 줌
* **std::unordered_map**은 충돌 발생 시 **체이닝**을 통해 노드를 저장하는데, 체이닝 또한 노드 연결이기에 캐시 히트율이 낮음

## 4. std::vector push_back(), emplace_back()의 차이

* `push_back()`과 `emplace_back()`은 둘 다 벡터의 가장 뒤에 요소를 추가하는 함수
* 대부분의 상황에서 동일하게 사용할 수 있으나 **내부 작동 방식**에 차이가 있음

### push_back()
* **rvalue**를 인자로 넣을 시 외부에서 **임시 객체**를 생성하고, 임시 객체를 함수에 인자로 넘겨줌
* 인자로 받은 임시 객체를 `push_back()` 내부에서 **이동 생성자**를 통해 다시 생성
* 내부에서 생성한 객체를 벡터에 추가한 이후 외부에서 생성한 임시 객체를 **소멸** 시킴

### emplace_back()
* **rvalue**를 인자로 넣을 시 외부에서 **임시 객체**를 생성하지 않고, **함수 내부**에서 임시 객체를 생성
* 내부에서 생성한 임시 객체를 바로 벡터에 추가하기에 `push_back()`과 비교하여 **이동 생성**과 **소멸**을 줄일 수 있음
* **lvalue**를 인자로 넣을 경우, `push_back()`과 동일하게 내부에서 복사 생성자를 호출함
* 하지만 `explicit` 생성자를 무시하며, **가독성** 및 **의도 전달**이 부족하기에 상황에 따라 선택해서 사용

### 예제 코드
* **이동 생성자, 소멸자**
    ```cpp
    #include <iostream>
    #include <vector>

    class Item{
    private:
        int Price;
    public:
        Item(int price) : Price(price){
            std::cout << "생성자 호출" << std::endl;
        }
        Item(Item&& other) noexcept : Price(std::move(other.Price)){
            std::cout << "이동 생성자 호출" << std::endl;
        }
        ~Item(){
            std::cout << "소멸자 호출" << std::endl;
        }
    };

    int main() {
        std::vector<Item> vec;
        vec.reserve(2);
        
        // 외부에서 객체를 생성한 후 인자로 삽입
        vec.push_back(Item(100));   // Output:
                                    // 생성자 호출
                                    // 이동 생성자 호출
                                    // 소멸자 호출

        // 받은 인자로 내부에서 소멸자를 실행
        vec.emplace_back(100);      // Output:
                                    // 생성자 호출
        return 0;
    } // Output:
    // 소멸자 호출
    // 소멸자 호출
    ```

* **`explicit` 생성자**
    ```cpp
    #include <iostream>
    #include <vector>

    class Item{
    private:
        int Price;
    public:
        explicit Item(int price) : Price(price){
            std::cout << "생성자 호출" << std::endl;
        }
    };

    int main() {
        std::vector<Item> vec;
        vec.reserve(2);
        
        // vec.push_back(100);  // 100은 int형이므로 컴파일 에러

        vec.emplace_back(100);  // 내부에서 생성자를 호출하므로 정상적으로 호출됨
                                // 원래 의도한 코드라면 괜찮지만, 의도하지 않은 동작이 발생할 수 있음
        return 0;
    } 
    ```

* **2차원 배열**
    ```cpp
    #include <iostream>
    #include <vector>

    int main() {
        std::vector<vector<int>> vec;
        
        // vec.push_back(100);  // 100은 int형이므로 컴파일 에러

        vec.emplace_back(100);  // 내부에서 생성자를 호출하므로 크기가 100인 vector<int>를 저장
                                // 의도하지 않은 경우에도 호출될 가능성이 있음
        return 0;
    } 
    ```
