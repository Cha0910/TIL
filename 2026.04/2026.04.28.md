#include <string>

#include <numeric>

### [징검다리](https://school.programmers.co.kr/learn/courses/30/lessons/43236)

```cpp
#include <string>
#include <vector>
#include <algorithm>

using namespace std;
bool check(const vector<int>& rocks, int n, int mid){
    int current = 0;
    int count = 0;
    
    for(const int& rock : rocks){
        if(rock - current < mid) count++;
        else current = rock;
        
        if(count > n) return false;
    }
    
    return true;
}

int solution(int distance, vector<int> rocks, int n) {
    int answer = 0;
    int left = 0, right = distance;
    
    sort(rocks.begin(), rocks.end());
    rocks.push_back(distance);
    
    while(left <= right){
        int mid = (left + right) / 2;
        
        if(check(rocks, n, mid)){
            answer = mid;
            left = mid + 1;
        }
        
        else right = mid - 1;
    }
    
    return answer;
}
```

* **이분탐색** 문제
* **가능한 거리**를 이분 탐색으로 찾음
* 현재 돌과 다음 돌의 거리를 구한 후 그 거리가 목표 거리보다 작으면, 돌을 제거
* 제거한 돌이 `n`보다 크면, 현재 목표 거리는 불가능 한 것

### [아방가르드 타일링](https://school.programmers.co.kr/learn/courses/30/lessons/181186)


```cpp
#include <string>
#include <vector>
using namespace std;

int solution(int n) {
    vector<int> answer = {0, 1, 3, 10};
    if(n<4) return answer[n]; 
    
    vector<long long> dp(n+1, 0);
    vector<long long> pref(3, 0); // [0] = 6, 9, 12 // [1] = 4, 7, 10 // [2] = 5, 8, 11
    vector<int> pattern = {4, 2, 2};
    long long MOD = 1000000007;

    dp[1] = 1, dp[2] = 3, dp[3] = 10;
    
    for(int i=4; i<dp.size(); i++){
        dp[i] = (dp[i-3] * 5 + dp[i-2] * 2 + dp[i-1]) % MOD;
        
        dp[i] = (dp[i] + pattern[i%3]) % MOD;
        
        dp[i] = (dp[i] + pref[i%3] * 4 + pref[(i+1)%3] * 2 + pref[(i+2)%3] * 2) % MOD;
        
        pref[i%3] = (pref[i%3] + dp[i-3]) % MOD;
    }
       
    return dp[n];
}
```

* **DP** 문제
* 다른 DP 문제들과 달리 모든 경우에 **유일한 방법**이 존재함
* 꽤 특이한 유형이라 이해가 많이 어려움

## 3. std::vector와 std::list

* `std::vector`와 `std::list`는 **C++ STL**에서 지원하는 **시퀀스 컨테이너**
* 둘의 사용 방식은 비슷하지만, **내부 구조**는 매우 차이가 크기에 각자의 장단점이 존재
* 대부분의 상황에서는 `std::vector`가 더 유리하지만 이유를 알아보고자 함

### std::vector

* **배열**기반의 자료구조

* 배열 기반이기에 데이터가 실제로 **연속된 메모리** 공간에 존재

* **임의 접근**과 **가장 뒤에 원소 추가 및 삭제**는 $O(1)$ 의 시간복잡도를 가짐

* 하지만 **임의의 위치에 원소 추가 및 삭제**는 벡터 안의 모든 값을 이동시켜야 하기에 $O(n)$ 의 시간복잡도를 가짐

* 벡터는 현재 원소의 개수보다 더 많은 공간을 할당해 두고 공간이 부족해지면 **추가로 공간을 할당 후 복사**

* 이는 매우 큰 비용을 가지기에 `reserve()` 혹은 `생성자`를 통해 공간을 미리 할당할 수 있음

### std::list

* **이중 연결 리스트** 기반의 자료구조

* 실제로 연속된 메모리 공간이 아닌 **포인터**를 통해 **논리적으로 연결**된 구조

* 데이터 외에도 앞, 뒤의 **포인터를 저장**해야 하므로 **더 큰 메모리 공간을 필요**로 함

* 임의로 접근할 수 없어 **순차 접근**을 통해 데이터에 접근해야 하며, 이는 $O(n)$ 의 시간복잡도를 가짐

* 임의의 위치에 원소 추가 및 삭제는 $O(1)$ 의 시간복잡도로 수행 가능

* 또한, 원소가 추가될 시 공간을 할당한 후 리스트에 붙여주기만 하면 되기에 **사전 할당**, **재할당**, **복사**가 필요 없음

* 하지만, **사전 할당**을 하지 않고 원소가 추가될 때마다 할당하는 것은 오히려 **오버헤드**를 가짐

### 차이점

| 구분 | std::vector | std::list |
| :--- | :--- | :--- |
| **내부 구조** | **연속된 메모리** 공간 배치 | 노드들이 **포인터**로 연결된 구조 |
| **접근 방식** | **임의 접근** | **순차 접근** |
| **접근 성능** | $O(1)$ | $O(n)$  |
| **맨 뒤 삽입/삭제** | $O(1)$  | $O(1)$ |
| **중간 삽입/삭제** | $O(n)$  | **$O(1)$**  |
| **메모리 할당** | 부족 시 **재할당 및 전체 복사** 발생 | 원소 추가 시마다 **개별 할당** |

### 캐시 효율성

* 차이점을 비교해 보면, 접근은 많이 하지 않고 임의의 위치에 할당, 삭제가 잦은 경우에는 `std::list`를 사용하는 게 유리해 보임

* 극단적인 경우에는 더 유리할 수 있겠지만, 대부분의 경우에서는 `std::vector`가 더 빠름

* **CPU**가 **메인 메모리**에 있는 데이터를 읽는 것은 **매우 높은 비용**을 가짐

* **현대의 CPU**는 이를 해결하기 위해 **캐시 메모리**를 활용함

* **캐시 메모리**는 **CPU** 내부에 있는 메모리이기 때문에 매우 빠른 속도로 접근 가능

* **CPU**가 **메인 메모리**에 있는 데이터를 읽을 때, 그 주소를 시작 주소로 하여 **일정 크기의(캐시 라인) 데이터를 캐시 메모리**에 가져옴

* `std::vector`는 연속된 메모리 공간을 갖기 때문에 다른 데이터가 **같이 캐시에 올라감**

* `std::list`는 논리적으로 연결된 구조이기 때문에 **캐시에 올라가지 않을 확률이 높음**

* 따라서 **순차 접근** 시 동일한 $O(1)$ 의 시간복잡도를 갖지만, **캐시 히트율**로 인해 `std::vector`가 더 빠른 속도를 가짐

* 심지어 **삽입**, **삭제** 또한 옮겨야 할 데이터의 수가 적다면, `std::vector`가 `std::list`보다 더 빠를 수 있음
