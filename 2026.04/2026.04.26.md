## 2. 스마트 포인터

### 스마트 포인터란?
* 전통적인 `C++` 에서 동적 할당할 때 `new`, `delete`를 사용하여 사용자가 직접 자원을 관리함
* 이러한 방식은 **메모리 누수**, **댕글링 포인터** 등과 같은 문제를 발생시킬 수 있음
* **스마트 포인터**는 **RAII** 원칙으로 메모리를 자동으로 관리해 위와 같은 문제들을 방지하기 위한 **클래스**

### 스마트 포인터의 작동 원리
* **스마트 포인터**는 생성자에서 메모리를 할당하고, 소멸자에서 해제하는 **RAII** 원칙으로 구현됨
* **스마트 포인터**는 내부적으로 new를 통해 메모리를 할당해 **원시 포인터**를 갖고있음
* 하지만, 사용자는 직접적으로 원시 포인터에 접근할 수 없으며 **연산자 오버로딩**을 통해 일반 포인터처럼 사용 가능
* 또한, **스마트 포인터**는 **소유권** 개념을 통해 **이중 해제를 방지**함

## 3. 스마트 포인터의 종류

### auto_ptr
* **C++98**에서 추가된 최초의 **스마트 포인터**, **독점적인 소유권**을 통해 이중 해제를 방지함

* `auto_ptr`은 복사가 되지 않고 **소유권이 이전**되기에 일반적인 복사 연산자와 다르게 동작함

* 이로 인해 이미 **해제된 메모리에 접근**하는 실수가 잦았음

* **동적 배열**을 할당해도 소멸자에서 `delete` 연산자로 해제했기 때문에, **사용자의 직접 해제**를 필요로 했음

* 이러한 단점들로 인해 **C++11**부터 **사용 중지**를 권고했으며, **C++17**에서 **삭제**됨

* **예제 코드** (C++17 미만에서만 실행 가능)
    ```cpp
    #include <iostream>
    #include <memory>

    int main() {
        std::auto_ptr<int> ptr1(new int(10));
        std::auto_ptr<int> ptr2 = ptr1; // 복사가 아닌 소유권이 ptr2로 넘어감

        if (ptr1.get() == nullptr) {
            std::cout << "ptr1 is nullptr" << std::endl;
        }

        // *ptr1 = 20; // 런타임 에러

        std::cout << "*ptr2: " << *ptr2 << std::endl;   // Output: *ptr: 10

        return 0;
    }
    ```

### unique_ptr
* `auto_ptr`의 단점을 개선하여 **C++11**에서 추가된 스마트 포인터

* `auto_ptr`과 같이 하나의 메모리에 대하여 **유일한 소유권**을 가지며, 복사 연산자 사용을 **컴파일러**에서 막음

* `unique_ptr`에서는 소유권 이전을 `move()` 함수를 통해 **명시적으로 이전**해 사용자의 실수를 방지

* 객체 생성 시 `make_unique` 함수를 통해 안정성과 가독성을 챙길 수 있음

* **예제 코드**
    ```cpp
    #include <iostream>
    #include <memory>

    int main() {
        std::unique_ptr<int> ptr1 = std::make_unique<int>(10);
        // std::unique_ptr<int> ptr2 = ptr1;            // 복사 시 컴파일 에러

        std::unique_ptr<int> ptr2 = std::move(ptr1);    // move() 함수로 명시적으로 소유권 이전

        if (ptr1.get() == nullptr) {
            std::cout << "ptr1 is nullptr" << std::endl;    // Output: ptr1 is nullptr
        }
        std::cout << "*ptr2: " << *ptr2 << std::endl;       // Output: *ptr: 10

        return 0;
    }
    ```

### shared_ptr
* `unique_ptr`와 달리 하나의 메모리를 **여러 포인터가 공유**할 수 있는 스마트 포인터

* **레퍼런스 카운트**를 통해 메모리를 해제해 **메모리 누수**와 **이중 해제**를 방지

* 해당 메모리를 참조하는 `shared_ptr`이 추가될 경우 **레퍼런스 카운트**가 `1` 증가하고, 사라지면 `1` 감소

* **레퍼런스 카운트**가 `0`이 되면 자동으로 메모리를 해제

* **레퍼런스 카운트**는 **Control Block**이라는 별도의 메모리 공간에서 관리하기에, 약간의 **오버헤드**가 존재

* `shared_ptr`은 **순환 참조**로 인해 해제가 안 되는 문제가 발생할 수 있으며, 이를 `weak_ptr`로 해결

* **예제 코드**
    ```cpp
    #include <iostream>
    #include <memory>

    int main() {
        // shared_ptr 생성
        std::shared_ptr<int> ptr1 = std::make_shared<int>(10);

        std::cout << ptr1.use_count() << std::endl;         // Output: 1
        
        {
            std::shared_ptr<int> ptr2 = ptr1;               // ptr1을 ptr2로 복사
            std::cout << ptr1.use_count() << std::endl;     // Output: 2
        }

        std::cout << ptr1.use_count() << std::endl;         // Output: 1
        
        return 0;
    } 
    ```

* **순환 참조 예제 코드**
    ```cpp
    #include <iostream>
    #include <memory>

    class Node{
    public:
        std::shared_ptr<Node> next;

        ~Node(){
            std::cout << "소멸자 실행" << std::endl;
        }
    };

    int main() {
        std::shared_ptr<Node> ptr1 = std::make_shared<Node>();
        std::shared_ptr<Node> ptr2 = std::make_shared<Node>(); 

        // 서로의 멤버가 서로를 가리키는 순환 참조
        ptr1->next = ptr2;
        ptr2->next = ptr1;

        return 0;
    } // 소멸자가 실행되지 않음
    ```

### weak_ptr
* `shared_ptr`의 **순환 참조** 문제를 해결하기 위한 **약한 참조 스마트 포인터**

* `weak_ptr`는 `shared_ptr`와 연동해 사용하며, `shared_ptr`의 **레퍼런스 카운트에 영향을 주지 않음**

* `weak_ptr`은 `shared_ptr`의 생명 주기에 영향을 주지 않으므로 참조 시 메모리가 **이미 해제** 되어있을 수 있음

* 따라서 `weak_ptr`로 참조 중인 메모리에 접근하기 위해서는 `lock()` 함수를 통해 `shared_ptr`로 전환해 **소유권**을 얻어야 함

* `lock()` 함수가 실패할 경우 `shared_ptr` 타입의 `nullptr`을 반환함

* **예제 코드**
    ```cpp
    #include <iostream>
    #include <memory>

    class Node{
    public:
        std::weak_ptr<Node> next;   // weak_ptr 사용

        ~Node(){
            std::cout << "소멸자 실행" << std::endl;
        }

        void Print(){
            std::cout << "노드 접근 성공" << std::endl;
        }
    };

    int main() {
        std::shared_ptr<Node> ptr1 = std::make_shared<Node>();
        std::shared_ptr<Node> ptr2 = std::make_shared<Node>(); 

        // 서로의 멤버가 서로를 가리키는 순환 참조
        ptr1->next = ptr2;
        ptr2->next = ptr1;

        // weak_ptr이기에 레퍼런스 카운트가 증가되지 않음
        std::cout << "ptr1 use count: " << ptr1.use_count() << std::endl;   
        std::cout << "ptr2 use count: " << ptr2.use_count() << std::endl;   
        // Output: 
        // ptr1 use count: 1
        // ptr2 use count: 1

        // lock()을 통해 shared_ptr<Node>로 전환
        if(std::shared_ptr<Node> ptr3 = ptr1->next.lock()){
            ptr3->Print();  // Output: 노드 접근 성공
        }
    
        return 0;
    } 
    // 레퍼런스 카운트가 0이 돼 정상적으로 소멸자가 실행
    // Output:
    // 소멸자 실행
    // 소멸자 실행
    ```
