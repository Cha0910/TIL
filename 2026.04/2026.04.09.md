## 2. 프로그래머스 문제 풀이

### [기능개발](https://school.programmers.co.kr/learn/courses/30/lessons/42586)

```cpp
#include <string>
#include <vector>
#include <cmath>

using namespace std;

vector<int> solution(vector<int> progresses, vector<int> speeds) {
    vector<int> answer; 
    int time = (100-progresses[0]) % speeds[0] == 0 ? 
            (100-progresses[0]) / speeds[0] : (100-progresses[0]) / speeds[0] + 1;
    int max = time;
    int count = 1;
    
    for(int i=1; i<progresses.size(); i++){
        time = (100-progresses[i]) % speeds[i] == 0 ? 
            (100-progresses[i]) / speeds[i] : (100-progresses[i]) / speeds[i] + 1;
        
        if(time > max) {
            max = time;
            answer.push_back(count);
            count = 1;
        }
        else {
            count++;
        }
    }
    answer.push_back(count);
    return answer;
}
```

* **그리디** 방식으로 풀이
* 가장 앞에 있는 작업이 종료된 시간을 기준으로, 그보다 먼저 끝난 작업들의 개수를 세는 방식
* 풀고 나서 보니 문제 유형이 **Queue** 였는데 이 방식이 더 나은 듯

### [미로 탈출 명령어](https://school.programmers.co.kr/learn/courses/30/lessons/150365)

```cpp
#include <string>
#include <vector>
#include <cmath>
using namespace std;

// n, m: 지도 크기 / x, y: 현재 위치 / r, c: 도착지 / k: 남은 이동
void DFS(const int& n, const int& m, int& x, int& y, const int& r, const int& c, int& k, 
         string& path, string& answer){
    if(answer.length() > 0) return;                 // 이미 최단 거리를 구했으면 탐색 안함
    
    int shortest = abs(r-x) + abs(c-y);             // 현재 위치에서 최단거리
    if(k - shortest < 0 || (k - shortest) % 2 == 1) // 남은 이동 - 최단 거리가 음수거나 홀수면 도달 불가
        return;
    
    if(x==r && y==c && k==0){   // 목적지에 도달하면 종료
        answer = path;
        return;
    }
    
    // d
    if(x+1 < n){
        path += 'd';
        DFS(n, m, ++x, y, r, c, --k, path, answer);
        path.erase(path.size()-1);
        x--; k++;
    }
    
    // l
    if(y-1 >= 0){
        path += 'l';
        DFS(n, m, x, --y, r, c, --k, path, answer);
        path.erase(path.size()-1);
        y++; k++;
    }
    
    // r
    if(y+1 < m){
        path += 'r';
        DFS(n, m, x, ++y, r, c, --k, path, answer);
        path.erase(path.size()-1);
        y--; k++;
    }
    
    // u
    if(x-1 >= 0){
        path += 'u';
        DFS(n, m, --x, y, r, c, --k, path, answer);
        path.erase(path.size()-1);
        x++; k++;
    }
}

string solution(int n, int m, int x, int y, int r, int c, int k) {
    string answer = "";
    string path = "";
    DFS(n, m, --x, --y, --r, --c, k, path, answer);
    if(answer.length() == 0) answer = "impossible";
    return answer;
}
```

* **DFS**, **백트래킹** 방식으로 풀이
* `k`의 최댓값이 `2500`이라 `4^2500` 가지의 경우의 수가 있는 것 같지만, 사전 순으로 탐색을 하고 중간중간 프루닝이 많이 돼서 시간 초과가 안 나는 것 같음
* 탐색으로 안 하고 수학적으로 계산해 보려다가 포기

### [쌍둥이 빌딩 숲](https://school.programmers.co.kr/learn/courses/30/lessons/140105)

```cpp
#include <string>
#include <vector>
#include <iostream>
using namespace std;

int solution(int n, int count) {
    int answer = 0;
    vector<vector<long long>> dp(n+1, vector<long long>(n+1, 0));
    dp[0][0] = 1;
    
    for(int i=1; i<dp.size(); i++){
        for(int j=i; j<dp.size(); j++){
            dp[i][j] = dp[i-1][j-1] + dp[i][j-1] * (j-1) * 2;
            dp[i][j] %= 1000000007;
        }
    }

    answer = dp[count][n];
    return answer;
}
```

* 코드는 간단한 **DP** 문제
* `dp[i][j]`는 `빌딩이 j개` 있을 때, 정면에서 `i개의 빌딩이 보이는 경우의 수`를 의미
* 빌딩을 역순으로 하나씩 쌓는다고 생각하면 편함
* 새로운 빌딩을 추가할 때, `i`를 동일하게 추가한다면 가장 앞이 아닌 어디에 넣어도 상관없음
* `i+1`개가 보이게 추가한다면, 가장 앞에 놓는 수밖에 없음
* 따라서 점화식은 위와 같이 됨

### [인사고과](https://school.programmers.co.kr/learn/courses/30/lessons/152995)

```cpp
#include <string>
#include <vector>
#include <algorithm>

using namespace std;

int solution(vector<vector<int>> scores) {
    int answer = 0;
    pair<int, int> target = {scores[0][0], scores[0][1]};
    
    sort(scores.begin(), scores.end(), [](const vector<int>& A, const vector<int>& B){
        if(A[0] != B[0]) return A[0] > B[0];
        else return A[1] < B[1];
    });
    
    int max = -1;
    for(int i=0; i<scores.size(); i++){
        if(max > scores[i][1]) {
            if(scores[i][0] == target.first &&  scores[i][1] == target.second)
                return -1;
            continue;
        }
        else max = scores[i][1];
        
        if(scores[i][0] + scores[i][1] > target.first + target.second) answer++;
    }
    
    return ++answer;
}
```

* **정렬**, **그리디** 문제
* 정렬을 어떻게 해야 할지 생각해 내는 것이 굉장히 어려웠음
* 근무 태도(앞의 점수)는 내림차순, 동료 평가(뒤의 점수)는 오름차순으로 정렬
* 동료 평가 점수는 오름차순이기에 근무 태도 점수가 같다면 `max`만 업데이트 됨
* 이후에 만난 동료 평가 점수가 `max`보다 낮으면 근무 태도 점수도 낮다는 것을 의미하므로 제외

## 3. Anim Graph

* `애니메이션 블루프린트(ABP)`내에 존재하는 매 프레임별 캐릭터가 **어떤 포즈**를 취해야 하는지 연산함
* 핵심적인 요소로는 **상태 머신(State Machines)**, **블렌드 스페이스(Blend Spaces)**, **애니메이션 노드(Anim Nodes)**, **컨트롤 리그(Control Rig)** 등이 있음

### 상태 머신(State Machines)
* 캐릭터의 현재 상태를 정의하여 행동 흐름을 제어함
* 오직 하나의 상태만을 갖고 있을 수 있음
* **트랜지션(Transition):** 상태 간의 이동 조건(Transition Rule)에 따라 다른 상태로 이동
* **트랜지션 블렌딩(Transition Blending):** 상태가 전환될 때 끊기지 않도록 엔진이 자동으로 두 상태의 포즈를 보간(Interpolation)하는 시간

### 블렌드 스페이스(Blend Spaces)
* 여러 개의 애니메이션 시퀀스를 **특정 변수에 따라 보간**하여 재생
* 1차원(1D) 또는 2차원(2D) 그래프 위에 애니메이션을 배치해 변수 값에 따라 애니메이션의 가중치를 계산

### 애니메이션 노드(Anim Nodes)
* 애님 그래프 내부에서 애니메이션 데이터를 조작, 혼합, 출력하는 모든 개별 블록들을 의미

### 컨트롤 리그(Control Rig)
* 애니메이션 시퀀스가 제공하는 고정된 포즈를 실시간 물리 환경, 상황에 맞춰 동적으로 수정
* **FK (Forward Kinematics):** 부모 뼈에서 자식 뼈로 순차적으로 움직임이 전달됨
* **IK (Inverse Kinematics):** 역으로 자식 뼈의 위치가 움직이면 부모 뼈의 움직임이 계산됨
* 주요 기능으로는 캐릭터가 자연스럽게 서 있을 수 있도록 하는 **Foot IK**, 시선을 따라 목, 척추를 자연스럽게 움직이는 **Look at**이 있음
