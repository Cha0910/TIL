### [연속 부분 수열 합의 개수](https://school.programmers.co.kr/learn/courses/30/lessons/131701)

```cpp
#include <string>
#include <vector>
#include <unordered_set>
using namespace std;

int solution(vector<int> elements) {
    int answer = 0;
    // 합을 저장하는 해시셋
    unordered_set<int> s;

    // 원형 큐를 선형으로 탐색하기 위해 확장
    elements.insert(elements.end(), elements.begin(), elements.end());  
    
    // 누적합 계산
    for(int i=1; i<elements.size(); i++)
        elements[i] += elements[i-1];
    
    // 구간합 게산 후 셋에 저장
    for(int i=0; i<elements.size()/2; i++)
        for(int j=1; j<=elements.size()/2; j++)
            s.insert(elements[j+i] - elements[i]);
    
    answer = s.size();
    return answer;
}
```

* `누적합`, `unordered_set`을 활용하여 해결하는 문제
* 원형 데이터의 특징은 **크기를 2배**로 늘리면 **선형으로 탐색**할 수 있다는 것
* 이후 **누적합**을 저장한 후 **구간 합**을 계산해 해시셋에 저장
* 다른 코테 문제에도 많이 나오는 **원형 데이터**, **누적합**을 배울 수 있는 좋은 문제인 듯

### [숫자 타자 대회](https://school.programmers.co.kr/learn/courses/30/lessons/136797)

```cpp
#include <string>
#include <vector>
#include <queue>
#include <climits>
using namespace std;

// 시작지와 막힌 지점을 받아, 다른 지점들까지의 최단 거리를 계산
vector<int> dijkstra(vector<vector<pair<int, int>>>& graph, int start, int blocked){
    vector<int> result(10, 100);
    priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>> pq;
    pq.push({0, start});
    result[start] = 1;
    
    while(!pq.empty()){
        int current = pq.top().second;
        int weight = pq.top().first;
        pq.pop();
        
        for(const pair<int, int>& edge : graph[current]){
            int next = edge.second;
            int newWeight = edge.first + weight;
            if(next == blocked) continue;
            if(newWeight < result[next]){
                result[next] = newWeight;
                pq.push({newWeight, next});
            }
        }
    }
    
    return result;
}

int solution(string numbers) {
    int answer = INT_MAX;
    
    // [출발점][막힌곳][도착점]
    vector<vector<vector<int>>> weight(10, vector<vector<int>>(10, vector<int>(10, INT_MAX)));   
    
    // [숫자자리][왼손위치][오른손위치] = 현재 가중치
    vector<vector<vector<int>>> dp(numbers.length()+1, 
                                           vector<vector<int>>(10, 
                                           vector<int>(10, INT_MAX)));
    // 시작 위치 초기화
    dp[0][4][6] = 0;
    
    // 그래프 연결 저장
    vector<vector<pair<int, int>>> graph = {
        {{3, 7}, {2, 8}, {3, 9}},                                           // 0
        {{2, 2}, {2, 4}, {3, 5}},                                           // 1
        {{2, 1}, {2, 3}, {3, 4}, {2, 5}, {3, 6}},                           // 2
        {{2, 2}, {3, 5}, {2, 6}},                                           // 3
        {{2, 1}, {3, 2}, {2, 5}, {2, 7}, {3, 8}},                           // 4
        {{3, 1}, {2, 2}, {3, 3}, {2, 4}, {2, 6}, {3, 7}, {2, 8}, {3, 9}},   // 5
        {{3, 2}, {2, 3}, {2, 5}, {3, 8}, {2, 9}},                           // 6
        {{2, 4}, {3, 5}, {2, 8}, {3, 0}},                                   // 7
        {{3, 4}, {2, 5}, {3, 6}, {2, 7}, {2, 9}, {2, 0}},                   // 8
        {{3, 5}, {2, 6}, {2, 8}, {3, 0}}                                    // 9
    };
    
    // weight 계산
    for(int i=0; i<10; i++){
        for(int j=0; j<10; j++){
            if(i == j) continue;
            weight[i][j] = dijkstra(graph, i, j);
        }
    }
    
    
    for(int i=0; i<numbers.length(); i++){
        int next = numbers[i] - '0';    // 다음 목표 숫자
        for(int j=0; j<10; j++){        // 왼손 위치
            for(int k=0; k<10; k++){    // 오른손 위치
                if(dp[i][j][k] == INT_MAX || j == k) continue;
                if(next != k) dp[i+1][next][k] = min(dp[i+1][next][k], dp[i][j][k] + weight[j][k][next]);
                if(next != j) dp[i+1][j][next] = min(dp[i+1][j][next], dp[i][j][k] + weight[k][j][next]);
            }
        }
    }
    
    // 주어진 숫자를 입력완료한 경우 중 최솟값을 저장
    for(int i=0; i<10; i++)
        for(int j=0; j<10; j++)
            answer = min(answer, dp[numbers.length()][i][j]);

    return answer;
}
```

* **다익스트라**, **DP**를 활용해 해결
* 어떤 손가락을 움직이냐에 따라 미래의 결과가 바뀌므로 **그리디**는 불가능
* 문자열의 길이가 최대 `100,000`이므로 **완전 탐색**도 무조건 시간초과
* 두 손가락의 위치가 고정되어 있을 때, 다음 숫자를 누르기 위한 비용은 고정되어 있으므로 **DP**로 해결 가능
* 그래프를 코드로 구해도 되겠지만, 노드가 10개밖에 안 되기에 하드코딩 함
* 시작점(움직일 손가락이 있는 버튼)과 막힌 지점(다른 손가락이 있는 버튼)으로 **다익스트라**를 실행해 각 상황마다 숫자의 가중치를 미리 계산
* 이후 구한 가중치로 **DP**연산을 하면 정답을 구할 수 있음

### [스티커 모으기](https://school.programmers.co.kr/learn/courses/18/lessons/1881)

```cpp
#include <vector>

using namespace std;

int solution(vector<int> sticker)
{
    int answer = 0;
    int length = sticker.size();
    
    // 스티커를 뗀 경우, 안 뗀 경우, 역순으로 뗀 경우, 안 뗀 경우
    vector<vector<int>> dp(4, vector<int>(length, 0)); 
    if(length < 2) return sticker[0];
    
    dp[0][0] = sticker[0]; dp[1][1] = sticker[0];
    dp[1][0] = 0; dp[0][1] = sticker[1];
    dp[2][0] = sticker[length-1]; dp[3][1] = sticker[length-1];
    dp[3][0] = 0; dp[2][1] = sticker[length-2];
    
    for(int i=2; i<sticker.size()-1; i++){
        dp[0][i] = max(dp[1][i-1], dp[0][i-2]) + sticker[i];
        dp[1][i] = max(dp[0][i-1], dp[1][i-1]);
        
        dp[2][i] = max(dp[3][i-1], dp[2][i-2]) + sticker[length-i-1];
        dp[3][i] = max(dp[2][i-1], dp[3][i-1]);
    }
   
    answer = max(dp[0][length-2], dp[1][length-2]);
    answer = max(answer, dp[2][length-2]);
    answer = max(answer, dp[3][length-2]);
    
    return answer;
}
```

* **DP**, **원형 데이터** 유형의 문제
* 이전에 풀었던, [도둑질](https://school.programmers.co.kr/learn/courses/30/lessons/42897)과 완벽하게 동일한 문제임
* 원형이라는 특징 때문에, 첫 번째 스티커를 뗀 여부가 마지막 스티커를 뗄 수 있는지 여부에 영향을 줌
* 따라서 **마지막 스티커를 제외**하고 최댓값을 구함
* 역순으로 **DP**를 같이 실행하면, 동일한 개념이 적용되므로 각 **DP**의 값중 최댓값이 정답임

### 접근 지정자
* `public`: 어디에서나 멤버에 접근이 가능, **struct**의 기본 접근 지정자이며 주로 

* `private`: 해당 **struct**, **class** 내부 또는 후술할 **friend** 함수에서만 접근 가능, **class**의 기본 접근 지정자

* `protected`: 외부에서는 접근할 수 없지만, 해당 **struct**, **class** 내부 또는 상속 관계를 갖는 **자식**은 접근 가능

### 혼용 상속 시 상속 접근 지정자

* `C++`에서 `struct`와 `class`는 기본 접근 지정자를 제외하고는 모두 동일하게 작동하기에 당연히 가능함

* 하지만, **기본 접근 지정자**가 다르다는 점 때문에 **암시적 상속**시 **상속 접근 지정자**가 다르게 적용 됨

* 자식이 `struct` 일 경우, `public`이 상속 접근 지정자, 자식이 `class`일 경우 `private`이 상속 접근 지정자로 설정됨

* 코드를 작성할 때 상속 접근 지정자를 **명시**하면 동일하게 사용 가능

```cpp
#include <iostream>
#include <string>

struct BaseStruct {
    int value = 10;
};

class BaseClass {
public:
    std::string name = "Name";
};

struct StructFromStruct : BaseStruct {
    // StructFromStruct : pulic BaseStruct 와 같음
    // 암시적 public 상속: 외부에서 value 접근 가능
};

struct StructFromClass : BaseClass {
    // StructFromClass : pulic BaseClass 와 같음
    // 암시적 public 상속: 외부에서 name 접근 가능
};

class ClassFromStruct : BaseStruct {
    // ClassFromStruct : private BaseStruct 와 같음
    // 암시적 private 상속: 외부에서 value 접근 불가
};

class ClassFromClass : BaseClass {
    // ClassFromClass : private BaseClass 와 같음
    // 암시적 private 상속: 외부에서 name 접근 불가
};

int main() {
    // A (struct:struct)
    StructFromStruct a;
    std::cout << "A (struct:struct): " << a.value << " (접근 성공)" << std::endl;

    // B (struct:class)
    StructFromClass b;
    std::cout << "B (struct:class): " << b.name << " (접근 성공)" << std::endl;

    // C (class:struct)
    ClassFromStruct c;
    // std::cout << "C (class:struct): " << c.value << std::endl; // private 상속이므로 접근 불가

    // D (class:class)
    ClassFromClass d;
    // std::cout << "D (class:class): " << d.name << std::endl; // private 상속이므로 접근 불가

    return 0;
}
```

### friend

* **접근 지정자를 무시**하고, 외부에서 `private`, `protected`로 선언된 멤버에 접근할 수 있게 해주는 키워드

* 어떤 `클래스 A`가 `friend` 에게 자신의 `private`, `protected`로 선언된 멤버에 접근 가능한 권한을 주는 것이라 생각하면 됨

* **friend 전역 함수**, **friend 클래스**, **friend 멤버 함수**로 총 세 가지의 사용 방법이 있음

```cpp
#include <iostream>
#include <string>

class Base; // 전방 선언
class FriendMemberFunc {
public:
    void MemberFunc(const Base& base); 
};

class Base {
private:
    int value = 100;

    // friend 전역 함수
    friend void GlobalFunc(const Base& base);

    // friend 클래스
    friend class FriendClass;

    // friend 멤버 함수
    friend void FriendMemberFunc::MemberFunc(const Base& base);
};


// friend 전역 함수
void GlobalFunc(const Base& base) {
    std::cout << "GlobalFunc : " << base.value << std::endl;
}

// friend 클래스
class FriendClass {
public:
    void AccessAll(const Base& base) {
        std::cout << "FriendClass (클래스 전체): " << base.value << std::endl;
    }
};

// friend 멤버 함수
void FriendMemberFunc::MemberFunc(const Base& base) {
    std::cout << "FriendMemberFunc (특정 멤버): " << base.value << std::endl;
}

int main() {
    Base base;

    GlobalFunc(base);                   // 출력: GlobalFunc : 100

    FriendClass friendClass;            
    friendClass.AccessAll(base);        // 출력: FriendClass (클래스 전체): 100

    FriendMemberFunc FriendMemberFunc;
    FriendMemberFunc.MemberFunc(base);  // 출력: FriendMemberFunc (특정 멤버): 100

    return 0;
}
```

### OOP의 4대 특징
* `추상화`: 객체의 중요한 부분만을 표현해 정의하는 것

* `상속`: 기존 클래스의 멤버를 물려받은 자식 클래스가 부모의 멤버를 **재사용**, **확장** 하는 것

* `다형성`: 어떤 객체의 멤버가 상황에 따라 **여러 형태로 변화**하는 것, **오버라이딩**, **오버로딩**을 통해 구현

* `캡슐화`: 객체 내부의 멤버를 하나의 캡슐로 묶어 외부에서의 **접근을 차단** 하는 것, `C++`은 **접근 지정자**를 통해 캡슐화를 제공함

### TObjectPtr\<T>

* 기존의 원시 포인터 `UObject*` 를 대체하기 위한 **스마트 포인터**

* 에디터 빌드 시에 **지연 로딩** 및 **액세스 추적**의 기능을 제공하며, 비 에디터 빌드 시에는 `UObject*`와 동일하게 동작함

* `TObjectPtr`은 함수로 전달되거나 변수에 저장될 때 `UObject*`로 변환되므로 성능 하락이 없음

* 기존의 `UObject*`가 사용 불가능한 것은 아니지만, 멤버 변수에 `TObjectPtr`을 사용하는 것을 권장함


### TSubClassOf\<T>
* 언리얼 엔진에서 지원하는 특정 클래스나, 특정 클래스의 하위 클래스만 참조할 수 있도록 제한하는 템플릿
* 이를 통해 타입 안정성과 에디터 편의성을 향상시킬 수 있음

### TArray\<T>
* `STL`의 `vector`와 유사한 컨테이너

* 연속된 메모리 위치를 가지며, 가비지 컬렉터를 지원함

* **주요 멤버 함수**

    | 함수명 | 설명 |
    |---|---|
    |**Add(Item)**|요소를 배열의 끝에 추가 (복사 또는 이동 발생)|
    |**Emplace(Item)**|요소를 배열의 끝에 추가 (배열 내부 메모리에 요소를 직접 생성)|
    |**AddUnique(Item)**|배열에 동일한 요소가 없으면 끝에 추가|
    |**Insert(Item, Index)**|지정한 인덱스 위치에 요소를 삽입|
    |**Append(OtherArray)**|OtherArray의 모든 요소를 현재 배열 끝에 추가|
    |**Remove(Item)**|배열 내에서 일치하는 모든 요소를 삭제|
    |**RemoveAt(Index)**|특정 인덱스의 요소를 삭제|
    |**RemoveAtSwap(Index)**|특정 인덱스의 요소를 삭제한 뒤, 마지막 요소를 그 자리에 채워 넣음|
    |**Empty(NewCapacity)**|배열의 모든 요소를 제거하며, 인자를 줄 경우 해당 크기만큼 메모리를 할당|
    |**Num()**|배열에 담긴 요소의 개수를 반환|
    |**IsValidIndex(Index)**|입력한 인덱스가 유효 범위인지 확인|
    |**Top()**|배열의 마지막 요소에 대한 참조를 반환|
    |**Contains(Item)**|배열에 특정 값이 포함되어 있는지 여부를 반환|
    |**Find(Item, OutIndex)**|특정 값의 인덱스를 찾음, 찾으면 true를 반환하고 OutIndex에 Index를 저장|
    |**Reserve(Number)**|메모리를 미리 할당|
    |**Shrink()**|사용되지 않는 여유 공간(Slack)을 해제|

### TSet\<T>

* `STL`의 `unordered_set`과 유사한 컨테이너
* **Sparse Array** 방식으로 메모리에 저장해 **캐시 히트** 효율을 높임, 가비지 컬렉터를 지원함

* 주요 멤버 함수

    |함수명(인자)|설명|
    |---|---|
    |**Add(Item)**|요소를 집합에 추가, 이미 있으면 기존 요소를 덮어씀|
    |**Emplace(Item)**|요소를 배열의 끝에 추가 (내부 메모리에서 요소를 직접 생성)|
    |**Remove(Item)**|일치하는 요소를 찾아 삭제|
    |**Contains(Item)**|특정 요소가 집합 내에 존재하는지 여부를 확인|
    |**Find(Item)**|요소를 찾아 해당 요소의 포인터를 반환|
    |**Num()**|집합에 들어있는 요소의 개수를 반환|
    |**Empty(NewCapacity)**|집합의 모든 요소를 제거하며, 인자를 줄 경우 해당 크기만큼 메모리를 할당|
    |**Append(OtherSet)**|다른 집합의 모든 요소를 현재 집합에 추가|
    |**Intersect(OtherSet)**|두 집합의 교집합|
    |**Union(OtherSet)**|두 집합의 합집합|
    |**Reserve(Number)**|메모리를 미리 할당|
    |**Shrink()**|사용되지 않는 여유 공간(Slack)을 해제|
    |**CreateIterator()**|집합을 순회하기 위한 가변 반복자를 생성
    |**CreateConstIterator()**|집합을 순회하기 위한 상수 반복자를 생성

### TMap\<KeyType, ValueType>

* `STL`의 `unordered_map`과 유사한 컨테이너

* `TSet<T>`과 동일하게 **Sparse Array** 방식으로 메모리에 저장해 **캐시 히트** 효율을 높임, 가비지 컬렉터를 지원함

* 주요 멤버 함수

    |함수명(인자)|설명|
    |---|---|
    |**Add(Key, Value)**|새로운 키-값 쌍을 추가 혹은, 기존 키의 값을 업데이트|
    |**Emplace(Key, Value)**|새로운 키-값 쌍을 추가 혹은, 기존 키의 값을 업데이트 (내부 메모리에서 요소를 직접 생성)|
    |**Remove(Key)**|특정 키와 그에 대응하는 값을 삭제, 삭제 성공 시 삭제된 개수를 반환|
    |**Find(Key)**|키에 해당하는 값의 포인터를 반환|
    |**FindOrAdd(Key)**|키를 찾으면 값을 반환하고, 없으면 새로 추가하여 반환|
    |**FindRef(Key)**|키에 해당하는 값의 복사본을 반환, 없으면 기본 생성된 값 반환|
    |**GenerateKeyArray(OutArray)**|맵의 모든 키들만 추출하여 TArray로 만듦|
    |**GenerateValueArray(OutArray)**|맵의 모든 값들만 추출하여 TArray로 만듦|
    |**Contains(Key)**|해당 키가 맵에 존재하는지 여부를 반환|
    |**Num()**|현재 맵에 저장된 키-값 쌍의 총 개수를 반환|
    |**Empty(NewCapacity)**|맵의 모든 요소를 제거하며, 인자를 줄 경우 해당 크기만큼 메모리를 할당|
    |**Append(OtherMap)**|다른 맵의 모든 요소를 현재 맵으로 복사하여 합침|
    |**Reserve(Number)**|지정된 개수만큼의 요소를 담을 메모리를 미리 할당
    |**Shrink()**|사용되지 않는 여유 공간(Slack)을 해제|
    |**CreateIterator()**|맵을 순회하기 위한 가변 반복자 생성
    |**CreateConstIterator()**|맵을 순회하기 위한 상수 반복자 생성|
