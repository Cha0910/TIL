## 2. 프로그래머스 문제 풀이

### [카펫](https://school.programmers.co.kr/learn/courses/30/lessons/42842)

```cpp
#include <string>
#include <vector>

using namespace std;

vector<int> solution(int brown, int yellow) {
    int sum = brown + yellow;
    
    for (int y=3; y*y<=sum; y++){
        if(sum % y == 0){ 
            int x = sum / y; 
            if((x-2) * (y-2) == yellow) return {x, y};
        }
    }
}
```

* **완전 탐색** 문제
* 가능한 `x`, `y`의 **모든 쌍을 검사** 해 정답을 찾음
* 미지수 2개, 식 2개기 때문에 수학적으로도 풀 수 있을 것 같음

### [스티커 모으기(2)](https://school.programmers.co.kr/learn/courses/30/lessons/12971)

```cpp
#include <iostream>
#include <vector>
using namespace std;

int solution(vector<int> sticker)
{
    vector<int> dp1(sticker.size(), 0), dp2(sticker.size(), 0);
    if(sticker.size() < 2) return sticker[0];
    
    dp1[0] = sticker[0]; dp1[1] = dp1[0]; 
    dp2[1] = sticker[1]; 
    
    for(int i=2; i<sticker.size(); i++){
        dp1[i] = max(dp1[i-2] + sticker[i], dp1[i-1]);
        dp2[i] = max(dp2[i-2] + sticker[i], dp2[i-1]);;
    }

    return max(dp1[sticker.size()-2], dp2[sticker.size()-1]);
}
```

* **DP** 문제
* 숫자가 붙었길래 뭔가 다를 줄 알았지만, [스티커 모으기](https://school.programmers.co.kr/learn/courses/18/lessons/1881)와 똑같은 문제
* 그래도 이전이 풀었던 기억을 살려 깔끔하게 풀이

## 3. 컨테이너

### 컨테이너란?
* 다른 객체들을 보관하는 **저장소**같은 개념
* **C++** 의 **STL** 에서는 다양한 컨테이너를 지원해 줌
* **STL** 에서 지원하는 컨테이너는 크게 **순차 컨테이너**, **연관 컨테이너**, **컨테이너 어댑터**로 나눌 수 있음

### 순차 컨테이너 (sequence container)
* 데이터를 **선형적**으로 저장하는 컨테이너
* 대표적인 **순차 컨테이너**는 `vector`, `array`, `list`, `forward_list`, `deque`가 있음

### 연관 컨테이너 (associative container)
* **key와 value의 관계**로 데이터를 저장하는 컨테이너
* 대표적인 **연관 컨테이너**는 `map`, `set`이 있으며 이는 기본적으로 오름차순으로 **정렬**됨
* `map`, `set`과 동일하지만, **중복을 허용**하는 `mulitset`, `multimap`도 존재
* 위 네 개의 컨테이너는 **정렬하지 않는** `unordered_`가 존재하며 내부적으로 **해시**를 사용함

### 컨테이너 어댑터 (Container Adapter)
* 기존의 **순차 컨테이너**를 기반으로 **인터페이스를 제한**한 컨테이너
* **컨테이너 어댑터**는 `stack`, `queue`, `prioriry_queue`가 존재하며 각각 `deque`, `deque`, `vector` 기본값으로 구현됨

## 4. std::map

* **C++** 의 **STL** 에서 지원하는 **key와 value의 관계**로 데이터를 저장하는 컨테이너

* 중복된 **key**를 허용하지 않으며, 기본적으로 **key**를 기준으로 오름차순 **정렬**함

* `std::map`은 내부적으로 **RB-Tree**로 구현되어 있음

* **RB-Tree**는 **자가 균형 이진 탐색 트리**이므로 삽입, 삭제, 접근에 $O(logn)$의 시간복잡도를 가짐

### std::multimap

* 중복된 **key**를 사용해야 하는 경우, `std::multimap`을 사용할 수 있음

* `std::multimap`은 중복 외에는 `std::map`과 동일하게 동작하지만 `[]` 연산자로 **접근은 불가능**

* `std::multimap` 또한 **RB-Tree**로 구현되어 있음

### std::unordered_map

* 정렬이 필요 없을 경우 **해시 테이블**로 구현된 `std::unordered_map`을 사용할 수 있음

* `std::unordered_map`은 삽입, 삭제, 접근에 평균 $O(1)$의 시간복잡도를 가짐

### std::flat_map

* **RB-Tree** 와 **해시 테이블**은 연속된 메모리 공간을 갖지 않기에 **캐시 히트율**이 낮음

* **C++23**에서 `std::vector` 기반의 `std::flat_set`, `std::flat_map`을 추가

* `std::flat_set`, `std::flat_map`은 삽입 시 $O(n)$, 접근 시 $O(logn)$ 시간복잡도를 가지지만, **캐시 히트율**이 높음

* 삽입이 빈번하지 않은 경우, `std::flat_set`, `std::flat_map`을 사용하는 것이 유리함
