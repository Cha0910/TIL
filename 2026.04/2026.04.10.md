### [의상](https://school.programmers.co.kr/learn/courses/30/lessons/42578)

```cpp
#include <string>
#include <vector>
#include <unordered_map>
using namespace std;

int solution(vector<vector<string>> clothes) {
    int answer = 1;
    unordered_map<string, int> counts;
    for(const vector<string>& data : clothes) counts[data[1]]++;
    for(const auto& [category, count] : counts)
        answer *= count+1;
    
    return --answer;
}
```

* **맵**를 이용하는 문제
* `unordered_map`에 선택지의 개수를 저장하고 경우의 수를 계산하면 됨
* 적어도 하나는 입어야 하므로 1을 빼고 리턴

### [선입 선출 스케줄링](https://school.programmers.co.kr/learn/courses/30/lessons/12920)

```cpp
#include <string>
#include <vector>
#include <iostream>
using namespace std;

int solution(int n, vector<int> cores) {
    int answer = 0;
    int left = 0, right = 10000 * 50000;
    int time;
    
    // 모든 일이 종료되는 시간을 이진탐색
    while(left < right){
        int mid = (left + right) / 2;
        long long count = cores.size();
        for(int i=0; i<cores.size(); i++)
            count += mid / cores[i];
        
        if(count >= n) {
            time = mid;
            right = mid;
        }
        else left = mid+1;
    }

    // 종료되는 시간 - 1 시간에 남는 일의 개수로 업데이트
    // 모든 코어를 종료되는 시간 - 1 시간에 다음 작업을 수행하기 까지 남은 시간으로 업데이트
    for(int i=0; i<cores.size(); i++){
        int count = 1 + ((time-1) / cores[i]);
        cores[i] -= ((time-1) % cores[i]);
        n -= count;
    }

    // 종료 되는 시간에 순차적으로 작업을 수행해 모든 작업을 종료하는 코어를 찾음
    for(int i=0; i<cores.size(); i++){
        if(cores[i] == 1) n--;
        if(n == 0) {
            answer = i + 1;
            break;
        }    
    }
    return answer;
}
```

* **이진 탐색**을 활용하는 문제
* 처음에 문제만 보고 **우선순위 큐**로 구현했다가 시간초과
* 핵심은 모든 프로세스가 종료되는 시간은 정해져 있다는 것
* 모든 프로세스가 종료되는 시간 `t`를 이진탐색으로 구한 후 `t-1` 시간의 결과를 구함
* `t-1` 시간의 결과에서 순차적으로 작업을 진행해 마지막 작업을 찾음
* 시간, 큐 같이 나오는 문제는 이진 탐색을 쓰는 경우가 많은 듯

### [캠핑](https://school.programmers.co.kr/learn/courses/30/lessons/1833)

```cpp
#include <vector>
#include <algorithm>

using namespace std;

int solution(int n, vector<vector<int>> data) {
    int answer = 0;
    
    // 2차원 벡터 축소
    vector<int> coodinateI(data.size()), coodinateJ(data.size());
    for(int i=0; i<data.size(); i++){
        coodinateI[i] = data[i][0];
        coodinateJ[i] = data[i][1];
    }
    
    // 정렬 후 중복 제거
    sort(coodinateI.begin(), coodinateI.end());
    coodinateI.erase(unique(coodinateI.begin(), coodinateI.end()), coodinateI.end());
    sort(coodinateJ.begin(), coodinateJ.end());
    coodinateJ.erase(unique(coodinateJ.begin(), coodinateJ.end()), coodinateJ.end());
    
    // data를 축소한 index로 변환
    for(vector<int>& coodinates : data) {
        for(int i=0; i<coodinateI.size(); i++){
            if(coodinates[0] == coodinateI[i]){
                coodinates[0] = i + 1;
                break;
            }  
        }
        for(int i=0; i<coodinateJ.size(); i++){
            if(coodinates[1] == coodinateJ[i]){
                coodinates[1] = i + 1;
                break;
            }  
        }
    }

    // 누적합 연산
    vector<vector<int>> pref(coodinateI.size()+1, vector<int>(coodinateJ.size()+1, 0));
    for(int i=0; i<data.size(); i++){
        int indexI = data[i][0];
        int indexJ = data[i][1];
        pref[indexI][indexJ]++;
    }
    for(int i=1; i<pref.size(); i++)
        for(int j=1; j<pref[i].size(); j++)
            pref[i][j] += pref[i][j-1] + pref[i-1][j] - pref[i-1][j-1];
        
    // 모든 쐐기 쌍에 대하여 계산
    for(int i=0; i<data.size(); i++){
        for(int j=i+1; j<data.size(); j++){
            int i1 = data[i][0], j1 = data[i][1];
            int i2 = data[j][0], j2 = data[j][1];
            if(i1 == i2 || j1 == j2) continue;
            int upI = i1 < i2 ? i1 : i2, downI = i1 > i2 ? i1 : i2;
            int leftJ = j1 < j2 ? j1 : j2, rightJ = j1 > j2 ? j1 : j2;
            upI++; leftJ++; downI--; rightJ--;
            int count = pref[downI][rightJ] 
                - pref[upI-1][rightJ] - pref[downI][leftJ-1] 
                + pref[upI-1][leftJ-1];
            if(count == 0) answer++;
        }
    }
    return answer;
}
```

* **2차원 벡터 압축**, **누적합**을 활용해 해결
* `x`, `y` 좌표의 범위가 `2^30`이지만 쐐기가 최대 `5000`개이므로 **희소 행렬**임
* 이를 축소해 최대 `5000 * 5000`의 배열로 변환
* 조건을 만족하는 쐐기 쌍은, 두 좌표 내부 직사각형의 합이 0인 쐐기 쌍이므로 누적합을 활용
* 누적합 계산에 최대 `5000 * 5000`, 쐐기 쌍 확인에 `5000 * 5000`이므로 통과 가능

## 3. Reflection, UHT, UBT, CDO

### Reflection
* 프로그램이 런타임 중 자기 자신의 구조를 조사하고, 상호작용 하는 기능
* `C++`은 자체적인 리플렉션 기능이 없기 때문에 언리얼 엔진은 **자체적으로 구축한 리플렉션 시스템**을 사용함
* `.h` 파일에 매크로를 작성해 사용할 수 있으며, 컴파일 전 **UHT**가 해당 정보를 수집해 리플렉션 시스템을 지원함
* 리플렉션 시스템을 통해 에디터 및 블루프린트 연동, GC 등을 지원함

### Unreal Header Tool
* 코드가 컴파일되기 전, 프로젝트 내의 모든 `.h` 파일을 스캔하여 매크로를 찾음
* UHT는 매크로의 정보를 파싱해 `.generated.h` 와 `.gen.cpp`를 `Intermediate` 폴더에 생성
* `#include "filename.generated.h"`을 가장 하단에 작성해야 하는 이유임
* 생성된 파일을 통해 리플렉션 시스템을 지원

### Unreal Build Tool
* 언리얼 엔진의 빌드를 총괄하는 시스템
* 빌드를 실행하면 **UBT**가 실행되어 `*.Build.cs`, `*.Target.cs` 파일을 읽어 컴파일할 파일의 목록, 순서를 결정
* 수정된 `.h` 파일이 존재하면 **UHT**를 호출해 해당 `.h` 파일에 대하여 `.generated.h` 파일을 생성하게 지시
* 이후 컴파일러를 실행해 컴파일 및 링킹하여 `.exe`, `.dll` 파일을 생성

### Class Default Object
* 특정 클래스의 **초기 상태**를 저장하고 있는 **인스턴스**
* 모든 `UCLASS()`는 유일한 **CDO**를 갖고 있음
* **CDO**는 엔진 실행 중, 클래스의 생성자가 실행된 후 실행됨
* 새로운 객체를 생성할 때, 모든 데이터를 초기화하지 않고 메모리만 할당 후 **CDO**를 복사해 성능을 챙김
* 또한, **에디터**나 **BP**의 기본값에도 **CDO**의 값이 사용됨
* 객체의 값이 수정되었을 때, 해당 객체의 모든 데이터를 저장하는 게 아닌 **CDO**와 다른 값만 저장하는 `Delta Serialization`에도 사용됨
