### [연속된 부분 수열의 합](https://school.programmers.co.kr/learn/courses/30/lessons/178870)

```cpp
#include <string>
#include <vector>

using namespace std;

vector<int> solution(vector<int> sequence, int k) {
    vector<int> answer(2);
    int left = 0, right = 0;
    int sum = sequence[0], length = sequence.size()+1;
    
    // 투 포인터
    while(right < sequence.size() && left <= right){     
        if(sum > k){                    // 수열의 합이 k 보다 크면
            sum -= sequence[left];      // 최솟값인 왼쪽 끝 원소를 뺌
            left++;                     // 커서를 오른쪽으로 옮김
        }
        else if(sum < k){               // 수열의 합이 k 보다 작으면
            right++;                    // 커서를 오른쪽으로 옮김
            if(right < sequence.size())
                sum += sequence[right]; // sum에 새로운 수를 추가
        }
        else{                           // 수열의 합이 k 이면
            if(right - left < length){  // 수열의 길이가 더 짧을 경우 저장
                answer[0] = left;
                answer[1] = right;
                length = right - left;
            }
            right++;                    // 커서를 옮겨 sum 값에 변화를 줌
            sum += sequence[right];
        }
    }
    
    return answer;
}
```

* **투 포인터** 문제로 많이 나오는 부분 수열의 합 문제
* 부분 수열의 합에 따라서 양쪽의 포인터를 움직여 부분 수열의 합을 바꿔감
* 조건에 맞는 부분 수열일 경우 정답으로 저장

### [입국심사](https://school.programmers.co.kr/learn/courses/30/lessons/43238)

```cpp
#include <string>
#include <vector>
#include <algorithm>

using namespace std;

long long solution(int n, vector<int> times) {
    long long answer = 0;
    sort(times.begin(), times.end());                       // 최대 처리 시간 계산을 위해 정렬
    
    long long min = 1;                                      // 최소 처리 시간
    long long max = times[times.size()-1] * (long long)n;   // 최대 처리 시간
    
    while(min <= max){
        long long mid = (min + max) / 2;    // 현재 루프의 처리 시간
        long long sum = 0;                  // mid 시간 동안 처리 가능한 사람의 수
        
        for(const int& time : times)
            sum += mid / time;
        
        // 시간이 남는 경우
        if(sum >= n){
            answer = mid;
            max = mid-1;
        }
        
        // 시간이 부족한 경우
        else min = mid+1;
    }
    
    return answer;
}
```

* **이분탐색** 알고리즘을 활용하여 해결
* 시간 n 분 동안 처리할 수 있는 **최대 사람의 수는 고정**되어 있다는 것을 이용
* 이분 탐색을 통해 **가장 최소의 시간**을 찾음

### [징검다리 건너기](https://school.programmers.co.kr/learn/courses/30/lessons/64062)

```cpp
#include <string>
#include <vector>

using namespace std;

// 최대 칸수가 k일 때 n 명이 건널 수 있는지 여부
bool check(const vector<int>& stones, int n, int k){
    int count = 0;
    for(const int& stone : stones){
        if(stone < n)                // 돌이 n 보다 낮으면
            count++;
        else count = 0;              // n보다 높은 돌이 있으면 초기화 
        if(count >= k) return false; // k번 연속으로 n보다 낮으면 건널 수 없음
    }
    return true;
}

int solution(vector<int> stones, int k) {
    int min = stones[0], max = stones[0];
    int answer = max;
    
    // 최댓값, 최솟값 저장
    for(const int& stone : stones){
        if(stone < min) min = stone;
        if(stone > max) max = stone;
    }
    
    // 이분 탐색
    while(min < max){
        int mid = (min + max) / 2;
        
        if(check(stones, mid, k)){  // mid 명이 건널 수 있으면
            answer = mid;
            min = mid + 1;
        }
        else max = mid;
    }
    return answer;
}
```

* k 크기의 부분 집합의 최댓값 중에서 최솟값을 찾는 문제
* k 크기의 부분 집합을 실제로 만들어서 최솟값을 구하면 `O(n^2)` 으로 시간초과
* 이분 탐색과 검증 함수를 구현해 `O(nlogn)` 으로 해결

### [땅따먹기](https://school.programmers.co.kr/learn/courses/30/lessons/12913)
```cpp
#include <vector>
#include <algorithm>

using namespace std;

int solution(vector<vector<int>> land)
{
    int answer = 0;
    int n = land.size(), c = 4;
    
    for(int i=1; i<n; i++){                             // 2행부터 시작
        for(int j=0; j<c; j++){                         // 1~4 열을 검사
            int max = land[i][j];                       // i행 j열의 원래 값
            
            for(int k=0; k<c; k++){                     // 이전 행의 1~4 열을 검사
                if(j == k) continue;                    // 동일한 열은 제외
                int temp = land[i][j] + land[i-1][k];   // 이전 행의 값과 현재 행의 값을 더함
                if(max < temp) max = temp;              // 최댓값이면 저장
            }
            
            land[i][j] = max;                           // 현재 칸까지 오는 가장 큰 값 저장
        }
    }
    
    answer = *max_element(land[n-1].begin(), land[n-1].end());  // DP 결과 최댓값 저장
    return answer;
}
```
* C++ 코딩테스트 완전 정복 수강 중 풀어본 **DP** 문제
* 바로 이전 행에서 같은 열을 제외한 나머지 칸들 중 **최댓값**을 선택해 현재 칸에 더함
* 마지막 행의 최댓값을 반환

## 3. 해시 테이블 (Hash Table)
* **Key-Value** 쌍으로 데이터를 저장하는 자료구조
* **해시 함수**를 통해 계산된 값을 인덱스로 하여 데이터를 저장함
* 대표적인 해시 함수는 나눗셈을 이용하는 `Division Method`,  곱셈을 이용하는 `Multiplication Method`, 문자열을 이용하는 `Digit Folding` 방법이 있음  
* 서로 다른 Key가 해시 함수를 거쳐 동일한 인덱스로 배정되는 **해시 충돌(Hash Collision)** 이 발생하면 주로 아래 두 가지 방법을 사용함

### 분리 연결법
* **충돌**이 발생하면 해당 버킷에 **연결 리스트**를 생성하여 데이터를 저장하는 방식
* `STL`의 **unordered 컨테이너**들이 이 방식을 사용
* 구현이 직관적이며, 데이터를 동적으로 저장 가능
* 리스트가 길어지면 검색 시간이 최악의 경우 `O(n)`

### 개방 주소법
* 충돌이 발생하면 테이블 내의 다른 **비어 있는 인덱스**에 데이터를 저장하는 방식
* **선형 탐사:** 충돌 발생 시 **고정된 간격**만큼 이동하며 빈 공간을 찾아 저장, **군집화** 현상이 발생할 수 있음
* **이차 탐사:** 충돌 시 **제곱수** 만큼 이동하며 빈 공간을 찾아 저장, 군집화가 다소 완화되긴 하지만 **이차 군집화** 발생 가능
* **이중 해싱:** 충돌 시 이동 간격을 **다른 해시 함수**로 결정하는 방식, 군집화를 방지하지만 **캐시 미스**의 확률이 올라감

## 4. 인바이런먼트 라이팅 (Environment Lighting)
* 개별 광원이 아닌, **월드 전체**에 적용되는 라이팅 및 배경 환경을 형성하는 데 사용하며 주로 아래의 라이팅, 컴포넌트를 사용함

* **Directional Light:** 무한히 먼 거리에서 평행하게 들어오는 직사광선, 주로 **태양광**을 나타내는데 사용

* **Sky Light:**  **씬의 배경**(하늘, 구름 등)을 캡쳐해 해당 색상과 밝기를 레벨 전체에 **주변광**으로 적용해 전체적인 색감을 조절

* **Sky Atmosphere:** 물리 기반 스카이 및 애트머스피어 렌더링 기술, 고도나 태양의 위치에 따른 **대기 현상** 표현

* **Exponential Height Fog:** 맵의 높이에 반비례하여 **밀도가 변하는 안개**를 생성해 거리감, 원근감을 표현

* **Volumetric Cloud:** 단순 텍스처가 아닌 입체적인 물리 기반 클라우드 렌더링 시스템, 빛의 가림이나 산란을 실시간으로 처리 가능

### 라이트 믹서 (Light Mixer)
* 월드에 존재하는 다양한 라이트를 하나의 창에서 조절할 수 있는 **인터페이스**
* 전등과 같은 일반적인 라이트를 조절하는 **Light Mixer**와 인바이런먼트 라이팅을 조절하는 **Env. Light Mixer**가 존재함
* 좌측 상단의 `Window` 메뉴에서 실행 가능

## 5. 사운드 추가

### 사운드 큐 (Sound Cue)
* 노드를 기반으로 제작, 수정할 수 있는 **오디오 에셋**
* 게임 내 상황에 따라 사운드를 **실시간**으로 조절할 수 있으며 **블루프린트**와 유사한 방식으로 구현할 수 있음

### 사운드 어테뉴에이션 (Sound Attenuation)
* 거리와 위치에 따라 **소리가 어떻게 변화**하는지를 정의하는 설정

* **어테뉴에이션 함수**를 통해 거리에 따라 비율을 결정할 수 있음
    * 선형(Linear): 거리에 비례하여 일정한 비율로 소리가 작아짐
    * 로그(Logarithmic): 거리가 가까울수록 소리의 변화 비율이 
    * 인버스(Inverse): 로그와 유사하지만 더 급격하게 변화
    * 로그 리버스(Log Reverse): 거리가 멀수록 소리의 변화 비율이 커짐
    * 내추럴 사운드(Natural Sound): 로그와 인버스의 중간지점
    * 커스텀(Custom): 직접 커스텀 함수를 정의하여 사용 가능

* **어테뉴에이션 셰이프**를 통해 사운드의 공간적 범위를 정의 할 수 있음
    * Sphere (구체): 중심점에서 모든 방향으로 균일하게 소리가 퍼짐, 대부분의 스팟 포인트에 적합함
    * Capsule (캡슐): 캡슐 모양으로 세로나 가로로 긴 형태의 범위, 파이프나 복도 등에 적합함
    * Box (박스): 직육면체 공간으로 사각형 모양의 방이나 배경에 적합함
    * Cone (원뿔): 특정 방향으로 뻗어나가는 원뿔 모양, 확성기나 스피커에 적합함
