## 2. 프로그래머스 문제 풀이

### [점 찍기](https://school.programmers.co.kr/learn/courses/30/lessons/140107)

```cpp
#include <string>
#include <vector>
#include <cmath>

using namespace std;

long long solution(int k, int d) {
    long long answer = 0;
    
    for(int i=0; i<=d; i+=k){
        long long y = sqrt(pow(d,2) - pow(i,2)); // 현재 x 좌표에서의 y 의 최대값
        answer += y / k + 1;                     // 몫을 더한 후 y 가 0일 때 점을 추가
    }
    
    return answer;
}
```

* O(n^2)으로는 간단하게 풀 수 있지만 문제의 조건이 `1 <= k, d <= 1,000,000` 이라 시간초과

### [추석 트래픽](https://school.programmers.co.kr/learn/courses/30/lessons/17676)

```cpp
#include <string>
#include <vector>
#include <sstream>

using namespace std;

// 문자열을 c 기준으로 나눔
vector<string> split(string str, char c){
    vector<string> result;
    string subword;
    stringstream ss(str);
    
    while (getline(ss, subword, c)) 
        result.push_back(subword);
    
    return result;
}

// hh:mm:ss.sss 형식의 시간 데이터를 int로 변경
int timeToms(string time){
    int ms = 0;
    vector<string> str = split(time, ':');
    vector<string> s = split(str[2], '.');
    
    ms += (stoi(str[0]) * 60*60*1000) + (stoi(str[1]) * 60*1000) + (stoi(s[0]) * 1000) + stoi(s[1]);
    
    return ms;
}

// 종료시간에서 실행된 시간을 뺀 후 반환
int getStartTime(int endTime, string T){
    string t = T.substr(0, T.size()-1);
    int startTime = stod(t) * 1000 - 1;
    return endTime - startTime;
}

int solution(vector<string> lines) {
    int answer = 0;
    
    for(string log : lines){
        vector<string> data = split(log, ' ');
        int endTime = timeToms(data[1]);    // 로그가 종료된 시점
        int checkTime = endTime + 999;      // 종료된 시점으로부터 1초
        int count = 0;
        
        // 종료된 시점으로부터 1초간 다른 로그와 겹치는게 몇개인지
        for(string otherLog : lines){
            vector<string> otherData = split(otherLog, ' ');
            int otherEndTime = timeToms(otherData[1]);
            int otherStartTime = getStartTime(otherEndTime, otherData[2]);
            if(!(otherEndTime < endTime || otherStartTime > checkTime))
                count++;
        }
        
        // 더 크면 저장
        answer = count > answer ? count : answer;
    }
    
    return answer;
}
```
* 현재 로그를 포함하면서 1초간 다른 로그를 가장 많이 포함하는 수를 세는 문제
* 로그의 수가 최대 2,000개이므로 O(n^2)으로 풀어도 해결 가능
* 현재 로그의 종료 시점을 기준으로 뒤로 1초 혹은 시작점 기준으로 앞으로 1초간 겹치는 로그의 수를 세면 됨

### [쿠키 구입](https://school.programmers.co.kr/learn/courses/30/lessons/49995)

```cpp
#include <string>
#include <vector>
#include <algorithm>

using namespace std;

int solution(vector<int> cookie) {
    int answer = 0;
    vector<int> pref(cookie.size()+1, 0);
    
    // 누적합 계산
    for(int i=1; i<pref.size(); i++){
        pref[i] += pref[i-1] + cookie[i-1];
    }
    
    for(int i=1; i<pref.size(); i++){
        for(int j=i; j<pref.size(); j++){
            int sum = pref[j] - pref[i-1];  // 첫째 아들의 과자 수
            
            // j+1번째 바구니 부터의 누적합 중 첫째의 과자 수와 같은 것이 있는지 확인
            if(binary_search(pref.begin()+j+1, pref.end(), pref[j] + sum))
               answer = sum > answer ? sum : answer;
        }
    }
       
    return answer;
}
```

* 누적 합과 **set**, **unordered_set**을 활용해서 풀었지만 효율성 테스트에서 두 방법 다 시간초과
* 문제의 조건이 `1 <= cookie <= 2,000` 이므로 O(n^2logn)으로 해결될거라고 생각했지만 잦은 insert, find 연산으로 시간초과가 되었다고 생각됨
* set을 사용하지 않고 누적 합의 뒷 부분을 `binary_search`로 탐색하여 해결

### [풍선 터트리기](https://school.programmers.co.kr/learn/courses/30/lessons/68646)

```cpp
#include <string>
#include <vector>

using namespace std;

int solution(vector<int> a) {
    int answer = 0;
    vector<bool> left(a.size(), false);     // 왼쪽에 있는 모든 풍선이 자신보다 큰지 여부
    vector<bool> right(a.size(), false);    // 오른쪽에 있는 모든 풍선이 자신보다 큰지 여부
    int leftMin = 1000000001, rightMin = 1000000001;    // 커서 위치까지의 최소값
    
    // left -> right
    for(int i=0; i<a.size(); i++){
        if(a[i] < leftMin){
            left[i] = true;
            leftMin = a[i];
        }  
    } 
    
    // right -> left
    for(int i=a.size()-1; i>=0; i--){
        if(a[i] < rightMin){
            right[i] = true;
            rightMin = a[i];
        }  
        if(left[i] || right[i]) answer++; // 어느 한쪽의 풍선이 모두 자신보다 크면
    }

    return answer;
}
```
* 어떤 풍선 하나를 기준으로 풍선을 두 그룹으로 나눔
* 각 그룹에서 임의로 뽑아서 터트리다 보면 결국 가장 작은 풍선이 남게 됨
* 그룹과 처음 풍선을 다시 합하면 세 개의 풍선이 남게되고 가운데에 처음에 뽑은 풍선이 자리함
* 중앙의 풍선이 셋 중 가장 크면 어떻게 터트려도 터지게 되므로 조건을 만족하는 풍선은 어느 한쪽의 모든 풍선이 자기보다 커야 함

## 3. 퍼즐 어드벤처 디자인하기

### NavMesh (Navigation Mesh)

* AI 캐릭터가 월드 내에서 목표 지점까지 이동 **스스로 이동**하게 하는 데이터 구조

* 기본적으로 **정적(Static)** 으로 생성하지만 레벨 구조가 변경되는 등 필요한 경우 **동적(Dynamic)** 으로 생성할 수 있으나 비용이 꽤 비쌈

* **NavMeshBoundsVolume:** 레벨 에디터에서 배치하는 볼륨, 해당 영역 안에 있는 지형, 메시, 충돌체를 분석하여 **AI가 이동 가능한 영역**을 계산해 초록색 영역으로 보여줌

* **RecastNavMesh:** 볼륨을 배치하면 자동으로 생성되는 액터, 최대 이동 경사각, 에이전트의 크기 등 **세부 옵션을 설정**할 수 있음

* **NavModifierVolume:** 특정 구역의 **이동 비용**을 설정하여 AI가 해당 경로를 **회피**하게 설정할 수 있음

* **NavLinkProxy:** NavMesh가 물리적으로 끊긴 지점을 **수동으로 연결**하는 도구

### Line Trace By Channel
* 적이 플레이어를 충분히 가까이 있고, 시야에 장애물이 없을 때만 추적할 수 있도록 구현하는데 사용
* 블루프린트의 `Line Trace By Channel` 노드의 결과를 기반으로 로직을 구현
* `Line Trace By Channel`의 **Hit Result** 구조체는 아래와 같으며 **C++** 에서도 동일함

    |멤버|정의|
    |---|---|
    |**Blocking Hit** | 히트가 블로킹 히트였는지 여부|
    |**Initial Overlap** | 트레이스가 시작하자마자 파묻힌 상태인지 여부|
    |**Time** | 트레이스 방향상의 임팩트 시간. [0.0 ~ 1.0]|
    |**Distance** | 트레이스 시작에서 충돌 위치까지의 월드 스페이스 거리|
    |**Location** | 히트한 순간 충돌체의 월드 스페이스 위치|
    |**Impact Point** | 히트의 절대 위치|
    |**Normal**	| 충돌한 물체가 발사한 물체에게 주는 노말 벡터|
    |**Impact Normal** | 충돌한 물체(표면)의 노말 벡터|
    |**Phys Mat** | 히트한 물체의 피지컬 머티리얼|
    |**Hit Actor** | 히트한 액터|
    |**Hit Component** | 히트한 액터의 컴포넌트|
    |**Hit Bone Name** | 스켈레탈 메시를 히트한 경우 해당 본의 이름|
    |**Bone Name** | Hit Bone Name과 유사하나 잘 사용하지 않음|
    |**Hit Item** | 인스턴스 중 충돌한 아이템의 인덱스|
    |**Element Index** | 충돌한 Collision Shape의 인덱스|
    |**Face Index** | 충돌한 삼각형 메시의 인덱스|
    |**Trace Start** | 트레이스 시작 위치|
    |**Trace End** | 트레이스 끝 위치|
