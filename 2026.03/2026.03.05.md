## 2. 프로그래머스 [테이블 해시 함수](https://school.programmers.co.kr/learn/courses/30/lessons/147354) 문제 풀이
```cpp
#include <string>
#include <vector>
#include <algorithm>

using namespace std;

int solution(vector<vector<int>> data, int col, int row_begin, int row_end) {
    int answer = 0;
    
    sort(data.begin(), data.end(), [col](const vector<int>& a, const vector<int>& b){
        if(a[col-1] == b[col-1]) return a[0] > b[0]; // col번째 컬럼의 값이 같으면 첫 번째 컬럼 기준 내림차순 정렬
        return a[col-1] < b[col-1];                  // 각 튜플을 col번째 컬럼의 값을 기준으로 오름차순 정렬
    });

    for(int i=row_begin; i<=row_end; i++){ 
        int s = 0;
        for(const int& val : data[i-1]){
            s += val % i;               // S_i 계산
        }
        
        if(i==row_begin) answer = s;    // 첫번 째 S_i 면 대입
        else answer ^= s;               // 이후 XOR 연산
    }   
    
    return answer;
}
```

## 3. CH2 5번 과제 도전 기능

### 타이머

* 일정 시간마다 반복되는 함수를 실행하기 위해 사용, **언리얼 엔진**에서는 `FTimerHandle`와 `FTimerManager` 클래스로 사용 가능

* 액터의 **이동**, **회전** 로직을 단순 반복문으로 구현하면 게임을 시작하자마자 액터의 위치가 마지막 반복의 위치로 이동되기에 언리얼 엔진의 `타이머`를 사용하여 일정 시간마다 함수를 호출함

* 액터가 생성될 때 `SetTimer()`를 사용하여 2초 간격으로 이벤트를 발생하게 구현

  ```cpp
  // MyActor.cpp

  void AMyActor::BeginPlay()
  {
    Super::BeginPlay();

    // 2초마다 RandomEvent 함수를 반복적으로 실행하는 타이머 설정
    GetWorld()->GetTimerManager().SetTimer(
      TimerHandle, this, &AMyActor::RandomEvent, 2.0f, true
    );
  }
  ```

* `AMyActor::RandomEvent()` 내부에 이 이벤트가 10번 실행되면 타이머를 멈추게 함

  ```cpp
  // MyActor.cpp 

  // 이벤트가 10회 실행되면 타이머를 멈추고 결과를 출력
  if (EventCount == 10) {
    GetWorld()->GetTimerManager().ClearTimer(TimerHandle);

    GEngine->AddOnScreenDebugMessage(-1, 5.0f, FColor::Yellow,
      FString::Printf(TEXT("Total Move Distance: %f, Total Turn Angle: %f"), TotalMoveDistance, TotalTurnAngle));

    GEngine->AddOnScreenDebugMessage(-1, 5.0f, FColor::Yellow,
      FString::Printf(TEXT("Total Move Events: %d, Total Turn Events: %d"), MoveCount, TurnCount));	
  }
  ```

### FMath

* 언리얼 엔진에서 제공하는 **수학 관련 유틸리티** 클래스

* 이동 방향 및 거리, 회전축과 각도를 **랜덤으로 설정**하기 위해 `FMath::RandRange()`와 **총 회전 각도**를 계산하기 위해 `FMath::Abs()`를 사용함

* 이외에도 `값 제한 (FMath::Clamp)`, `보간 (FMath::Lerp)`, `삼각함수 (FMath::Sin, FMath::Cos)` 등 다양한 기능을 제공하며 [공식문서](https://dev.epicgames.com/documentation/ko-kr/unreal-engine/math-material-expressions-in-unreal-engine)에서 확인할 수 있음

## 4. 퍼즐 어드벤처 디자인하기

### 레벨 윤곽 작업

* **블로킹 볼륨:** 시각적으로 보이지 않는 **충돌체**, 이탈 방지, 지형 단순화 등에서 주로 사용

* **텍스트 라벨:** **뷰포트 안에 텍스트**를 넣는 액터, 기획 의도 메모, 구역 표시 등을 시각화할 때 주로 사용

### Material, Material Instance

* **Material:** 물체의 표면을 어떻게 보일지 결정하는 **셰이더**, 수정할 경우 셰이더 코드를 다시 **컴파일**해야함

* **Material Instance:** 부모 Material을 복사하여 파라미터만 수정해 사용하는 인스턴스, 컴파일이 필요없어 **실시간**으로 확인 가능

### Construction Script

* 블루프린트의 인스턴스를 생성 시 컴포넌트 리스트 다음으로 실행되는 단계

* `이벤트 그래프`는 **런타임** 중 실시되지만, `Construction Script`는 에디터에서 실행되어 뷰포트에서 **실시간**으로 확인 가능

* 액터를 생성, 이동, 수정할 때마다 실행되므로 복잡한 연산은 **성능 하락**의 원인이 될 수 있음

* 계단, 울타리 같은 **절차적 생성**, Mesh, Material의 **외형 수정**, 각 액터의 속성을 **랜덤으로 설정** 등에 주로 사용

### Blueprint Interface

* 이름만 있고 구현은 없는 **함수** 하나 이상으로 된 집합, 이를 다른 블루프린트에 추가하면 인터페이스의 함수들을 갖게 됨

* 두 액터가 상호작용 시 상대방 액터를 **캐스팅**할 필요 없이 인터페이스를 보유하고 있는지만 확인하여 상호작용 가능하게 해 액터 간의 **의존성**을 제거함

* 동일한 인터페이스로 다른 동작을 하게끔 구현 가능하게 해 **다형성**을 챙김
