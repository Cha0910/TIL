## 2. 프로그래머스 [시소 짝꿍](https://school.programmers.co.kr/learn/courses/30/lessons/152996) 문제 풀이

```cpp
#include <string>
#include <vector>
#include <unordered_map>

using namespace std;

long long solution(vector<int> weights) {
    long long answer = 0;
    unordered_map<int, long long> count;
    
    // 해당 몸무게가 몇 명 있는지 저장
    for(int weight : weights){
        count[weight]++;
    }

    for(int weight : weights){ 
        count[weight]--;  // 중복 계산 방지를 위한 자신 제외
        
        answer += count[weight];              // 1:1
        answer += count[weight * 2];          // 1:2
        if(weight % 2 == 0) {
            answer += count[weight / 2];      // 2:1
            answer += count[weight / 2 * 3];  // 2:3
        }
        if(weight % 3 == 0) {
            answer += count[weight / 3 * 2];  // 3:2
            answer += count[weight / 3 * 4];  // 3:4
        }
        if(weight % 4 == 0) 
             answer += count[weight / 4 * 3]; // 4:3
    }
    return answer;
}
```

* 현재 코드는 `weights`를 순회하면서 가능한 조합을 찾기에 `weights.size()` 만큼 반복문이 실행됨

* 풀고 나서 다시 생각해 보니 `weight`를 기준으로 **조합**을 활용해서 풀면 `count.size()` 만 실행해서 해결 가능하다고 생각됨

* 문제의 조건이 `2 ≤ weights의 길이 ≤ 100,000`이기에 현재 코드로도 충분히 해결 가능

## 3. 게임 개발자를 위한 C++ 문법

### 스마트 포인터
* `C++`에서 포인터 사용 중 발생할 수 있는 **메모리 누수**, **댕글링 포인터**, **이중 해제** 문제를 발생하지 않게 자동으로 관리해주는 **클래스**

* 기존의 `C++`에서의 포인터는 동적 할당을 하면 프로그래머가 직접 해제해야 했지만 스마트 포인터는 **메모리를 자동으로 해제**하는 역할을 함 

* **`unique_ptr`**
  * 객체의 **유일한** 소유권을 관리하는 스마트 포인터

  * 동일한 메모리를 가리키는 스마트 포인터가 동시에 존재할 수 없음
  
  * 복사는 불가능 하지만 **이동**은 가능

    ```cpp
    #include <iostream>
    #include <memory> 
    using namespace std;

    int main() {
        // unique_ptr 생성
        unique_ptr<int> ptr1 = make_unique<int>(10);
        cout << *ptr1 << endl; // Output: 10

        // unique_ptr<int> ptr2 = ptr1; // 컴파일 에러 (복사 불가)
        unique_ptr<int> ptr2 = move(ptr1); // 이동

        if (!ptr1) cout << "ptr1 is nullptr" << endl; // Output: ptr1 is nullptr
        cout << *ptr2 << endl;                        // Output: 10

        return 0;
      }
    ```

* **`shared_ptr`**
  * 하나의 객체를 **여러 포인터**가 참조할 수 있는 스마트 포인터

  * 내부적으로 **레퍼런스 카운터**를 관리하여 카운터가 **0**이 되면 메모리를 해제

  * 순환 참조가 발생할 수 있음

    ```cpp
    #include <iostream>
    #include <memory>
    using namespace std;

    int main() {
        shared_ptr<int> ptr1 = make_shared<int>(100); // shared_ptr 생성
        cout << ptr1.use_count() << endl;             // Output: 1

        {
            shared_ptr<int> ptr2 = ptr1;        // p2에 복사
            cout << ptr1.use_count() << endl;   // Output: 2
        } // 스코프가 종료되며 p2가 해제 됨

        cout << ptr1.use_count() << endl;       // Output: 1

        return 0;
    } 
    ```

* **`weak_ptr`**
  * 레퍼런스 카운트를 증가시키지 않는 약한 참조 방식으로 동작하는 스마트 포인터

  * `shared_ptr`과 같이 사용되며 **순환 참조** 문제를 해결하기 위해 사용

  * 직접 접근할 수 없으며, `lock()`을 통해 `shared_ptr`로 변환해야 접근 가능함

    ```cpp
    #include <iostream>
    #include <memory>
    using namespace std;

    int main() {
        shared_ptr<int> s_ptr = make_shared<int>(10);
        weak_ptr<int> w_ptr = s_ptr;

        cout << s_ptr.use_count() << endl;      // Output: 1

        // 데이터에 접근하려면 lock()을 써야 함
        if (shared_ptr<int> locked_ptr = w_ptr.lock()) 
        {
            cout << *locked_ptr << endl;        // 접근 성공, Output : 10
            cout << s_ptr.use_count() << endl;  // Output: 2
        }                                       // 스코프가 종료되며 weak_ptr로 변경
        
        cout << s_ptr.use_count() << endl;      // Output: 1
        s_ptr.reset();                          // 원본 해제

        if (w_ptr.expired()) {
            cout << "s_ptr is expired" << endl; // 원본이 해제 되었으므로 유효하지 않음
        }

        return 0;
    }
    ```

    ```cpp
    #include <iostream>
    #include <memory>
    using namespace std;

    class B;

    class A {
    public:
        shared_ptr<B> b_ptr;
        ~A() { cout << "A destroyed" << endl; }
    };

    class B {
    public:
        weak_ptr<A> a_ptr; // weak_ptr
        ~B() { cout << "B destroyed" << endl; }
    };

    int main() {
        auto a = make_shared<A>();
        auto b = make_shared<B>();

        a->b_ptr = b;
        b->a_ptr = a; // weak_ptr로 참조

        cout << a.use_count() << endl; // Output: 1 (b의 a_ptr은 weak_ptr이므로 카운트가 증가하지 않음)
        cout << b.use_count() << endl; // Output: 2 (b와 a의 b_ptr)

        return 0;
    } // A가 먼저 해제 된 후 B가 해제 됨
    ```

### Unreal Engine Reflection System

* **Reflection:** 프로그램이 **런타임 중** 자기자신의 구조를 조사하고, 상호작용하는 기능

* `C++`은 자체적인 리플렉션 기능이 없기 때문에 언리얼 엔진은 **자체적으로 구축한 리플렉션 시스템**을 사용함

* 컴파일러가 코드를 컴파일하기 전 **UHT(Unreal Header Tool)** 이 먼저 실행되어 코드의 **매크로** 정보를 수집해 `.generated.h`와 `.generated.cpp` 파일을 생성함

* 생성된 파일과 함께 컴파일을 진행해 리플렉션을 지원함

* 언리얼 엔진에서 주로 사용하는 매크로는 아래와 같음

  |매크로 | 기능|
  |---|---|
  |`UCLASS()`| 해당 C++ 클래스를 리플렉션 시스템에 등록|
  |`UPROPERTY()`| 변수를 리플렉션 시스템에 등록|
  |`UFUNCTION()`| 멤버 함수를 리플렉션 시스템에 노출 등록|
  |`USTRUCT()`| 구조체를 리플렉션 시스템에 등록|

### Unreal Engine Garbage Collection

* `UObject`를 기반으로 하는 객체만 관리하여 더 이상 사용되지 않는다고 판단되는 객체들을 **주기적**으로 해제함

* `GC`는 게임에서 삭제되지 않는 핵심 객체들의 집합인 **루트셋**에서 시작해 **마크 앤 스윕 알고리즘**으로 **도달 가능한지** 판단하고 도달하지 못하는 객체들을 삭제함

* `GC`가  객체를 추적하기 위해서는 `UPROPERTY()` 매크로를 등록하여 엔진이 해당 데이터가 어떤 `UObject`를 가리키는지 알게 해줘야 함

## 4. 퍼즐 어드벤처 디자인하기

### Widget Blueprint

* 게임의 UI를 **시각적**으로 설계할 수 있도록 지원하는 에디터

* 시각적인 화면을 구성하는 **디자이너**, 기능을 구현하는 **그래프**의 두 가지 모드로 분류됨

* **디자이너**

    * **Palette:** UI에 추가할 수 있는 다양한 위젯 타입이 모두 포함되어 있음, 드래그로 배치 가능

    * **Hierarchy:** UI 컴포넌트의 구조를 보여줌, 위젯 간의 부모-자식 관계를 설정하며 이는 UI 레이아웃과 렌더링에 영향을 줌

    * **Visual Grid:** UI를 시각적으로 확인할 수 있는 그리드

    * **Details:** 선택한 위젯의 속성을 수정, **Is Variable**을 체크해야 그래프 모드에서 제어 가능함

* **그래프:** 일반 블루프린트의 그래프와 동일
