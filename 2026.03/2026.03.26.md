### [다리를 지나는 트럭](https://school.programmers.co.kr/learn/courses/30/lessons/42583)

```cpp
#include <string>
#include <vector>
#include <queue>

using namespace std;

int solution(int bridge_length, int weight, vector<int> truck_weights) {
    int current_bridge_weight = 0;
    int time = 0;
    queue<int> trucks;
    queue<pair<int, int>> bridge;
    
    for(const int& truck : truck_weights) 
        trucks.push(truck);
    
    while(!trucks.empty() || !bridge.empty()){
        time++;
        
        // 다리에서 트럭 빼기
        if(!bridge.empty()){
            const pair<int, int>& first_truck = bridge.front();
            if(time >= first_truck.second){
                bridge.pop();
                current_bridge_weight -= first_truck.first;
            }
        }
        
        // 다리에 트럭 넣기
        if(!trucks.empty()){
            const int& truck_weight = trucks.front();
            if(bridge.size() < bridge_length && current_bridge_weight + truck_weight <= weight){
                bridge.push({truck_weight, time + bridge_length});
                current_bridge_weight += truck_weight;
                trucks.pop();
            }
        }
    }
    return time;
}
```

* **큐**를 이용하는 문제
* **다리를 큐**로 구현해 여분의 중량과 트럭 대수가 있으면 트럭을 넣음
* 도착 예정 시간을 큐에 함께 저장하여 도착시간이 되면 큐에서 뺌

### [N으로 표현](https://school.programmers.co.kr/learn/courses/30/lessons/42895)

```cpp
#include <string>
#include <vector>
#include <set>

using namespace std;

int solution(int N, int number) {
    int answer = 10;
    vector<set<int>> dp(10);
    dp[1].insert(N);
    
    for(int i=2; i<9; i++){
        int temp = N;
        for(int j=1; j<i; j++){
            temp *= 10;
            temp += N;
        }         
        dp[i].insert(temp);
        
        for(int j=1; j<i; j++){
            for(int k=1; k<i; k++){
                if(j+k == i){
                    for(const int& a : dp[j]){
                        for(const int& b : dp[k]){
                            dp[i].insert(a+b);
                            if(a > b) dp[i].insert(a-b);
                            dp[i].insert(a*b);
                            if(b != 0) dp[i].insert(a/b);
                        }
                    }
                }
            }
        }
    }
    
    for(int i=1; i<10; i++)
        for(int a : dp[i])
            if(a == number && i < answer) 
                return i;

    return -1;
}
```

* **DP**와 **완전탐색**을 둘 다 이용하는 문제
* DP에는 **N개의 수**를 이용하여 **만들 수 있는 숫자**들을 저장함
* **N개의 수**를 이용해 만들 수 있는 수는 `M + K = N`인 **DP[M]**, **DP[K]** 의 수를 사칙연산 한 수임
* 가능한 모든 수를 탐색하여 DP에 저장한 다음 조건에 만족하는 N을 찾음

### [파괴되지 않은 건물](https://school.programmers.co.kr/learn/courses/30/lessons/92344)

```cpp
#include <string>
#include <vector>

using namespace std;

int solution(vector<vector<int>> board, vector<vector<int>> skill) {
    int answer = 0;
    vector<vector<int>> pref(board.size()+1, vector<int>(board[0].size() + 1, 0));
    
    // 시작점, 끝점 마크
    for(const vector<int>& data : skill){
        int type = data[0];
        int r1 = data[1], c1 = data[2], r2 = data[3], c2 = data[4];
        int degree = type == 1 ? -data[5] : data[5];
        
        pref[r1][c1] += degree;
        pref[r1][c2+1] -= degree;
        pref[r2+1][c1] -= degree;
        pref[r2+1][c2+1] += degree;
    }

    // 가로 방향 누적합
    for(int i=0; i<pref.size(); i++){
        int curr = 0;
        for(int j=0; j<pref[i].size(); j++){
            curr += pref[i][j];
            pref[i][j] = curr;
        }
    }
    
    // 세로 방향 누적합
    for(int i=0; i<pref[0].size(); i++){
        int curr = 0;
        for(int j=0; j<pref.size(); j++){
            curr += pref[j][i];
            pref[j][i] = curr;
        }
    }
    
    // 누적합 결과와 보드를 합
    for(int i=0; i<board.size(); i++){
        for(int j=0; j<board[i].size(); j++){
            board[i][j] += pref[i][j];
            if(board[i][j] > 0) answer++;
        }
    }
    
    return answer;
}
```

* **2차원 누적합**을 이용하는 문제
* **차분 배열**을 누적합하여 건물의 총 피해량 및 회복량을 계산

## 3. 추상 클래스와 인터페이스
* CH2 팀 프로젝트를 진행하며 Item 클래스를 만드는데 인터페이스와 추상 클래스 중 무엇을 선택할지 고민함
* 예시의 UML을 따라 인터페이스로 구현했지만, 아이템의 종류가 적어 모든 아이템의 인터페이스 함수가 동일하게 작성됨
* 이번 기회에 추상 클래스와 인터페이스의 주된 차이, 사용처를 공부해 봄

### 추상 클래스 (Abstract Class)
* 내부에 하나 이상의 **순수 가상 함수**를 포함하는 클래스
* **일반 함수**, **멤버 변수**를 가질 수 있음
* **IS-A** 관계

### 인터페이스 (Interface)
* **모든 멤버 함수**가 순수 가상 함수이고 **멤버 변수가 없는** 클래스
* 반드시 **가상 소멸자**를 가져야함
* **다중 상속**이 가능함
* **CAN-DO** 관계

### 공통점 및 차이점
* **공통점**
    * 순수 가상 함수를 포함하므로, 인스턴스화가 불가능함
    * 다형성을 제공함

* **차이점**
    * 추상 클래스는 **멤버 변수**나 **일반 함수**를 가질 수 있지만, 인터페이스는 **순수 가상 함수**만 가질 수 있음
    * 추상 클래스는 **다중 상속이 불가능**하지만, 인터페이스는 **다중 상속**이 필요할 때 주로 사용
