## 2. 프로그래머스  [멀쩡한 사각형](https://school.programmers.co.kr/learn/courses/30/lessons/62048) 문제 풀이

```cpp
#include <numeric>
using namespace std;

long long solution(int w,int h) {
    long long answer = 0;
    long long lw = w, lh = h;        // 오버플로우 대비 long long형 변경
    long long ratioW, ratioH;        // 가로 세로 비 저장
    long long gcdWH = gcd(lw, lh);   // 가로, 세로의 최소 공배수
    ratioW = lw / gcdWH;             // 가로의 비
    ratioH = lh / gcdWH;             // 세로의 비
    
    answer = (lw * lh) - ((ratioW + ratioH - 1) * gcdWH); // 사용하지 못하는 사각형 개수는 가로 + 세로 -1
    
    return answer;
}
```

## 3. 디자인 패턴

### 싱글톤(Singleton) 패턴
* 클래스의 인스턴스를 **하나만** 생성해 이를 공유하여 사용하는 것

* **메모리 효율성**, **데이터 공유**, **명확성**이 향상되지만

* 하지만 **결합도 증가**, **유닛 테스트의 어려움**, **멀티스레드 환경에서의 동기화 처리** 등의 단점이 있음

* 대체로 **전역 변수**와 유사한 장단점을 가지고 작동함

* 아래 예제는 **`Meyers' Singleton`** 방식으로 구현한 코드이며 이 방식은 `C++11` 이후 버전에서 멀티스레드 환경에서도 안전하게 생성됨

* 하지만 데이터 접근 시 `Race Condition`이 발생할 수 있으며, 이는 `std::mutex`를 활용하여 동기화 처리를 해야함

    ```cpp
    #include <iostream>
    #include <string>

    class Singleton {
    public:
        static Singleton& getInstance() {
            // 처음 호출될 때 단 한 번만 생성 (지연 초기화)
            static Singleton instance;
            return instance;
        }

        // 복사 및 참조 방지
        Singleton(const Singleton&) = delete;
        Singleton& operator=(const Singleton&) = delete;

    private:
        // 생성자를 private으로 설정
        Singleton() {
            std::cout << "Singleton 인스턴스 생성" << std::endl;
        }

        // 소멸자를 private으로 설정
        ~Singleton() {
            std::cout << "Singleton 인스턴스 소멸" << std::endl;
        }
    };

    int main() {
        Singleton& s1 = Singleton::getInstance(); // Output: Singleton 인스턴스 생성
        Singleton& s2 = Singleton::getInstance(); // 이미 생성된 인스턴스 참조를 반환

        return 0;
    } // Output: Singleton 인스턴스 소멸
    ```

### 데코레이터(Decorator) 패턴
* 기존의 클래스 코드를 수정하지 않고, `wrapper` 객체를 덧붙여 기능을 추가하는 것

* 상속과 달리 객체를 **새로운 객체 안에 포함**하는 식으로 구현

* 데코레이터는 아래의 네 가지 구성 요소를 가짐
    * `Component`: 기본 구조를 정의하는 추상 클래스 또는 인터페이스
    * `ConcreteComponent`: 실제 기능을 구현한 클래스
    * `Decorator`: 자신이 감싸고 있는 객체에 기능을 추가
    * `ConcreteDecorator`: 덧붙일 기능을 구현한 클래스

    ```cpp
    #include <iostream>
    #include <string>

    using namespace std;

    // Component
    class Text {
    public:
        virtual string draw() = 0;
        virtual ~Text() {}
    };

    // ConcreteComponent
    class PlainText : public Text {
    public:
        string draw() override { return "Text"; }
    };

    // Decorator
    class TextDecorator : public Text {
    protected:
        Text* wrappedText; // 원본 객체를 감쌈 (Wrapper)
    public:
        TextDecorator(Text* t) : wrappedText(t) {}
        ~TextDecorator(){delete wrappedText;}
    };

    // 4. ConcreteDecorator
    class AddDecorator : public TextDecorator {
    public:
        AddDecorator(Text* t) : TextDecorator(t) {}
        string draw() override {
            return wrappedText->draw() + "+"; 
        }
    };

    class SubDecorator : public TextDecorator {
    public:
        SubDecorator(Text* t) : TextDecorator(t) {}
        string draw() override {
            return wrappedText->draw() + "-"; 
        }
    };

    int main() {
        Text* myText = new PlainText(); // 기본 객체 생성
        cout << myText->draw() << endl; // Output: Text

        myText = new AddDecorator(myText);
        cout << myText->draw() << endl; // Output: Text+

        myText = new SubDecorator(myText);
        cout << myText->draw() << endl; // Output: Text+-
        
        delete myText;
        return 0;
    }
    ```

### 옵저버(Observer) 패턴
* 객체간 **일대다** 관계를 정의하여 하나의 `주체(Subject)`의 상태가 변경되면 이 상태를 관찰하고 있는 다수의 객체인 `관찰자(Observer)`에게 알림을 보내는 디자인 패턴

* 옵저버 패턴은 크게 **Subject**와 **Observer**로 구성
    * **Subject:** 상태를 관리하며 등록된 옵저버들에게 **알림(Notify)** 을 보냄
    * **Observer:** 데이터의 변경을 전달받는 인터페이스

* **Subject**는 **Observer**가 구체적으로 어떤 클래스인지 알 필요 없이 인터페이스만 구현되어 있으면 연결 가능함

* 기본적으로 알림이 가는 순서가 정해지지 않으며, 구현하더라도 **결합성**과 **복잡성**이 올라가 권장하지 않음
    ```cpp
    #include <iostream>
    #include <vector>
    #include <string>
    #include <algorithm>

    using namespace std;

    // Observer Interface
    class IObserver {
    public:
        virtual ~IObserver() {}
        virtual void update(string message) = 0;   
    };

    // Subject
    class Subject {
    private:
        vector<IObserver*> observers;
        string message;

    public:
        // 옵저버 등록
        void attach(IObserver* observer) {
            observers.push_back(observer);
        }

        // 옵저버 해제
        void detach(IObserver* observer){
            observers.erase(remove(observers.begin(), observers.end(), observer), observers.end());
        }

        // 메시지 업데이트
        void setMessage(string newMessage) {
            message = newMessage;
            notify();
        }

        // 옵저버들에게 알림
        void notify() {
            for (IObserver* observer : observers) {
                observer->update(message);
            }
        }
    };

    // Concrete Observer
    class Observer : public IObserver {
    private:
        string name;

    public:
        Observer(string _name) : name(_name) {}

        void update(string message) override {
            cout << name << " Observer: " << message  << endl;
        }
    };

    int main() {
        Subject subject;
        Observer* A = new Observer("A");
        Observer* B = new Observer("B");
        
        subject.attach(A);
        subject.attach(B);

        subject.setMessage("Subject Update"); // Output: A Observer: Subject Update
                                              //         B Observer: Subject Update

        subject.detach(A);
        subject.detach(B);
        
        delete A;
        delete B;
        
        return 0;
    }
    ```
