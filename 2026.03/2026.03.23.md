### [택배상자](https://school.programmers.co.kr/learn/courses/30/lessons/131704)

```cpp
#include <string>
#include <vector>
#include <stack>

using namespace std;

int solution(vector<int> order) {
    int answer = 0;
    stack<int> sub;
    int cur = 0;
    int n = 1;
    while(n <= order.size()+1){
        // 현재 상자와 order의 상자가 같을 때
        if(cur < order.size() && order[cur] == n){
            answer++;
            cur++; 
            n++;
        }
        
        // 스택의 탑에 order의 상자가 있을 때
        else if(!sub.empty() && order[cur] == sub.top()){
            sub.pop();
            answer++;
            cur++;
        }
        
        // 둘 다 아니면 스택에 현재 상자를 넣음
        else{
            sub.push(n);
            n++;
        }   
    }
    
    return answer;
}
```

* Stack, 구현 문제
* 특별한 알고리즘 없이, 스택 자료구조를 활용하여 문제 조건을 그대로 구현

### [110 옮기기](https://school.programmers.co.kr/learn/courses/30/lessons/77886)

```cpp
#include <string>
#include <vector>

using namespace std;
int getCount(string& str){
    string result = ""; // 110을 지울 문자열
    int count = 0;      // 110의 개수
    for(int i=0; i<str.size(); i++){
        result += str[i];
        if(result.length() >= 3 && result.substr(result.length()-3) == "110"){
            result.erase(result.length()-3);
            count++;
        }
    }
    str = result;   // 110을 삭제한 문자열을 str에 저장
    return count;   
}

vector<string> solution(vector<string> s) {
    for(string& str : s){
        int count = 0;
        int result = getCount(str);     // str의 110들을 삭제하고 110의 개수를 저장
        
        for(int i=str.length()-1; i>=0; i--){
            if(str[i] == '0') break;    // 가장 뒤의 0 위치를 찾으면 중단
            else str.erase(i, 1);       // 1이면 지움
            count++;                    // 지운 1의 개수를 저장
        }
        for(int i=0; i<result; i++) str += "110";   // 가장 뒤의 0 뒤에 110을 붙임
        for(int i=0; i<count; i++) str += '1';      // 지운 1의 개수만큼 1을 뒤에 붙임
    }
    return s;
}
```

* 문자열을 순회하여 존재하는 **110**을 전부 제거 후 개수를 저장
* 이후 **110**이 제거된 문자열의 가장 뒤 `0`을 찾은 뒤 그 뒤에 **110** 을 붙임
* 처음에는 `substr`, `erase`를 사용해서 **110**을 지워 `O(n^2)`의 시간 복잡도 가짐, 이 방법으로도 정답이었지만 아슬아슬하게 정답 처리된 듯
* 이후에 스택으로 지우게 처리해 `O(n)`으로 최적화

### [표현 가능한 이진트리](https://school.programmers.co.kr/learn/courses/30/lessons/150367)

```cpp
#include <string>
#include <vector>

using namespace std;

// 2진수 문자열로 변환
string numberToBinary(long long number){
    string result = "";
    while(number > 0){
        result += number % 2 == 0 ? '0' : '1';
        number /= 2;
    }
    return result;
}

// 양쪽 자식노드가 조건에 만족하는지
bool checkChild(const string& binary, int root, int dist, bool flag){
    if(dist == 0) return true;              // 리프노드면 true 반환
    int left = root - dist;                 // 왼쪽 자식 노드 인덱스
    int right = root + dist;                // 오른쪽 자식 노드 인덱스
    bool canLeft = true, canRight = true;   // 자식 노드 조건 만족 여부
    bool rootFlag = flag;                   // 이전 부모 중 더미 노드 존재 여부
    if(binary[root] == '0')                 // 현재 노드가 더미 노드면
        rootFlag = false;
    
    // 이전에 더미 노드가 있었고, 자식 노드 중 1인 노드가 있으면
    if(!rootFlag && (binary[left] == '1' || binary[right] == '1')) return false;
    
    // 좌우 자식 확인
    canLeft = checkChild(binary, left, dist / 2, rootFlag);
    canRight = checkChild(binary, right, dist / 2, rootFlag);
    
    return canLeft && canRight;
}

vector<int> solution(vector<long long> numbers) {
    vector<int> answer(numbers.size(), 0);
    vector<int> numNodes = {1, 3, 7, 15, 31, 63};   // 포화 이진 트리의 노드 개수
    
    for(int i=0; i<numbers.size(); i++){
        string binary = numberToBinary(numbers[i]);
        int length = binary.length();
        
        // 노드 개수를 포화 이진 트리로 맞춰줌
        for(int numNode : numNodes){
            if(length <= numNode){
                int addLength = numNode - length;
                for(int j=0; j<addLength; j++)
                    binary += '0';
                break;
            }
        }
        
        int root = (binary.length() - 1) / 2;
        if(checkChild(binary, root, (root+1)/2, true)) answer[i] = 1;
    }
    return answer;
}
```

* 문제의 조건을 만족하기 위해서는 `1`의 값을 갖는 어떤 **자식 노드의 모든 부모 노드는** `1`의 값을 가져야 함
* 이진 트리를 재귀적으로 탐색하여 위 조건을 만족하는지 확인
* 탐색 중 조건에 어긋나는 노드가 있으면 바로 정지

## 3. Git 기초

### add, commit, push

* **add:** 작업 디렉토리(Working Directory)의 변경 내용을 **Staging Area**에 올리는 작업
    * `git add <파일명>`, `git add .` (전체 파일)

* **commit:** Staging Area에 올라온 파일들을 하나의 **버전(Snapshot)** 으로 로컬 저장소에 기록
    * `git commit -m "커밋 메시지"`

* **push:** 로컬 저장소에서 완료된 커밋들을 **원격 저장소(Remote Repository)** 로 업로드
    * `git push origin <브랜치명>`

### fetch, pull

* **fetch:** 원격 저장소의 최신 이력을 로컬로 가져오기만 하고, 내 코드와 합치지는 않음
    * `git fetch origin`
* **pull:** 원격 저장소의 내용을 가져와서 현재 내가 작업 중인 로컬 브랜치에 **병합(Merge)**
    * `git pull origin <브랜치명>`
### branch
* 메인 코드에서 분리되어 **독립적인 작업 공간**을 만드는 기능
    * **생성:** `git branch <브랜치명>`
    * **삭제:** `git branch -D <브랜치명>`
    * **이름 변경:** `git branch -m <이전 브랜치명> <새로운 브랜치명>`

* **checkout:** 현재 작업 중인 브랜치에서 **다른 브랜치로 전환**, 특정 커밋 상태로 이동
    * **이동:** `git checkout <브랜치명>`
    * **생성 및 이동:** `git checkout -b <브랜치명>`

### merge, conflict
* **merge:** 분리되어 작업하던 브랜치를 다시 하나로 **합치는 과정**, 보통 개발 브랜치에서 작업 후 `main` 으로 합칠 때 사용
    * `git checkout main` -> `git merge <기능브랜치명>`

* **conflict:** 서로 다른 브랜치에서 **같은 파일의 같은 부분을 동시에 수정**하고 merge할 때 발생
    * `<<<<<<< HEAD`, `=======`, `>>>>>>> <충돌 브랜치명>` 사이의 내용을 수정한 후 다시 커밋하여 해결

## 4. 나이아가라 VFX 시스템 (Niagara VFX System)
* 언리얼 엔진에서 **비주얼 이펙트**를 제작하는 데 사용되는 주요 툴
* 나이아가라는 `System - Emitter - Module`의 계층 구조를 가짐
* 나이아가라 VFX 시스템에는 네 개의 핵심 컴포넌트가 존재함

### 시스템 (System)
* 가장 최상위의 컨테이너, 레벨에 실제로 스폰되는 **최종 객체**
* 여러 개의 `이미터`를 하나로 묶어 관리하고, 전체 이펙트의 타임라인을 제어
* 시스템 에디터의 `타임라인(Timeline)` 패널에서 시스템에 어떤 `이미터`가 포함되어 있는지 확인하고, 관리할 수 있음

### 이미터 (Emitter)
* 나이아가라 시스템에서 **파티클이 생성**되는 곳
* 시각적 효과를 구성하는 **개별 논리 단위**이며 하나의 시스템에는 여러 이미터가 존재 할 수 있음
* 이미터는 스택으로 구성되고 스택 내에는 여러 그룹이 존재함
* 각 그룹은 순차적으로 실행되며 각 그룹의 역할은 아래와 같음
    1. `이미터 스폰(Emitter Spawn):` 이미터가 처음 활성화될 때 **단 한 번 실행**되며, **초기 구성 및 디폴트 값을 정의**
    2. `이미터 업데이트(Emitter Update):` 이미터가 살아있는 동안 **매 프레임 실행**, 파티클의 **스폰을 정의**
    3. `파티클 스폰(Particle Spawn):` 파티클이 처음 생성될 때 한번 실행되며, **파티클의 초기값**을 설정
    4. `파티클 업데이트(Particle Update):` 생성된 모든 파티클에 대해 매 프레임 실행, **파티클의 수명**에 따라 프레임별로 변경해야 하는 사항을 정의
    5. `이벤트 핸들러(Event Handler):` 특정 이벤트가 발생했을 때 호출, 연쇄 반응을 만들 때 주로 사용
    6. `렌더(Render):` 모든 물리 연산과 데이터 갱신이 끝난 후, **최종적으로 화면에 그릴 때 실행**

### 모듈 (Module)
* 실제 수학적 연산과 물리 로직을 수행하는 컨테이너
* 이미터 내부의 `모듈 스택`에 저장되어 순차적으로 실행됨
* 내부적으로는 `HLSL(High-Level Shading Language)` 스크립트로 작성되어 필요시에 수정 가능하며, **노드**를 통해 시각적으로 빌드도 가능

### 파라미터 (Parameter)
* 나이아가라 시뮬레이션 데이터의 **추상 표현(변수)**
* 모듈들이 읽고 쓰는 **데이터 값**
* 변수의 `Scope`를 명확히 하기 위해 `네임스페이스(Namespace)` 개념을 사용
