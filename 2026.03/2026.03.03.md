## 2. C언어 학습

### 포인터 연산자 우선순위
* `C/C++ 라이브세션 강의 자료`의 `포인터` 부분의 예제 중 출력을 정확히 예상하지 못함 
* **증감 연산자**와 **역참조 연산자**의 우선순위와 포인터와 관련된 연산자들의 우선순위를 학습

  ```cpp
  #include <stdio.h>

  int main(void)
  {
      int Nums[3] = { 12, 34, 56 };
      int* Ptr = Nums;

      int Num1 = *Ptr++;    // *(Ptr++) Ptr의 현재값을 참조 후 주소 이동(후위)
      int Num2 = *++Ptr;    // *(++Ptr) Ptr의 주소 이동 후 참조(전위)
      int Num3 = ++*Ptr;    // ++(*Ptr) Ptr의 주소 참조 후 값 증가(전위)
      int Num4 = (*Ptr)++;  // (*Ptr)++ Ptr의 주소 참조 후 값 증가(후위)

      printf("%d %d %d %d", Num1, Num2, Num3, Num4); 
      // Output: 12 56 57 57

      return 0;
  }
  ```

  | 우선순위 | 연산자 | 기호 | 결합 방향 | 설명 |
  | --- | --- | --- | --- | --- |
  | **1** | 후위 증감 | `a++`, `a--` | `L → R` | 연산 후 값 변경 |
  |  | 멤버 접근 | **`->`**, `.` | `L → R` | **포인터를 통한 구조체 접근** |
  |  | 배열 첨자 | **`[]`** | `L → R` | **포인터 인덱스 접근** |
  | **2** | 전위 증감 | `++a`, `--a` | **`R → L`** | 값 변경 후 연산 |
  |  | 역참조 | **`*`** | **`R → L`** | **주소로 찾아가기** |
  |  | 주소 추출 | **`&`** | **`R → L`** | **변수의 주소 값 얻기** |
  |  | 캐스트 | `(type)` | **`R → L`** | 포인터 형변환 |

### typedef
* 기존의 자료형에 새로운 별칭을 부여하는 키워드 
* 주로 접미사 `_t`를 붙여 사용
* 기본 자료형 축약, 구조체 선언 등에서 사용
* **typedef**는 다양한 방식으로 선언할 수 있으며 아래의 예제와 같음

  ```cpp
  // Sample 1
  struct Info
  {
    int age;
    char name[20];
  };
  typedef struct Info Info_t;

  // Sample 2
  typedef struct Info
  {
    int age;
    char name[20];
  } Info_t;

  // Sample 3
  typedef struct
  {
    int age;
    char name[20];
  } Info_t;
  ```

### Deep Copy, Shallow Copy

* **Deep Copy(깊은 복사)** 
  * 새로운 메모리 공간을 할당받은 뒤, 그 안에 **원본 데이터**를 복사
  * 복사본을 수정해도 **원본에 영향이 없음**
  * 소멸 시 독립적으로 해제 가능
* **Shallow Copy(얕은 복사):**
  * 포인터 변수가 가리키는 **주소값**만 복사
  * 복사본을 수정하면 **원본의 데이터도 수정**
  * 복사본을 소멸하면 원본 데이터는 해제된 메모리를 가리키는 **Dangling Pointer** 발생

## 3. 발사체 구현

### 충돌 함수 구현
* `FirstPersonProjectile` **클래스**에 액터가 다른 컴포넌트나 액터와 충돌할 때 호출되는 `AActor`의 함수 `OnHit()`로 발사체의 충돌 이벤트를 정의

  ```cpp
  UFUNCTION()
  void OnHit(UPrimitiveComponent* HitComp, AActor* OtherActor, UPrimitiveComponent* OtherComp, FVector NormalImpulse, const FHitResult& Hit);
  ```

  | 매개변수 | 타입 | 설명 |
  | --- | --- | --- |
  | **`HitComp`** | `UPrimitiveComponent*` | 부딪힌 컴포넌트 |
  | **`OtherActor`** | `AActor*` | 부딪힌 액터 (벽, 적, 장애물 등) |
  | **`OtherComp`** | `UPrimitiveComponent*` | 충돌을 생성한 컴포넌트 |
  | **`NormalImpulse`** | `FVector` | 충돌의 노멀 충격량 |
  | **`Hit`** | `FHitResult` | 시간, 거리, 위치 같은 충돌 이벤트의 데이터|

### 발사체 설정
* `FirstPersonProjectile` **클래스**에서 발사체의 움직임을 처리하기 위해 `UProjectileMovementComponent`을 추가한 후 초기화 함
* `UProjectileMovementComponent`는 매 프레임 위치를 계산하여 발사체의 궤적과 충돌 후 반응을 제어하며, 주로 사용하는 속성은 아래와 같음

  | 분류 | 속성명 (Property) | 타입 | 기본값 | 설명 |
  | --- | --- | --- | --- | --- |
  | **속도 제어** | **`InitialSpeed`** | `float` | `0.0` | 발사체의 **초기 속력** |
  |  | **`MaxSpeed`** | `float` | `0.0` | 발사체의 **최대 속력** (0은 무제한) |
  | **회전/방향** | **`bRotationFollowsVelocity`** | `bool` | `false` | 발사체의 메쉬가 이동 방향에 맞춰 **회전할지** 여부 |
  |  | **`bShouldBounce`** | `bool` | `false` | 벽이나 장애물에 충돌했을 때 파괴되지 않고 **튕길지** 여부 |
  | **물리/중력** | **`ProjectileGravityScale`** | `float` | `1.0` | 발사체에 적용되는 **중력의 배율** |
  |  | **`Bounciness`** | `float` | `0.6` | 튕길 때의 **탄성 수치** |
  |  | **`Friction`** | `float` | `0.0` | 튕기거나 표면을 긁을 때 적용되는 **마찰력** |
  | **컴포넌트** | **`UpdatedComponent`** | `USceneComponent*` | `nullptr` | 이 무브먼트 컴포넌트가 **이동시키고 제어할 대상** |

### 캐릭터 카메라 방향 구하기

* 발사체의 발사 방향을 파악하기 위해 `LineTraceSingleByChannel()` 함수를 통해 플레이어가 바라보고 있는 `월드 좌표`를 구함
* `LineTraceSingleByChannel()`은 시작 지점에서 끝 지점까지 가상의 선을 그려 가장 먼저 충돌하는 액터를 찾는 함수이며 아래와 같은 형태를 가짐

  ```cpp
  bool LineTraceSingleByChannel(
      struct FHitResult& OutHit,                      // 충돌 결과 저장
      const FVector& Start,                           // 시작 지점
      const FVector& End,                             // 끝 지점
      ECollisionChannel TraceChannel,                 // 충돌 채널
      const FCollisionQueryParams& Params,            // 세부 설정
      const FCollisionResponseParams& ResponseParam   // 응답 설정
  ) const
  ```
  | 매개변수 | 타입 | 설명 |
  | --- | --- | --- |
  | **`OutHit`** | `FHitResult&`  | 충돌 시 발생한 상세 정보를 저장 |
  | **`Start`** | `FVector`   | 월드 좌표상의 `Ray` 시작 위치 (보통 카메라나 총구 위치) |
  | **`End`** | `FVector`  | 월드 좌표상의 `Ray` 종료 위치, `Start + (방향 * 거리)` 공식을 주로 사용 |
  | **`TraceChannel`** | `ECollisionChannel` | 어떤 종류의 물체와 부딪힐지 결정 |
  | **`Params`** | `FCollisionQueryParams` | 트레이스를 어떻게 수행할지 설정 |
  | **`ResponseParams`** | `FCollisionResponseParams` | 트레이스가 특정 채널을 만났을 때 반응을 설정(`ECR_Ignore`, `ECR_Overlap`, `ECR_Block`) |

* `TraceChannel`로 Ray가 어떤 물체와 부딪힐지 설정할 수 있으며 `ECollisionChannel`의 주요값은 아래와 같음
  | 채널명 (Enum) | 설명 |
  | --- | --- |
  | **`ECC_WorldStatic`** | 움직이지 않는 바닥, 벽, 천장 등 고정된 지형지물 |
  | **`ECC_WorldDynamic`** | 움직이는 발판, 열리는 문, 인터랙션이 있는 소품 등 가변적인 물체 |
  | **`ECC_Pawn`** | 플레이어나 AI 캐릭터 |
  | **`ECC_Visibility`** | 눈에 보이는 모든 물체 |
  | **`ECC_Camera`** | 카메라 전용 |
  | **`ECC_PhysicsBody`** | `Simulate Physics`가 활성화되어 실시간 물리 연산이 적용되는 물체 |
  | **`ECC_Vehicle`** | 자동차나 비행기 등 탑승 가능한 물체 |
  | **`ECC_Destructible`** | 부서질 수 있는  물체 |
