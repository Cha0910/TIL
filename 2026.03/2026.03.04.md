## 2. C++ Template(템플릿)

### Template

* 여러 타입으로 동작할 수 있는 코드를 만드는 틀, 이를 통해 일반화 프로그래밍(Generic Programming)을 구현할 수 있음
* 특정 데이터 타입에 종속되지 않고, **타입을 매개변수화**하여 함수나 클래스를 생성하는 도구

### Function Template (함수 템플릿)

* 하나의 함수로 여러 개의 타입을 처리할 때 사용
* 함수의 로직은 같은데 다루는 데이터 타입이 다를 때 사용
* 타입을 정해주는 `명시적 호출`, 컴파일러가 인자로 추론하는 `암시적 호출`이 있음

  ```cpp
  template <typename T>
  T Add(T a, T b) 
  {
      return a + b;
  }

  // 사용 예시
  Add<int>(3, 5);      // T가 int로 치환됨 (명시적 호출)
  Add(1.1, 2.2);       // T가 double로 치환됨 (암시적 호출)
  ```

### Class Template (클래스 템플릿)

* 여러 개의 데이터 타입을 담아야 하는 클래스 구조를 만들때 사용
* `컨테이너 클래스(Stack, Vector)`같은 다양한 타입을 담아야 하는 구조에 주로 사용

  ```cpp
  template <typename T>
  class Box 
  {
  private:
      T data;
  public:
      Box(T d) : data(d) {}
      T GetData() { return data; }
  };

  // 사용 예시
  Box<int> intBox(100);      // int를 담는 Box 객체 생성
  Box<string> strBox("TIL"); // string을 담는 Box 객체 생성
  ```


### Template과 헤더 파일

* **문제:** 과제 3번을 진행하며 `Inventory.h`에 Template를 선언한 후 `Inventory.cpp`에 구현하는 식으로 코드를 작성했지만, 컴파일 중 **링커 오류**가 발생함

* **원인:** 
  * 템플릿 함수는 컴파일 중 컴파일러가 `Inventory.cpp`의 구현을 읽어도 타입을 알 수 없기 때문에 기계어 코드를 생성하지 않고 어딘가에서 함수를 호출 하면 기계어 코드를 생성하는 **Instantiation(구체화)** 단계를 가짐

  * 어딘가에서 템플릿 함수를 호출했을 때 `Inventory.h`만 읽을 수 있으므로 함수의 구현은 알 수 없어 기계어 코드를 생성하지 않게 됨

  * 오브젝트 파일에 템플릿 함수의 기계어 코드가 없어 링커 오류가 발생한 것

* **해결:** 이를 해결하기 위해 `Inventory.h`에 템플릿 함수의 선언과 구현을 작성하여 기계어 코드를 생성할 수 있게 작성해야 함
